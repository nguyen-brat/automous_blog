Entropy-Aware Adjusted Base Frequency for Extending Context Window of Large Language Models

This paper is fixing a very specific long-context failure. A RoPE-based model can be pushed beyond its pretraining window, but once more tokens start competing for attention, the model can stop concentrating well enough to use that extra space. The answer here is not a new transformer block. It is a two-part repair: change RoPE’s base frequency with ABF, then add a layer-aware attention-logit scaling rule that keeps later layers focused without disturbing the original window.

You can keep the whole method in your head as a short pipeline:

The paper treats long-context failure as two related but different problems. One is positional geometry drifting away from what the model knows. The other is attention becoming too spread out as the sequence gets longer. ABF tackles the first problem. Dynamic logit scaling tackles the second.

Why long prompts make attention blur

If you imagine attention as a spotlight, a longer prompt simply creates more things for that spotlight to illuminate. Softmax does not automatically keep the spotlight narrow just because the sequence is longer. As the number of tokens grows, attention mass can get spread across too many candidates, and the model starts to lose the sharp focus it needs for long-range reasoning.

The paper measures that effect with an entropy-style score over the attention distribution. Figure 1 is the clue that motivates the whole method.

What to notice is not the exact value of the score, but the shape of the curves.

  • The first two layers are the ones that still show a steady rise as the context grows.
  • The later layers stay much more similar even when the number of contextual tokens doubles from 1,024 to 2,048.
  • That tells the paper that some layers already behave in a length-invariant way, while the earliest layers still need to be protected from extra distortion.

Intuition: if attention starts spreading across many more tokens, this entropy-style statistic moves with it. The uniform-attention reference in the figure marks the most diffuse end of the scale.

Symbols:

  • p_i: attention probability assigned to contextual token i
  • i: index over contextual tokens

Read-aloud meaning: add up each attention probability times its log to get one score for how spread out the attention distribution is.

Why it matters: this is the diagnostic the paper uses to justify layer-aware scaling. The method is not just trying to make long contexts possible. It is trying to keep the later layers from losing concentration as those contexts grow.

That observation leads to the paper’s main design choice: do not apply the same sharpening everywhere. The first two layers are already behaving differently from the rest, so the intervention should match that structure instead of overriding it.

ABF changes the positional map first

ABF, or adjusted base frequency, is the quieter half of the method. RoPE uses a base frequency b to determine how fast each rotary band turns. The paper makes that base much larger, at 50,000. In plain language, that slows the rotary phase progression so the long-context version of the model stays closer to the positional geometry it already knows from pretraining.

Formula card: RoPE frequency ladder

Purpose: show how RoPE turns one base frequency into a stack of phase speeds.

Equation:

θj=b−2j/d\theta_j = b^{-2j/d}θj​=b−2j/d

Intuition: the base b controls how quickly the rotary bands advance. Raising b stretches the phase ladder, so the model sees a gentler change in position as tokens move farther apart.

Symbols:

  • \theta_j: angular frequency of the j-th rotary band
  • b: RoPE base frequency
  • j: feature-pair index
  • d: attention-head dimension

Read-aloud meaning: each RoPE band gets its own frequency, and the base frequency determines how quickly those bands rotate.

Why it matters: ABF is not a new attention mechanism. It is a gentler coordinate system for longer sequences, which makes the fine-tuning problem easier to learn.

That matters because a lot of long-context tricks only work by forcing the model into a more aggressive remapping. ABF is trying to do less violence to the original representation. It preserves the pretrained behavior inside the original window better than methods that uniformly squeeze every position, and that makes fine-tuning more productive.

The spotlight gets sharper only where it needs to

The second half of the paper is the more distinctive one. Instead of using one constant attention temperature for every token and every layer, it makes the scaling position-aware and layer-aware.

The paper’s interpretation of YaRN is the key here. Once you think about softmax as a mechanism that has to distribute probability mass over more and more tokens, it becomes natural to ask for a rule that keeps the distribution from becoming too flat. That is exactly what the entropy-aware scalar is doing.

Formula card: layer-aware dynamic attention scaling

Purpose: sharpen attention only when the model is moving beyond the easy pretrained regime, while leaving the earliest layers untouched.

Equation:

t={1,if layer index is 0 or 1max⁡(log⁡ci,1),otherwiset=\begin{cases} 1,&\text{if layer index is 0 or 1} \\ \max(\log_{c} i,1),&\text{otherwise} \end{cases}t={1,max(logc​i,1),​if layer index is 0 or 1otherwise​

Intuition: the first two layers keep their original attention behavior. Later layers get a position-dependent sharpening factor that grows slowly with token position, but never drops below one.

Symbols:

  • t: attention-logit scaling factor applied before softmax
  • c: pretrained context-window size, used as the logarithm base
  • i: token position index
  • layer index: the Transformer block number, starting from 0

Read-aloud meaning: do nothing in the first two layers. After that, increase sharpening with position, but never weaken attention below the pretrained setting.

Why it matters: this keeps the original context window safe while giving later layers a controlled way to stay focused when more tokens start competing for attention.

The lower bound is the guardrail. If a token is still inside the original pretrained window, \log_{c} i is at most 1, so the scale stays at 1. That means the model is not being asked to relearn the part of the context range it already handles well. Only when positions move farther out does the rule start to sharpen attention more aggressively.

The layer exemption is just as important. The entropy plot suggests that the earliest layers are the ones still changing with length, while the later layers already have a fairly stable concentration profile. So the method leaves layers 0 and 1 alone and applies the scaling only where the model already seems willing to preserve focus.

How the whole recipe flows

The full method is easiest to understand as a sequence of small repairs rather than one big rewrite.

Imagine a chat model that was trained on 4k tokens and is now asked to handle a much longer conversation. ABF first keeps the positional map from feeling too foreign. The long-conversation supervised fine-tuning then teaches the model how to operate in the extended range. The entropy-aware scaling rule finally keeps the later layers from becoming too soft when many more tokens are in play.

The important thing is that the paper is not trying to make every layer do the same job. It is trying to preserve the model’s short-context competence while adding a separate mechanism that becomes active only when long-range competition starts to blur the attention pattern.

What the experiments are really showing

The paper is also making a practical point: long-context extension does not have to require a huge adaptation budget. Figure 2 shows that the ABF family keeps benefiting from more data, while PI, NTK-By-Parts, and YaRN flatten out earlier.

What to notice in Figure 2 is the slope, not just the endpoints.

  • ABF-based methods keep improving as the training set grows.
  • PI, NTK-By-Parts, and YaRN reach a point where additional data helps much less.
  • The entropy-aware ABF variant is already competitive with a very small budget, which is the paper’s main efficiency claim.

That is why the paper emphasizes the 100-sample, 6-step result. The model is not being retrained from scratch. It is being nudged into a better positional regime, and the logit scaling gives that regime a useful inductive bias. In the paper’s own experiments, that was enough to extend LLaMA-2-7B-Chat to 16,384 tokens and still get strong long-context behavior.

The paper also reports that the proposed method is the only one in this comparison that stays robust when extrapolating directly to 32k. That matters more than a single benchmark score, because it says the method is not just fitting one fixed length. It is learning a control rule that still works when the window gets bigger than the fine-tuning target.

Robustness across window sizes is the harder test

The real question for a context-extension method is not “can it score well at one target length?” It is “does it still behave sensibly when the context window changes?” Figure 3 checks exactly that.

What to notice here is the difference between using a bigger window and actually using that window well.

  • PI, NTK-By-Parts, and YaRN improve when the context goes from 4k to 8k, but then degrade when pushed to 16k.
  • ABF and the entropy-aware method keep gaining within the fine-tuning length.
  • The entropy-aware method is the only one reported to maintain performance when directly extrapolating to 32k.

That is the most important robustness result in the paper. It suggests the method is not just fitting one fixed training length. The dynamic scaling rule is doing something useful as the window grows, because it keeps later-layer attention concentration from collapsing when the model is asked to look farther out than before.

A practical training shortcut

The paper also explores how the training data should be arranged for a given downstream task. The useful lesson is that long in-domain samples help, but they are not the only way to get started.

When the authors test different curricula on GovReport and line retrieval, short in-domain samples alone are not enough to give the best long-context adaptation. Starting from a model already fine-tuned on 1,000 ShareGPT conversations is a much better base, and it can transfer surprisingly well to both long and short evaluation settings.

That gives the method a nice practical shape: use long conversations to teach the model to survive longer contexts, then adapt from that base to a target task. In other words, the method is not just a new positional trick. It is also a recipe for how to stage the adaptation.

What this method is really buying you

The paper’s main contribution is not attention entropy by itself and not ABF by itself. It is the combination.

ABF changes the positional geometry so the model is less shocked by the longer window. The dynamic scaling then controls how sharply the model distributes attention once more tokens are in play. Put differently, ABF handles the map, and layer-aware scaling handles the spotlight.

That division of labor is why the method feels more stable than a constant temperature rule. Constant scaling can over-sharpen the front of the prompt or interfere with the pretrained regime. A dynamic rule that stays at 1 inside the original window and grows only later is much easier to justify.

What to keep in mind

This is a good method, but it is not a universal positional fix.

  • It is evaluated on RoPE-based chat models, not every transformer family.
  • It still depends on supervised fine-tuning data, so it is not a zero-data extrapolation trick.
  • The strongest claims are on selected long-context downstream tasks, so the gains are practical rather than universal.
  • The entropy-based rule is an interpretation of YaRN’s scalar, not a theorem that proves all long-context models should behave this way.

What to remember

If you remember only one sentence, make it this:

Entropy-aware ABF extends context by changing RoPE’s base frequency, then using a layer-aware, position-aware logit scale to keep attention concentration stable as sequences get longer.

The easiest way to remember the method is as two separate dials:

  • one dial keeps the positional geometry close to what the model already knows,
  • the other dial keeps later-layer attention from getting too diffuse when long prompts create more competition.

That is the conceptual payoff of the paper. Long-context extension works better when you treat position and concentration as related but distinct problems.
