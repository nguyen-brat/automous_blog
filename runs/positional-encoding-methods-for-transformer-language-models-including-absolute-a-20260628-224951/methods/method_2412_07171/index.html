Breaking the Stage Barrier: A Novel Single-Stage Approach to Long Context Extension for Large Language Models

HARPE is easiest to understand as a way of breaking the long-context curriculum into parallel pieces. Standard RoPE extension often says: first adapt to 32k, then 64k, then 128k. HARPE says: keep one training run, but let different attention heads live at different RoPE scales so the model can cover several effective lengths at once.

Think of it as giving the model a set of positional clocks with different tick speeds. Some heads keep a tighter clock for local syntax. Others rotate more slowly, so they stay useful when tokens are far apart. During continual pretraining, those heads collectively play the role that staged RoPE adaptation used to play one step at a time.

The paper’s pipeline figure makes that shift easy to see.

What to notice is not just the arrows, but the shape of the training process. The multi-stage path on the left grows context length by repeatedly stopping and retuning. HARPE on the right keeps the run single-stage, but spreads multiple positional scales across the attention heads. That is the stage barrier the paper is trying to break.

Why the shared-base trick hits a wall

RoPE starts from one shared base frequency b that determines the whole rotation schedule. Increasing b slows the rotations down, which helps longer sequences survive the decay that usually hurts far-away tokens. The trouble is that one global change is blunt. If every head gets the same scale, the model can lose some of its local sharpness without fully gaining long-range reach.

That is why many long-context models rely on a multi-stage schedule such as 8k -> 16k -> 32k -> ... -> 128k. The gradual path gives the model time to adjust to the new geometry. But it also creates a lot of manual tuning, and the paper’s point is that this tuning is doing work that could be distributed inside the model itself.

HARPE moves that diversity from the training schedule into the attention heads.

One head, one positional clock

Intuition: RoPE turns token positions into rotations. If every head uses the same base, every head rotates at the same pace. If each head gets its own base b_h, the model gets a mix of rotation speeds. Some heads stay focused on nearby structure, while others remain useful farther away.

Symbols:

  • \Theta_h: the set of RoPE frequencies used by head h
  • \theta_{h,i}: the frequency for the i-th rotary pair in head h
  • b_h: the base assigned to head h
  • d: the head dimension
  • i: the rotary-pair index

Read-aloud meaning: head h builds its positional rotations from its own base instead of sharing one global base with every other head.

Why it matters: this is the core mechanism that lets one continual-pretraining stage imitate several context-length stages at once.

The important move is not just “more bases.” It is head-level diversity. Each head still computes ordinary attention, and the model still preserves relative-position behavior through RoPE. HARPE only changes the rotary base that feeds each head’s position geometry.

That means the model keeps the same Transformer skeleton, the same head count, and the same target context length. What changes is the positional scale each head sees during training.

You can read that as a built-in division of labor. Heads with smaller bases keep a faster positional clock and tend to stay sharp for local patterns. Heads with larger bases rotate more slowly and stay readable farther out. Together, they approximate a small fleet of staged models living inside one network.

How the paper chooses the base set

The paper considers two ways to build the head-wise base set.

The first is uniform spacing between a minimum base and a maximum base. That is the simplest option: just spread the heads across the interval and let each head receive a different scale.

The second is more interesting. HARPE uses a peak-valley complementarity search, which is based on the oscillatory shape of RoPE. If one base is weak at a certain distance, another base may be strong there. So the search looks for a set of bases whose attention waveforms fill each other’s valleys and peaks instead of lining up in the same places.

The intuition is almost like choosing a chorus of instruments rather than one solo voice. If every head rings at the same frequency, the model gets redundant coverage. If the heads are intentionally offset, the combined response covers more of the positional landscape.

In the paper’s best setting, the bases are searched over the range from 1M to 5M with a 30k stride. The ablation is useful because it shows the base selection is not cosmetic. Uniform spacing works, but the complementarity search with 30k stride is the best performer on the upgraded Needle-in-a-Haystack test.

What training looks like

Once the base set is fixed, HARPE keeps the training recipe simple: continue pretraining directly at the final 128k target length. There is no 32k warm-up, no 64k checkpoint handoff, and no staged schedule to babysit.

That simplicity is the main engineering win. The model still preserves relative-position behavior, short-context competence, head count, and target context support. The difference is that the positional burden is shared across heads instead of across training phases.

So the data flow is really:

pretrained RoPE LLM + per-head base set -> single-stage continual pretraining at 128k -> long-context model

The paper’s claim is that this internal diversity is enough to make the model feel like it has seen multiple context-length stages, even though the optimizer only runs one stage.

Why the results are convincing

The result tables matter because they test three different failure modes: long-text perplexity, retrieval under stress, and short-context retention.

On sliding-window perplexity, HARPE reaches 3.02 on Proof-pile and 3.54 on GovReport. Those numbers are essentially tied with or slightly better than multi-stage ABF, which is the main staged baseline.

On the upgraded Needle-in-a-Haystack benchmark, the improvement is clearer. HARPE averages 86.82, compared with 81.36 for multi-stage ABF. That benchmark is more than a single-needle toy. It includes multi-key, multi-value, and multi-query cases, so it checks whether the model can actually retrieve the right information when the long context contains distractions and more than one target type.

Short-context quality also stays intact. On the five short-context benchmarks, HARPE averages 62.75, slightly above multi-stage ABF at 62.69. That matters because long-context tuning often breaks ordinary behavior in the process of helping the model at extreme lengths.

The RULER benchmark tells the same story at a broader scale. HARPE averages 70.40, ahead of llama-2-7b-80k at 67.85 and LWM at 67.67. So the method is not only good at one retrieval task. It keeps working on a more diverse long-context suite.

The ablation on base schemes explains why the search matters. Uniform spacing is decent, but the peak-valley complementarity search with 30k stride is the best option. That is the part worth remembering: the method works because the heads are different in a structured way, not because they are merely assigned arbitrary numbers.

What the paper is really teaching

HARPE is not a new attention architecture. It is a training strategy for positional diversity.

The paper’s useful lesson is that long-context adaptation does not have to be staged only in time. It can also be staged across heads. One head can behave like a short-range specialist, another like a medium-range specialist, and another like a long-range specialist, all within the same run. The model then learns to route different kinds of distance information through the heads that are best suited to it.

That is why the method feels simple once you see it. It does not rewrite the Transformer. It keeps RoPE, keeps continual pretraining, and keeps the final target length. The only thing it changes is the positional scale distribution across heads.

What to keep in mind

There are three caveats.

First, the evidence is centered on continual pretraining. The paper does not show the same story for supervised fine-tuning or instruction tuning, so transfer beyond pretraining remains open.

Second, the method is demonstrated mainly on one RoPE-based 7B backbone. That makes the result convincing for the intended setting, but it leaves scaling questions open.

Third, the base choice matters. HARPE is not “any random mix of head bases works.” The paper’s own ablation shows that poor base settings can lose robustness at the longest lengths.

So the durable takeaway is narrower than the headline but more useful: if you want a single-stage long-context recipe for a RoPE LLM, make the heads disagree on purpose. Let them cover different positional scales, and let the combined model behave like a staged curriculum without actually running one.
