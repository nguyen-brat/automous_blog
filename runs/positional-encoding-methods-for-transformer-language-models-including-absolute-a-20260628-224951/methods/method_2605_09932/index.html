FocuSFT: Bilevel Optimization for Dilution-Aware Long-Context Fine-Tuning

FocuSFT treats long-context failure as a training problem. The model can already ingest long sequences, but standard supervised fine-tuning often spends its attention on the wrong tokens: positionally privileged tokens and attention sinks soak up the budget, so the gradient sees a diluted version of the context. FocuSFT adds a tiny temporary memory during training, lets that memory sharpen the current example, and then updates the real model from the sharpened view. At inference time, nothing extra remains.

The overview figure makes that split easy to see.

What to notice in the figure:

  • the dashed inner loop is temporary and sample-specific;
  • the outer loop is the actual supervised fine-tuning update;
  • the fast weights are thrown away after each step, so only the base model survives.

That is the whole method in one picture. The rest of the paper explains how the temporary memory changes what the model learns from a long training sequence.

Why plain SFT gets diluted

A transformer does not look at every token equally. Each query position gets a fixed attention budget, and softmax forces that budget to be divided across the tokens that are visible to it. In a causal model, the earliest tokens are especially visible, so they become natural sinks for excess attention. Once those sinks start to absorb the budget, semantically important content tokens get less gradient signal than they should.

Intuition: each query has to spend a fixed amount of probability mass. If a few privileged tokens attract most of it, the rest of the context gets starved.

Symbols:

  • \alpha_{i,j}: attention weight from query position i to token j
  • z_{i,j}: raw attention score between query i and key j
  • o_i: output representation at position i
  • v_j: value vector for token j
  • T: total number of tokens in the sequence

Read-aloud meaning: turn raw pairwise scores into a probability distribution, then use that distribution to mix the value vectors.

Why it matters: this is what dilution means in practice. The model is not short of tokens; it is short of attention budget for the tokens that actually matter.

Under standard SFT, that budget problem gets baked into the gradients. The model learns from representations that already overemphasize the easy-to-reach tokens, so long-context supervision can reinforce the wrong habit instead of fixing it.

Give each example a scratch memory

FocuSFT responds by splitting the training step into two nested loops. The inner loop creates a temporary LoRA memory for the current example. The outer loop then trains the base model using the adapted representation produced by that memory. The important point is that the inner loop is not a second model in the final system. It is a short-lived helper that disappears after the update.

Formula card: the bilevel objective

Purpose: choose base weights that stay good after a brief adaptation to the current long example.

Equation:

min⁡θ Louter ⁣(θ,ϕ(K)(θ)),ϕ(K)=InnerLoop⁡(ϕ(0),θ,x)\min_{\theta}\,\mathcal{L}_{\text{outer}}\!\bigl(\theta,\phi^{(K)}(\theta)\bigr),\qquad \phi^{(K)}=\operatorname{InnerLoop}\bigl(\phi^{(0)},\theta,\mathbf{x}\bigr)θmin​Louter​(θ,ϕ(K)(θ)),ϕ(K)=InnerLoop(ϕ(0),θ,x)

Intuition: first build a temporary helper for the current sequence, then train the real model using the helper’s sharpened view of that sequence.

Symbols:

  • \theta: base model weights that persist after training
  • \phi^{(0)}: fresh temporary LoRA fast weights reinitialized at the start of the step
  • \phi^{(K)}: adapted temporary weights after K inner updates
  • \mathcal{L}_{\text{outer}}: outer supervised fine-tuning loss on response tokens
  • \mathbf{x}: the training sequence, split into context and response tokens
  • K: number of inner-loop gradient steps

Read-aloud meaning: update the main model only after a temporary per-example memory has been adapted to the current context.

Why it matters: this is the core abstraction. The model no longer learns from a raw diluted sequence. It learns from a sequence that has already been reorganized by a tiny context-specific memory.

The inner loop uses lightweight LoRA adapters on feed-forward layers near the top of the network, so the temporary memory has enough capacity to matter without turning the method into a second full transformer. The paper also uses a first-order approximation, which means the outer update treats the adapted fast weights as fixed. That avoids the cost of full second-order bilevel optimization while keeping the main effect.

The inner and outer loops also optimize the same next-token objective on the response tokens. That shared loss keeps the temporary memory pointed at the same task the outer update cares about.

The step-by-step flow is simple:

  • split each sequence into context tokens and response tokens;
  • adapt the temporary LoRA weights on the current example;
  • compute the response loss using the adapted weights;
  • update only the base model;
  • discard the fast weights.

That last step matters. The temporary memory is a training aid, not an inference feature.

Let context tokens see each other

The second ingredient is a different attention mask during training. FocuSFT keeps responses causal, but it lets context tokens attend to one another bidirectionally. This is the direct answer to the sink problem. Under a purely causal mask, the first few tokens are uniquely visible to all later tokens, which makes them attractive places for attention to pile up. If the context can read itself as a block, that asymmetry weakens.

Purpose: let context tokens communicate freely while keeping response generation causal.

Equation:

Intuition: context tokens can look across the whole context, but response tokens still have to predict left to right.

Symbols:

  • \mathcal{C}: positions that belong to the context
  • \mathcal{R}: positions that belong to the response
  • M_{i,j}: attention-mask entry for query position i and key position j

Read-aloud meaning: context tokens get a bidirectional view of the prompt, and response tokens only see earlier tokens.

Why it matters: this is the simplest way to remove the asymmetry that encourages sinks, without changing the decoding rule used at inference time.

This same mask is used in both the inner and outer loops. That consistency is not cosmetic. If the inner loop sees one attention geometry and the outer loop learns under another, the temporary memory can become a source of distortion instead of sharpening.

Why the mask alone is not enough

The ablation story is the key cautionary note. Bidirectional attention by itself underperforms standard SFT. That sounds surprising until you remember the mismatch: the model is trained with richer visibility, but it still has to run causally at inference time. Without the inner loop, that extra visibility does not get organized into a useful parametric memory, so the training signal can drift away from what the final model actually needs.

FocuSFT fixes that by making the richer context view useful. The inner loop turns the current example into a temporary scratch memory. The outer loop then learns from the sharpened representation produced by that scratch memory. In other words, the bidirectional mask is not the whole answer. It is the environment in which the memory adaptation becomes effective.

What changes in practice

The measured behavior lines up with that story. On BABILong, FocuSFT improves standard SFT by +14.2, +10.2, +10.2, and +9.6 points at 4K, 8K, 16K, and 32K. On RULER CWE at 16K, it rises from 72.9% to 81.1%. On GPQA with agentic tool use, pass@1 moves from 15.6% to 19.4%.

The attention analysis is even more revealing. The sink mass drops by 529x, and total context engagement rises from 13.5% to 41.4%. So the model is not just getting better scores by accident. It is spending its attention budget differently. More of the budget reaches the actual prompt content, and less of it gets trapped in the initial sink positions.

The tradeoff is on the training side. The method adds about 1.71x training-time overhead because of the inner-loop updates, but it does not add inference overhead. Once training is done, the fast weights are gone and the base model decodes normally.

What to remember

FocuSFT is best thought of as a scratch-memory trick for long-context training. Instead of asking the base model to learn from a diluted long sequence in one pass, it gives that sequence a temporary LoRA memory, lets the memory sharpen the context, and then updates the real model from the sharpened view. Bidirectional context attention removes the sink-friendly asymmetry that causal masking creates, but the inner loop is what makes that richer view useful.

The method is promising, but the paper is also clear about its limits:

  • it was tested on one model family and a fixed 3K-sample corpus;
  • it was not evaluated for RLHF or DPO training stages;
  • it costs more to train, even though inference stays unchanged.

The durable lesson is broader than this one paper. If long-context fine-tuning spends its attention on the wrong tokens, simply widening the window is not enough. You also have to change how the model learns to spend attention while it is training.
