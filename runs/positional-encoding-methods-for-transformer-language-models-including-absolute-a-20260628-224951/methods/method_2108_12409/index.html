Train Short, Test Long: Attention with Linear Biases Enables Input Length Extrapolation

ALiBi, short for Attention with Linear Biases, is a deceptively small change to Transformer attention. Instead of adding positional embeddings to the token stream, it subtracts a distance-based penalty from each query-key score before softmax. The model still compares content in the usual way, but farther-away tokens pay a recency tax. That is the whole point: train on shorter chunks, keep the efficiency benefits of short training, and still have the model behave sensibly when inference asks for much longer sequences.

You can keep the whole method in one line:

Position is no longer a separate vector that rides alongside the token. It becomes part of the routing decision inside attention.

Look at the figure “ALiBi bias added to attention scores” as a mask-like gradient. Each head gets its own slope, so the oldest keys in a causal window are pushed down the most and the newest key gets zero penalty. The picture matters because it makes ALiBi feel less like a new architecture and more like a smarter version of the causal mask.

ALiBi attention logit update

Purpose. Show where the positional bias enters attention.

Equation.

Intuition. The content score still comes from the query-key dot product, but a fixed slope subtracts more from distant keys than from nearby ones.

Symbols.

  • qi\mathbf{q}_iqi​: the query vector at position iii.
  • K\mathbf{K}K: the matrix of key vectors visible to token iii under causal masking.
  • mmm: the head-specific slope, fixed before training, that controls how strongly distance is penalized.
  • iii: the current query position.
  • softmax⁡\operatorname{softmax}softmax: the normalization step that turns the biased logits into attention weights.

Read-aloud meaning. Take the usual attention scores for token iii, subtract a distance penalty that grows as keys get farther back, and normalize the result into attention weights.

Why it matters. This is the entire ALiBi trick. Position enters after the dot product, so the model keeps normal content similarity while getting a strong recency bias for free.

The important detail in the vector [-(i-1), ..., -1, 0] is that it is aligned with the causal window. The farthest visible key gets the largest negative correction, and the current token gets none. So the score curve is not a learned table of positions. It is a straight line that tilts each head toward recent context.

Why the slopes are fixed but not identical

ALiBi does not use one shared distance rule for every head. It gives different heads different slopes, and the slopes are arranged as a geometric schedule. In a small-head model, that means some heads get a steep penalty and stay very local, while others get a gentler penalty and can keep more distant keys in play. The paper also shows that this schedule transfers well across datasets and model sizes, so it is not something you need to retune every time you move to a new corpus.

That fixed schedule is part of the method’s appeal. The authors tried learning the slopes, but that did not improve extrapolation and it slowed training a bit. In other words, the slope values work better as an inductive prior than as another thing for the optimizer to rediscover. The model gets a spectrum of recency lenses, not a new parameter table to memorize.

The simplest way to think about those heads is this: one head can be the “look nearby” head, another can be the “keep a wider window open” head. The geometric spacing makes that spread smooth instead of abrupt, so the model has a graded set of locality preferences rather than a single binary choice.

Why the implementation stays cheap

The code change is tiny. ALiBi is implemented by adding the bias into the causal mask before softmax, so the core attention operation does not need a new kernel or a new pass over the sequence. There is no extra positional embedding table, and there is no position information injected into the value vectors at all. Position lives in the score matrix, not in the payload that gets averaged.

That is why the method is attractive in practice. It keeps the runtime essentially unchanged, but it does make the training mask a little larger because each head needs its own bias slice. So the tradeoff is very favorable: a small memory increase for a much better extrapolation story.

This is also the cleanest way to see the method’s relationship to the older position tricks. Sinusoidal embeddings add position at the input and hope the model can generalize. Rotary methods inject position into queries and keys at every layer. T5 bias also works at the score level, but it learns a more flexible distance table and pays for that flexibility with extra compute. ALiBi keeps the score-level idea and strips the mechanism down to a fixed linear ramp.

What the experiments are really saying

The paper’s strongest claim is not that ALiBi wins by a tiny margin on one benchmark. It is that the same simple distance penalty can be trained on short subsequences and still work on much longer ones without a heavy runtime tax. On WikiText-103, the authors train a 512-token ALiBi model and evaluate it on 3,072-token sequences; it reaches 18.40 perplexity, which beats the sinusoidal baseline trained directly on 3,072-token sequences at 18.67. That is the paper’s core “train short, test long” result in one number.

The larger-scale result makes the point even clearer. In the 1.3B-parameter CC100+RoBERTa experiment, the ALiBi model trained on 1,024-token sequences matches or beats the sinusoidal model trained on 2,048-token sequences while using less memory and reaching the target perplexity about 11% faster on average. The same slope schedule also transfers to Toronto BookCorpus without retuning, which is important because it says the trick is not tailored to one dataset’s quirks.

Seen together, those results tell a consistent story. ALiBi is not just an extrapolation hack. It is a cheap, reusable positional prior that often lets you buy better long-context behavior by moving the position signal into the attention logits instead of into the token embeddings.

The part that is easy to miss

Longer windows do not automatically mean the model is using more history. The paper checks this by switching from nonoverlapping evaluation to sliding-window evaluation, where overlapping chunks give every prediction more left context. In that setting, ALiBi’s curve stays much flatter as the validation length grows.

That observation points to the early token curse. When you split a long sequence into chunks, the first tokens in each chunk have very little left context, so they are disproportionately hard to predict. If you make the chunk longer, fewer predictions live in that under-informed zone. Some of the apparent gain from longer nonoverlapping windows therefore comes from reducing this boundary effect, not from the model suddenly exploiting arbitrarily distant memory.

The authors are careful about this. ALiBi’s best gains usually appear around roughly 2x the training length, and beyond that the returns flatten. So the method is excellent at making short-trained transformers more robust on longer inputs, but it is not a full answer to truly unlimited long-context reasoning.

What to remember

ALiBi is easiest to remember as a recency lens built into the attention logits. Each head gets its own fixed linear tax on distance, so nearby tokens are naturally preferred but long sequences do not break the model. The trick works because it moves position into the score matrix instead of into embeddings, keeps the implementation simple, and gives the model a prior that survives short-to-long extrapolation.

Its limit is just as important: it improves extrapolation and stability, but it does not solve the harder problem of fully exploiting histories far beyond the training window.
