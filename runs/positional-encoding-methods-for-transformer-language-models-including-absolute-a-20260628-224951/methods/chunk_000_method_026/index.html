DIET: Decoupled Positional Attention for Transformers

DIET is built around a simple but important idea: do not force position to travel through the same input embedding stream as token content. Instead, move position into the attention score itself, and let each head carry its own positional and segment bias. That gives the model a cleaner division of labor. Content decides whether two tokens match. Position decides how that match should be shaped. Segment information decides whether tokens from different parts of the input should interact differently.

You can think of it like giving every attention head its own map of the sequence. The token embeddings still say what each token is, but the head gets a separate structure-aware signal for where those tokens sit and which segment they came from.

That is the core move in DIET. The paper then asks a sharper question: if position can be injected in more than one place, where should it go?

Why the input layer is the wrong place for position

The standard Transformer recipe adds position embeddings to token embeddings before attention ever sees them. That sounds harmless, because the model still gets order information. But the paper argues that this wiring creates a bottleneck.

When position is added at the input, token identity and position are mixed together too early. The query and key projections then compress that mixed signal through the same low-dimensional head space. In other words, the attention score matrix can only express so much structure because both content and position have already been squeezed through the same bottleneck.

The paper makes two related points here:

  • The rank of the input-added absolute attention matrix is limited by the head projection width.
  • The gradient signal for the token embeddings and the positional embeddings is identical at the point where they are added, so the two sources of information are forced to learn through the same downstream pathway.

So the paper is not saying absolute positions are useless in principle. It is saying the usual input-addition route is the wrong place to wire them.

What to notice in the figure:

  • the baseline attention matrix is more constrained and visibly lower-rank;
  • the per-head positional version has more degrees of freedom;
  • the difference is not about adding more token semantics, but about giving the position signal its own channel inside attention.

That visual is the paper’s main argument in one image. The gain comes less from “absolute vs relative” as a label and more from moving position into attention itself.

Diet-Abs: absolute position after the query and key projections

Here is the main absolute-position version of DIET.

Formula card: Diet-Abs attention

Purpose. Show how content, absolute position, and segment structure are combined in one attention score.

Equation.

Ai,jABS=(Xi:WQ)(Xj:WK)⊤/d+(PQPK⊤)i,j+ES(S(i),S(j))A_{i,j}^{\mathrm{ABS}}=(X_{i:}W_Q)(X_{j:}W_K)^\top/\sqrt{d}+(P_QP_K^\top)_{i,j}+E_S(S(i),S(j))Ai,jABS​=(Xi:​WQ​)(Xj:​WK​)⊤/d​+(PQ​PK⊤​)i,j​+ES​(S(i),S(j))

Intuition. First compute ordinary content attention. Then add a learned position-only score for the pair of token indices. Then add a learned segment score so sentence-pair tasks can treat same-segment and cross-segment interactions differently.

Symbols.

  • Ai,jABSA_{i,j}^{\mathrm{ABS}}Ai,jABS​: attention logit from token iii to token jjj.
  • XXX: matrix of token hidden states.
  • Xi:X_{i:}Xi:​, Xj:X_{j:}Xj:​: the iii-th and jjj-th rows of XXX.
  • WQW_QWQ​, WKW_KWK​: query and key projection matrices.
  • ddd: the attention scaling dimension used in the denominator.
  • PQP_QPQ​, PKP_KPK​: low-rank position matrices for the query and key sides.
  • dpd_pdp​: the width or rank of the positional matrices.
  • ES(S(i),S(j))E_S(S(i),S(j))ES​(S(i),S(j)): learned segment interaction score for the two segments containing tokens iii and jjj.
  • S(i)S(i)S(i): the segment id of token iii.

Read-aloud meaning. Compute normal attention, then add separate absolute position and segment scores before softmax.

Why it matters. Position is no longer forced to share the same bottleneck as token content. It becomes its own channel in the score matrix, so the model can spend extra capacity on structure instead of folding everything into the token embedding.

The useful detail here is that PQPK⊤P_QP_K^\topPQ​PK⊤​ depends only on position, not on the actual tokens. That means the positional score can be precomputed for a batch and cached at inference time. It also means the positional width dpd_pdp​ is a real design knob. If you want more positional capacity, increase dpd_pdp​. The paper calls the larger-width version Diet-Abs-Rank-kkk.

In practice, the paper finds that layer-wise sharing works well for Diet-Abs. Sharing reduces parameters, and for this absolute variant it also acts like a useful regularizer. The best XTREME result in the paper uses Diet-Abs with dp=128d_p=128dp​=128 and layer-wise sharing.

Diet-Rel: the same decoupling, but with a distance bias

Diet-Abs gives each pair of positions an absolute learned score. Diet-Rel keeps the same per-head attention design, but swaps the absolute position term for a relative one. Instead of asking “what is the score for position 57?”, it asks “what is the score for an offset of -3 or +5?”

Formula card: Diet-Rel attention

Purpose. Show the relative-position version of the same decoupled attention idea.

Equation.

Ai,jREL=(Xi:WQ)(Xj:WK)⊤/d+Ri−j+ES(S(i),S(j))A_{i,j}^{\mathrm{REL}}=(X_{i:}W_Q)(X_{j:}W_K)^\top/\sqrt{d}+R_{i-j}+E_S(S(i),S(j))Ai,jREL​=(Xi:​WQ​)(Xj:​WK​)⊤/d​+Ri−j​+ES​(S(i),S(j))

Intuition. Use normal content attention, then add a learned distance bias and a segment bias.

Symbols.

  • Ai,jRELA_{i,j}^{\mathrm{REL}}Ai,jREL​: attention logit from token iii to token jjj.
  • Ri−jR_{i-j}Ri−j​: learned bias for the relative offset between positions iii and jjj.
  • All other symbols are the same as in Diet-Abs.

Read-aloud meaning. Score tokens by content, then tilt the score by how far apart they are and by which segments they belong to.

Why it matters. This version keeps the same “position belongs inside attention” philosophy, but gives the model an explicit shift bias. That is a better fit when relative distance matters more than exact absolute index.

The paper simplifies the T5-style relative scheme by removing log-binning and layer-wise parameter sharing. The result is cleaner and cheaper. Empirically, Diet-Rel is competitive, but the paper also shows that sharing hurts Diet-Rel more than it helps. That is a useful clue: relative position already has a strong shared structure, so over-sharing can remove flexibility instead of adding it.

Segment encoding is part of the same story

DIET does not treat segment ids as a separate side feature. It moves them into the same attention-score space as position. That is what the ES(S(i),S(j))E_S(S(i),S(j))ES​(S(i),S(j)) term does.

This matters most on tasks with multiple segments, such as sentence-pair classification or question answering. In those settings, the model should not only know where a token is. It should also know whether two tokens come from the same sentence, the question side, or the passage side. A segment embedding added to the input can work, but the paper’s point is that structural information is often better handled where attention decisions are actually made.

The gain is modest but consistent. Across the paper’s GLUE and XTREME ablations, moving segment information into attention improves both absolute and relative variants.

Why the method stays cheap

One reason DIET is attractive is that it does not buy expressivity by adding a heavy module.

Diet-Abs adds one extra position-only matrix term, but that term is independent of token content. So it can be computed once per batch and cached at inference. Diet-Rel is even lighter: it is basically a learned distance bias table plus the same segment interaction term.

That is why the paper reports near-zero training and inference overhead for Diet-Abs and Diet-Rel relative to the baseline BERT setup. Other positional methods can add noticeable cost, especially at smaller model sizes and during inference. DIET’s message is that a cleaner wiring of position can be cheaper, not more expensive.

How it extends to long-range transformers

The same decoupling idea also works when the attention mechanism itself is compressed. The paper applies Diet-Abs to Linformer, where the key stream is projected to a lower-rank form before attention is computed.

The point is not the exact projection matrix. The point is that the positional score still lives inside attention, even when the content path is low-rank. That makes the position design orthogonal to the long-context compression trick. In the paper’s GLUE results, this upgrade improves the Linformer-style setting from 82.0 to 82.7 average.

That is a useful general lesson: if a long-context method already changes the attention geometry, you can often still improve it by decoupling position from token content instead of adding position back at the input.

What the experiments are really saying

The experiments repeat the same pattern across GLUE, XTREME, and translation:

  • per-head positional injection beats input-added absolute embeddings;
  • Diet-Abs can match or exceed strong relative methods when the positional width is chosen well;
  • Diet-Rel stays competitive when you want a more explicit relative bias;
  • moving segment information into attention helps both families.

The anchor result for the absolute variant is easy to remember: Diet-Abs with dp=128d_p=128dp​=128 and layer-wise sharing reaches 85.2 on GLUE and 58.9 on XTREME. That is the cleanest sign that the design is not just theoretically tidy; it also holds up on transfer and cross-lingual benchmarks.

The rank sweep tells the same story. Increasing dpd_pdp​ from 64 to 128 helps, which fits the paper’s rank argument. Pushing all the way to 512 does not keep improving things, so more positional rank is not automatically better. The model needs enough room for position, but not an unlimited amount.

The translation results line up too. Once position is moved into attention, all of the per-head variants do better than input-added absolute embeddings.

Where DIET is strong, and where it is not

The strongest part of the paper is the mechanism, not the benchmark leaderboard. DIET makes a convincing case that the location of positional information matters as much as whether that information is absolute or relative.

The limitations are worth keeping in mind:

  • Diet-Abs may need the right rank and sharing choice to stay parameter-efficient.
  • The paper is much more centered on encoder-style benchmarks than on large autoregressive language modeling.
  • Relative variants can still be the better choice if you want an explicit shift bias rather than an absolute position map.

So the paper does not argue that absolute position should win everywhere. It argues that once position is moved into attention, absolute position becomes much more viable than the standard input-addition recipe makes it look.

What to remember

DIET’s main lesson is that position should be a first-class part of the attention score, not just a decoration on the input embedding. Once you separate content, position, and segment structure, each head gets more freedom to represent the patterns it actually needs. Absolute position can then work well again, relative position remains strong when shift bias matters, and both can do so with near-baseline cost.

If you want one mental shortcut, keep this one: DIET gives each head its own positional map instead of painting position onto the tokens once at the input. That small change is what unlocks the better rank, the better transfer results, and the cleaner story about how positional encoding should work.
