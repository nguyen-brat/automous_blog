Long-Context Generalization with Sparse Attention

Softmax is a comfortable default when the context is short. It is a shaky default when the context gets wide. Every extra token still gets some probability mass, so the useful evidence has to compete with a growing crowd of distractors. Over long prompts, that crowd becomes the problem.

This paper takes a cleaner route. It replaces softmax with alpha-entmax, which can delete irrelevant tokens exactly, then adds a learned length-aware scale per head so sparsity stays useful instead of becoming too sharp or too diffuse.

That is the whole method in miniature. First make attention sparse. Then teach each head how sparse it should be as the context grows.

When attention stops being a blur

The easiest way to understand the paper is to imagine a query that really needs just one or two tokens from a very long prompt. Softmax does not know how to truly ignore the rest. It keeps paying a little tax to every token, so the useful match gets thinner as the context grows.

Alpha-entmax changes that. It keeps only the tokens that clear a threshold and gives the rest exactly zero weight. The result is not just a different normalization rule. It is a different attention graph, because irrelevant edges can disappear entirely.

What to notice in this image is the transition from smooth to selective. As the temperature drops, the distribution stops spreading itself across all tokens and starts concentrating on a small support. Eventually some tokens drop out completely. That exact-zero behavior is the big idea.

Formula card: alpha-entmax as sparse attention

Purpose: turn raw attention scores into a probability distribution that can remove low-value tokens entirely.

Equation:

α-entmax(z)i=[(α−1)zi−τ(z)]+1α−1\alpha\text{-entmax}(\mathbf{z})_i = \left[(\alpha-1)z_i - \tau(\mathbf{z})\right]_+^{\frac{1}{\alpha-1}}α-entmax(z)i​=[(α−1)zi​−τ(z)]+α−11​​

Intuition: compare each logit to a learned threshold. Scores above the threshold survive with positive mass. Scores below it become exactly zero.

Symbols:

  • \mathbf{z}: the vector of attention logits for one query.
  • z_i: the logit for token i.
  • \alpha: the sparsity control, with larger values making the output sparser.
  • \tau(\mathbf{z}): the threshold chosen so the output sums to 1.
  • [\cdot]_+: max(0, \cdot), so negative values are clipped to zero.

Read-aloud meaning: each token keeps weight only if its score clears a threshold set by the whole logit vector.

Why it matters: this exact-zero behavior is what lets the method ignore junk tokens instead of just downweighting them.

The paper uses \alpha = 1.5 as the practical default. That sits in the middle ground between dense softmax-like behavior and the very hard selection of sparsemax.

The theoretical story follows naturally from that thresholding. With softmax, adding more tokens always makes the distribution more diluted. With alpha-entmax, a new token can simply land below threshold and get ignored without forcing the existing useful weights to shrink. That is why the paper says alpha-entmax resists attention dispersion, helps preserve distinct token representations, and reduces the number of gradient routes that have to carry long-range signal.

Why sparse attention still needs a length knob

There is a catch, though. Sparse attention is not automatically good sparse attention.

If a head is tuned for a short prompt, the same head can become too aggressive when the prompt is much longer. The raw score range usually widens with context length, so a fixed sharpness setting can start pruning too hard. The head that used to behave like a focused reader can turn into an overconfident filter.

That is the reason for ASEntmax. Instead of using one fixed scale, the paper rescales the logits before alpha-entmax with a learned function of context length.

Formula card: ASEntmax scaling rule

Purpose: retune how sharp a sparse attention head should be as the context length changes.

Equation:

ASEntmax(z)=α-entmax((δ+β(log⁡n)γ)z)\mathrm{ASEntmax}(\mathbf{z}) = \alpha\text{-entmax}\left((\delta+\beta(\log n)^{\gamma})\mathbf{z}\right)ASEntmax(z)=α-entmax((δ+β(logn)γ)z)

Intuition: the model first stretches or shrinks the logits using a length-aware scale, then applies sparse entmax. That lets the same head stay selective on short contexts without becoming brittle on long ones.

Symbols:

  • \mathbf{z}: the attention logits before normalization.
  • n: the current context length.
  • \delta: a baseline scale.
  • \beta: the learned scale strength.
  • \gamma: the learned length-growth exponent.
  • \alpha: the entmax sparsity setting.

Read-aloud meaning: multiply the logits by a learned function of context length, then run sparse entmax on the rescaled scores.

Why it matters: this is the part that keeps sparsity useful instead of letting it become too sharp or too flat when the sequence gets longer.

The important detail is that the paper does not treat \beta and \gamma as global constants. It learns them per head from the current query stream, with softplus keeping \beta positive and tanh keeping \gamma bounded. A negative \gamma widens support as length grows. A positive \gamma sharpens it. That means different heads can learn different sparsity schedules instead of being forced into one universal temperature.

What to notice here is that the heads do not agree with one another. Some want gentle changes with length, some want stronger retuning, and a simple \delta + \beta \log n curve is not flexible enough for all of them. The \gamma power gives the model a second degree of freedom, which is why the fit improves across heads.

This is also why ASEntmax is best thought of as a wrapper around sparse attention, not a replacement for it. When \beta = 0, the method falls back to standard alpha-entmax. So the scaling rule is not throwing away the original sparse mechanism. It is giving it a better focus control.

Because the scale is applied to the logits rather than baked into a custom kernel, the paper can keep the entmax computation efficient and reuse optimized implementations.

What exact zeros buy you

Once a head can drop irrelevant tokens completely, three long-context problems get a lot easier to reason about.

  • The attention distribution stops leaking mass to every token, so relevant matches do not get diluted just because the prompt is long.
  • Token states stay more distinct, because the model is no longer averaging over a giant dense cloud of weak connections.
  • Gradient flow is less squashed, because the network has fewer active paths to route information through.

That last point is easy to miss, but it matters. In a dense attention graph, almost every token can influence almost every later token, which creates a huge number of weak paths. Sparse support turns that into a much smaller active subgraph. Fewer paths, but stronger ones.

That is the deep reason alpha-entmax helps the model preserve representations and resist over-squashing. It does not make long context magically easy. It just stops the network from wasting effort on edges that should not have been there in the first place.

Positional encoding becomes part of the mechanism

Once attention becomes sparse, positional encoding is no longer a side choice. It helps decide which edges survive the threshold.

NoPE keeps the logits content-driven. ALiBi adds a distance penalty, which makes sparse attention behave like a hard local window once the thresholding kicks in. RoPE turns position into a rotation, which can make the surviving support more periodic and more fragmented. In other words, the positional encoding does not just bias a dense distribution anymore. It helps choose the active graph.

What to notice here is the division of labor. NoPE heads can stay content-driven and reach farther when the query asks for semantic retrieval. ALiBi heads are naturally local, with a clearer cutoff. RoPE heads produce more irregular patterns because distance and frequency interact inside the score itself.

That is why the paper introduces NAPE, a simple hybrid that splits the heads between NoPE and ALiBi. Half the heads stay free to chase content, and half the heads keep a recency bias. This is a practical compromise: local heads handle nearby structure, while content-driven heads still have a path to distant evidence.

ASEntmax and NAPE complement each other. NAPE gives the model a useful positional shape. ASEntmax decides how sharply each head should trace that shape as the context length changes.

What the experiments say

The experiments are easiest to understand if you read them as stress tests for the mechanism rather than as a scoreboard.

On retrieval-style synthetic tasks, the combination of sparse attention and length-aware scaling is the strongest recipe. A good anchor is MQMTAR: ASEntmax keeps 95.3% exact-match accuracy at 65K tokens after training on just 64. On RULER’s S-NIAH-1 retrieval benchmark, it stays at 97.4% accuracy at 16K after training on 2K. Those are the kinds of numbers you get when the model can keep the right tokens alive without letting the rest of the context wash them out.

The same pattern shows up in language modeling. The paper reports the best long-context perplexity trend on ArXiv and PubMed among the compared methods, while short-context performance stays broadly competitive. That matters, because a long-context trick is only useful if it does not ruin normal prompts.

The limits are just as informative.

  • Tasks that need broad global ordering, such as Sort, remain hard.
  • The benefits depend strongly on the positional encoding choice, with NAPE treated as the practical default.
  • Fixed sparsity can be too aggressive when many tokens matter at once, which is exactly why the learned \gamma needs to be able to widen the support.

The Copy task makes that last point concrete. There, the model learns negative \gamma values, which spread attention out instead of collapsing it into a tiny support. So ASEntmax is not “more sparsity at all costs.” It is “the right amount of sparsity for the task and the context length.”

What to remember

The paper is really making one clean argument:

softmax spreads attention too thin, alpha-entmax can zero out junk, and ASEntmax teaches each head how to adjust that sparsity as the context grows.

If you want a compact memory hook, use this one:

  • softmax = dense blur,
  • alpha-entmax = exact-zero filtering,
  • ASEntmax = length-aware focus control,
  • NAPE = local heads plus content-driven heads.

That combination is why the method works. It removes irrelevant tokens, keeps the sharpness adaptable, and gives positional encoding a role that matches sparse attention instead of fighting it.
