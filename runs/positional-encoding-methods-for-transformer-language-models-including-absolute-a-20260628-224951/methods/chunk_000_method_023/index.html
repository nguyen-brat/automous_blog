Focused Transformer: Contrastive Training for Context Scaling

FoT is a way to teach a transformer’s memory to search rather than drift. The model still uses ordinary attention, but the extra context is no longer treated as one undifferentiated pile of old tokens. During fine-tuning, it is shown which memory entries come from the same document and which ones are unrelated. That contrast is what makes the key-value space easier to search when the context gets crowded.

The paper’s central claim is not “give the model more tokens.” It is “teach the model to ignore the wrong tokens once the memory grows.” That distinction matters because long context fails most often through distraction, not through raw capacity.

Figure 2 makes the training picture concrete.

What to notice is that the method is not adding a separate retriever on the side. It changes how a few attention layers behave during training so they learn to treat the previous context as useful memory and unrelated documents as distractors. The rest of the transformer stays standard.

A memory layer that can look beyond the window

FoT only turns selected layers into memory layers. At inference time, each query in one of those layers attends to two places at once: the ordinary local context before the query, and a shortlist of memory pairs that look most similar to the query. The shortlist comes from top-k key search in memory; the paper uses exact kNN search, and in the LongLLaMA retrofit it later swaps that for dense attention because the behavior is similar and the implementation is simpler.

Formula Card: Memory attention in a focused layer

Purpose: let one attention layer read both the local past and the most relevant stored memory.

Equation:

o=∑(k,v)∈Mtop∪C<qs(k) v\mathbf{o}=\sum_{(\mathbf{k},\mathbf{v})\in M_{\mathrm{top}}\cup C_{<q}} s(\mathbf{k})\,\mathbf{v}o=(k,v)∈Mtop​∪C<q​∑​s(k)v s(k)=softmax([⟨q,k⟩τ]k∈Mtop∪C<q)s(\mathbf{k})=\mathrm{softmax}\left(\left[\frac{\langle \mathbf{q},\mathbf{k}\rangle}{\tau}\right]_{\mathbf{k}\in M_{\mathrm{top}}\cup C_{<q}}\right)s(k)=softmax([τ⟨q,k⟩​]k∈Mtop​∪C<q​​)

Intuition: first retrieve the memory entries whose keys align best with the query, then blend those memory values with the tokens already visible in the local window.

Symbols:

  • \mathbf{o}: the attention output for the current query
  • \mathbf{q}: the current query vector
  • \mathbf{k}: a candidate key vector
  • \mathbf{v}: the value vector paired with that key
  • M_{\mathrm{top}}: the top-k memory key-value pairs retrieved from stored memory
  • C_{<q}: the tokens in the local context that appear before the query
  • \tau: a learnable temperature that controls how sharp the softmax is
  • \langle \mathbf{q},\mathbf{k}\rangle: the query-key similarity score

Read-aloud meaning: score the local past and the best memory matches together, then average their values with more weight on the pairs that look most relevant.

Why it matters: this is the actual memory-attention mechanism. The model does not need a new decoder; it needs a better way to choose which old key-value pairs deserve attention.

A useful detail is that FoT does not make memory and local context into two separate worlds. The layer merges them in one attention pool. The paper tried a gated combination too, but the simpler shared pool worked just as well and avoided extra parameters. That is one reason the method is easy to bolt onto existing transformers.

The memory itself grows incrementally. As the layer processes a document, it stores the key-value pairs it has already seen, so later queries can look back into them. In other words, reading creates the archive that future tokens will search.

Another subtle choice is how position is handled. The additional memory is not supposed to behave like a second copy of the local timeline. In the FoT setup, the extra memory layers can drop positional encodings so the retrieval path stays focused on content, not on exact old positions. In the LongLLaMA retrofit, the authors keep the normal local RoPE on the short window for compatibility, but they treat memory keys as if they were at the same starting position. That keeps the model close to an ordinary LLaMA checkpoint while still letting the memory act like global context.

Training the model to ignore the wrong history

The interesting part of FoT is that training is already doing retrieval work. Instead of giving each document a separate memory store and asking the model to learn from retrieved items later, FoT trains the memory layer directly on a batch of documents.

Each batch element is one document. The model sees the current local chunk for that document, plus the previous chunk from the same document. Those are the positives. It also sees d-1 chunks from other documents in the batch. Those are the negatives. The attention computation stays differentiable all the way through those past key-value pairs, so the key-value space itself gets shaped by the loss.

Formula Card: Crossbatch training

Purpose: train the same attention layer on same-document positives and unrelated-document negatives.

Equation:

o=∑(k,v)∈pδ∪Ccurrδ,<qs(k) v\mathbf{o}=\sum_{(\mathbf{k},\mathbf{v})\in p^\delta\cup C_{\mathrm{curr}}^{\delta,<q}} s(\mathbf{k})\,\mathbf{v}o=(k,v)∈pδ∪Ccurrδ,<q​∑​s(k)v s(k)=softmax([⟨q,k⟩τ]k∈pδ∪Ccurrδ,<q)s(\mathbf{k})=\mathrm{softmax}\left(\left[\frac{\langle \mathbf{q},\mathbf{k}\rangle}{\tau}\right]_{\mathbf{k}\in p^\delta\cup C_{\mathrm{curr}}^{\delta,<q}}\right)s(k)=softmax([τ⟨q,k⟩​]k∈pδ∪Ccurrδ,<q​​)

Intuition: give the layer the previous context from the same document plus a few unrelated contexts, and let normal attention learning figure out which entries deserve focus.

Symbols:

  • p^\delta: the extra key-value pairs supplied for document \delta, made from the previous same-document context and d-1 other documents
  • C_{\mathrm{curr}}^{\delta,<q}: the part of the current local context before the query q
  • d: the number of context groups mixed into one training example
  • \mathbf{q}: the current query vector
  • \mathbf{k}: a candidate key vector
  • \mathbf{v}: the value vector paired with that key
  • \tau: the softmax temperature

Read-aloud meaning: when training on one document, also show the model a few other documents, so it learns to prefer the same-document memory over distracting lookalikes.

Why it matters: this is the contrastive part of FoT. The method does not need a new contrastive loss; the ordinary language-model objective is enough once the batch is arranged this way.

That d knob controls how crowded the classroom is. If d is too small, the model never learns to separate memory from background noise. If d is too large too early, it can start ignoring the previous context altogether. The paper therefore starts with a small d and later increases it, which is a practical way to teach focus before asking for harder discrimination.

Compared with Memorizing Transformer, the difference is easy to state: FoT trains the memory path through the previous context itself and keeps the gradients flowing through those key-value pairs, while MT relies on a more detached memory mechanism. FoT also does not need long documents in training to the same degree, which is important because many real corpora are full of short files.

Why distraction is the real enemy

Figure 3 shows the failure mode that FoT is trying to fix. If the model has access to a lot of memory but has not learned to separate useful keys from irrelevant ones, attention gets spread around almost evenly.

What to notice is the trend, not the exact curve. The more unrelated contexts the layer can see, the easier it is for the positive context to get drowned out unless training has already taught the model to focus. That is the distraction issue: memory grows, but useful mass does not automatically stay on the right memory.

Purpose: measure how much attention the positive same-document memory actually receives.

rd=∑jw1j∑i=1d∑jwij≈1dr_d=\frac{\sum_j w_{1j}}{\sum_{i=1}^{d}\sum_j w_{ij}}\approx \frac{1}{d}rd​=∑i=1d​∑j​wij​∑j​w1j​​≈d1​

Intuition: if attention is spread evenly across all d contexts, the positive share shrinks like one over the number of contexts.

Symbols:

  • w_{ij}: attention weight assigned to the j-th pair in the i-th context group
  • r_d: fraction of total attention mass that lands on the positive same-document context
  • d: the number of context groups visible to the layer
  • i=1: the positive group from the current document

Read-aloud meaning: the more contexts the model sees, the smaller the positive share becomes unless training makes the positive keys stand out.

Why it matters: this is the simplest quantitative statement of why long memory can hurt. More memory is not helpful if the model cannot tell which memory belongs to the current thread.

That is also the deeper reason crossbatch helps perplexity later on long-document tasks. The model is not just given more storage; it is taught to use the storage selectively.

How LongLLaMA turns the idea into a retrofittable checkpoint

The paper then applies FoT to OpenLLaMA 3B and 7B checkpoints and calls the result LongLLaMA. This is the practical promise of the method: you do not have to rebuild a model from scratch to get longer effective context. You add memory layers, fine-tune them with crossbatch, and keep the base model largely intact.

The retrofit has three important engineering choices:

  • Only a subset of layers becomes memory-aware, so the change is targeted rather than global.
  • The fine-tuning context is still much shorter than the longest inference context the model eventually handles.
  • The LongLLaMA variant keeps the normal local positional encoding so it remains compatible with ordinary LLaMA inference code.

That last point is subtle but important. The authors do not lean on a new positional trick to get the result. The extra memory is shaped mainly by the training procedure. In that sense, the positional story is almost conservative: the local window keeps familiar LLaMA behavior, while the memory path is treated as global content that should not be over-anchored to exact positions.

Figure 1 shows the payoff.

What to notice is not just that the curve stays high. It is that the model keeps working far beyond the 8k training window. The 3B checkpoint reaches 94.5 percent at 100k context and still 73 percent at 256k. That is the telltale sign that FoT learned a retrieval behavior that extrapolates, instead of a brittle window-size trick that dies as soon as the prompt gets longer.

The same story shows up in downstream use. Longer prompts let few-shot tasks fit more demonstrations, so classification and question-answering accuracy can improve as the context grows. The point is not that more context is always better; the point is that, with FoT, extra context can stay useful instead of becoming noise.

What the paper is honest about

FoT is not free. Exact kNN search does not scale smoothly to very large memory banks, so the retrieval infrastructure becomes a real systems problem. Larger values of d also help, but they are limited by hardware. And because the method fine-tunes memory-enabled layers, it adds training cost rather than giving you a zero-cost upgrade.

There is also a behavior limit that the paper is careful about: if the negatives are weak or the memory becomes too noisy, attention can spread again and the retrieval quality collapses. So the method depends on the quality of the contrastive pressure, not just on the existence of memory.

What to remember

FoT is easiest to remember as “teach the memory to focus.”

It does three things in order:

  • add memory attention to a few layers,
  • train that memory with same-document positives and unrelated-document negatives,
  • use the sharpened key-value space to stretch context far beyond the fine-tuning window.

That is why the paper matters. It shows that long-context scaling is not only a positional encoding problem. It is also a discrimination problem. The model must learn what to ignore as carefully as it learns what to fetch.
