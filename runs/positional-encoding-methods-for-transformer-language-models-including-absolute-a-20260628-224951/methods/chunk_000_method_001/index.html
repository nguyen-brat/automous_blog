Context-Adaptive Positional Encoding (CAPE)

CAPE is easiest to understand as a small learned controller sitting on top of a relative-bias Transformer. The model still computes ordinary query-key similarity, but the positional term is no longer a fixed distance curve. Instead, CAPE lets a two-layer MLP look at the current attention pattern and decide how much positional pressure should remain, weaken, or be reinforced for this exact pair of tokens.

That matters because a fixed distance penalty is only an average answer. In one context, a far token should stay downweighted. In another, a distant token is exactly the one the model needs, and the default penalty is too blunt. CAPE tries to keep the useful inductive bias of relative position while making it reactive to the content already present in the sequence.

The problem CAPE is trying to solve

Additive relative positional encodings are already a strong interface: they inject position directly into pre-softmax attention scores instead of hiding it inside token embeddings. But most of the common versions still freeze the distance rule after training. Alibi, Kerple, and FIRE all give the model a positional prior, yet once training ends that prior is the same for every input.

That static behavior is the real bottleneck. A transformer reading a sentence is not always in the same situation. Sometimes the best match is local. Sometimes the meaningful signal is far away. If the model always pays the same distance tax, it either misses long-range evidence or overcommits to distant noise.

Think of a pair like “Hakuna” and “Matata” in a long paragraph. If the content match is strong, the model should be allowed to relax the usual distance penalty. If the pair is unrelated, the penalty can stay strong. CAPE is built around that idea: let semantics influence how position behaves.

Start from the ordinary additive bias

CAPE does not replace attention. It starts from the usual additive relative-bias form and then learns how to adapt it.

ARPE(X)=XWQ(XWK)⊤+BA_{\mathrm{RPE}}(X)=XW_Q(XW_K)^\top+BARPE​(X)=XWQ​(XWK​)⊤+B

Intuition: compute the normal content-based attention scores, then add a positional bias matrix on top.

Symbols:

  • XXX: token representations entering the attention block
  • WQW_QWQ​: query projection matrix
  • WKW_KWK​: key projection matrix
  • BBB: base positional bias matrix built from pairwise positions

Read-aloud meaning: normal attention scores get a position-dependent offset before softmax.

Why it matters: CAPE plugs into this exact slot. It keeps additive relative position, but makes the bias itself context-sensitive.

The paper uses Alibi, Kerple, and FIRE as examples of this additive family. They differ in how they shape BBB, but they all share the same limitation: once training is over, the bias is still a fixed function of position rather than a function of the current sequence.

What CAPE changes

CAPE keeps the base bias, but it no longer trusts that bias to be final. The method feeds both the content similarity matrix and the base positional bias into a small MLP and asks it to produce an adaptive correction.

Formula card: context-adaptive bias correction

Purpose: show how CAPE reshapes positional bias using the current attention context.

Equation:

ACAPE(X)=XWQ(XWK)⊤+f(XWQ(XWK)⊤,B)A_{\mathrm{CAPE}}(X)=XW_Q(XW_K)^\top+f\left(XW_Q(XW_K)^\top,B\right)ACAPE​(X)=XWQ​(XWK​)⊤+f(XWQ​(XWK​)⊤,B)

Intuition: use the current query-key similarity and the base positional bias as inputs to a learned function that returns a new positional term.

Symbols:

  • XXX: token representations
  • WQW_QWQ​: query projection matrix
  • WKW_KWK​: key projection matrix
  • BBB: base positional bias matrix from an additive RPE method
  • fff: learned adaptive bias function, implemented as a two-layer LeakyReLU MLP

Read-aloud meaning: ordinary attention scores are adjusted by a learned positional term that depends on both semantics and the original bias.

Why it matters: this is the main departure from static positional encoding. Distance is no longer the only thing deciding how strong the bias should be.

The important part is that the MLP operates on pairs, not on the whole sequence as one opaque object. For each query-key pair, it can see the current similarity pattern and the starting positional prior, then emit a correction for that pair. That gives CAPE enough flexibility to mimic familiar additive curves when it should, while still allowing the bias to bend under different contexts.

What the learned bias looks like

The paper’s visualization makes the design goal concrete.

What to notice here is not a single fixed decay curve. Some heads stay strongly local, while others create “anti-local” behavior and pull attention toward far-away keys when the context justifies it. That is the practical meaning of adaptivity: CAPE is not forced into one universal distance law.

It also explains why the method is more than a fancy reparameterization of FIRE. FIRE uses an MLP too, but it learns position-to-bias mappings without looking at the current semantic similarity. CAPE adds that missing signal, so the same distance can be treated differently depending on what the tokens are already saying to each other.

Residual correction is the safer version

The paper also presents a residual form that keeps the original bias visible and lets the MLP act as a correction term.

Formula card: residual CAPE

Purpose: show the version that preserves the base positional bias explicitly.

Equation:

ACAPE(X)=XWQ(XWK)⊤+B+f(XWQ(XWK)⊤,B)A_{\mathrm{CAPE}}(X)=XW_Q(XW_K)^\top+B+f\left(XW_Q(XW_K)^\top,B\right)ACAPE​(X)=XWQ​(XWK​)⊤+B+f(XWQ​(XWK​)⊤,B)

Intuition: keep the original distance prior, then add a learned adjustment on top.

Symbols:

  • XXX: token representations
  • WQW_QWQ​: query projection matrix
  • WKW_KWK​: key projection matrix
  • BBB: base positional bias matrix
  • fff: adaptive correction function

Read-aloud meaning: the model keeps the prior positional rule and learns only the correction needed for the current context.

Why it matters: when the base bias is already reasonable, a residual correction is easier to optimize and less likely to throw away useful locality.

This is also where the paper’s wiring choices become interesting. In the experiments, concatenation-based variants outperform the add-residual form, and the two concat variants stay close to each other. The practical lesson is not that one exact wiring is magical. It is that CAPE works best when it can read both the content signal and the base bias without collapsing them too early.

Why multi-head CAPE is stronger

The single-head story is the cleanest way to see the idea, but the real model runs in multi-head attention. CAPE extends by feeding all headwise similarity values and positional biases into the same MLP at once.

That means the adaptive controller sees a richer semantic picture. Instead of deciding in isolation for one head, it can coordinate across heads and return one correction per head. The paper describes this as a concatenation of the per-head similarity and bias signals, with the MLP outputting one adaptive value for each head.

This joint processing is useful for two reasons.

First, different heads often specialize. One head may track local syntax, another may track longer-range evidence, and a third may care about a different retrieval pattern. If the adaptive module sees all of them together, it can preserve that specialization instead of forcing every head into the same curve.

Second, the same logic stays parallelizable. CAPE is still an attention-side modification, not a separate recurrent pass over the sequence.

The tradeoff: more flexibility, more compute

The cost is real. CAPE adds pairwise MLP work on top of the base attention computation, so longer sequences make the overhead more visible. The paper also shows that larger hidden widths help expressiveness, but they raise cost too.

The useful practical takeaway is that small hidden sizes already help, but too-small widths can underfit. The authors recommend keeping the hidden dimension near the head count so the adaptive module has enough capacity without becoming wasteful.

That tradeoff is why CAPE should be read as a precision tool, not a free upgrade. It is strongest when the base additive bias is already a decent prior and the extra correction can meaningfully adapt that prior to the current sequence.

What the experiments say in plain language

The empirical story matches the design intuition.

  • On Arxiv with training length 128, CAPE-Kerple reaches perplexity 5.00 at evaluation length 8192, while plain Kerple is 31.93.
  • On the 1024 training-length setting, CAPE-FIRE reaches 3.91 at evaluation length 8192 and beats Alibi’s 4.28.
  • On CHE, CAPE with Kerple or FIRE is best on 10 of 11 permutation-variant tasks.

The pattern is consistent: CAPE helps most when the model has to decide, in context, whether distance should still matter. It improves not only far extrapolation, but also in-window performance, which suggests it is not just loosening locality everywhere. It is using the whole sequence more intelligently.

Limits to keep in mind

CAPE is not a universal replacement for every positional method. It depends on a chosen additive relative-bias family, so it naturally fits Alibi, Kerple, and FIRE-style schemes rather than rotary methods directly. It also adds memory and compute overhead, especially as context length grows.

The method is strongest when the base bias is already meaningful and the adaptive MLP has enough width to learn a useful correction. If the base bias is weak, or if the hidden dimension is too small, CAPE can add cost without enough payoff.

What to remember

The simplest way to remember CAPE is this: it turns positional bias from a fixed curve into a learned correction that listens to the current attention pattern. Distance still matters, but it is no longer the only thing that matters.

That is why the method is useful. It preserves the long-context intuition of additive relative bias, but gives the model a way to say, “this far token is actually important right now.” In a Transformer, that small shift is often enough to make long-context behavior much more robust.
