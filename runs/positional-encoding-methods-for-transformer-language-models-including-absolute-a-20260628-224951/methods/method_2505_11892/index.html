How to Compute the RoPE Attention in Almost Linear Time

RoPE changes the shape of attention. That is the whole reason this paper exists.

For ordinary fast-attention algorithms, the useful mental model is “the score matrix is close to low rank, so approximate it that way.” RoPE breaks that habit. Once position is folded into the score through rotations, the matrix is no longer the kind of object those older tricks were built for. The paper’s move is to stop forcing RoPE back into the low-rank box and instead exploit the structure RoPE actually leaves behind: offset patterns that behave like Toeplitz matrices.

RoPE starts with rotation, not with a lookup table

The most basic RoPE operation is a 2D rotation block. A pair of coordinates gets rotated by an angle that depends on position.

Formula card: the 2D rotation block

Purpose: show the primitive operation that injects position into RoPE.

Equation:

R(θ)=[cos⁡θ−sin⁡θsin⁡θcos⁡θ]R(\theta)= \begin{bmatrix} \cos\theta & -\sin\theta \\ \sin\theta & \cos\theta \end{bmatrix}R(θ)=[cosθsinθ​−sinθcosθ​]

Intuition: instead of adding a position embedding as a separate vector, RoPE rotates each feature pair by a position-dependent angle. Position becomes geometry.

Symbols:

  • R(\theta): the 2D rotation matrix used inside RoPE blocks
  • \theta: the rotation angle for that feature pair

Read-aloud meaning: rotate a two-dimensional feature pair by an angle tied to position.

Why it matters: the full RoPE operator is just a block diagonal stack of these 2D rotations. Once you stack many such blocks, the attention score depends on relative offset, not just raw content.

That is the key thing to remember. RoPE does not merely tag tokens with position. It makes the query-key comparison itself position-aware, so pairs at different distances are compared through different phases.

The paper solves a more general problem than vanilla RoPE

The algorithm does not only handle the exact RoPE form from the introduction. It solves a broader attention problem called ARAttC, where the offset i-j selects a sparse matrix W_{i-j}. RoPE is a special case of this setup.

Formula card: the generalized RoPE attention target

Purpose: define the output the algorithm is trying to approximate.

Equation:

Ai,j=exp⁡ ⁣(Qi,∗Wi−jKj,∗⊤/d),D=diag⁡(A1n),ARAttC=D−1AVA_{i,j}=\exp\!\left(Q_{i,*}W_{i-j}K_{j,*}^{\top}/\sqrt{d}\right), \qquad D=\operatorname{diag}(A\mathbf{1}_n), \qquad \mathsf{ARAttC}=D^{-1}AVAi,j​=exp(Qi,∗​Wi−j​Kj,∗⊤​/d​),D=diag(A1n​),ARAttC=D−1AV

Intuition: compute a position-dependent score for each query-key pair, exponentiate it, normalize each row, and use the result to mix the value vectors.

Symbols:

  • Q, K, V: the query, key, and value matrices, each in \mathbb{R}^{n\times d}
  • W_{i-j}: the position-dependent weight matrix chosen by relative offset
  • d: the embedding dimension
  • A: the unnormalized attention matrix
  • D: the diagonal matrix of row sums of A
  • \mathbf{1}_n: the length-n all-ones vector
  • \mathsf{ARAttC}: the normalized attention output the paper wants to approximate

Read-aloud meaning: score each token pair using a relative-position matrix, turn scores into softmax-style weights, and apply those weights to the values.

Why it matters: this is the exact abstraction the paper needs. RoPE fits inside it, but the definition is broader, so the algorithm covers more than one fixed rotary recipe.

The bounded-entry assumptions are part of the story too. The paper works in the regime where the entries of Q, K, and V are bounded by B, and each W_i is sparse with support size O(d). That is not a cosmetic detail. It is the condition that keeps the polynomial approximation under control.

Why the old low-rank intuition stops working

The usual polynomial-method fast-attention story is: approximate the exponential by a low-degree polynomial, and use that polynomial to turn a hard dense matrix into something low-rank enough to multiply quickly.

RoPE breaks the first assumption, not the exponential itself. The problem is the matrix before the exponential. With relative-position rotations, the score matrix can vary by diagonal instead of by low-rank factors. In the simplest mental picture, entries on the same diagonal share the same offset pattern. That is Toeplitz structure.

This is the wrong shape for a low-rank argument, but it is the right shape for FFT-based matrix multiplication.

  • Low rank says “compress by a few global directions.”
  • Toeplitz says “compress by relative offset.”
  • RoPE keeps the second kind of structure, not the first.

That shift is the paper’s main conceptual move.

The linear warm-up shows the hidden structure cleanly

The paper first strips away the exponential and studies RoPE linear attention. This is a good move, because it lets you see the matrix skeleton before the softmax-like nonlinearity gets in the way.

Once you isolate one active coordinate pair (l1, l2) from the sparse support, the score matrix slice has a very clean form: a diagonal scaling from the query side, a Toeplitz core that depends only on i-j, and another diagonal scaling from the key side.

Formula card: one coordinate slice becomes rescaled Toeplitz

Purpose: show the exact structure that makes the linear RoPE case FFT-friendly.

Equation:

Ai,jℓ1,ℓ2=Qi,ℓ1(Wi−j)ℓ1,ℓ2Kj,ℓ2,Aℓ1,ℓ2=diag⁡(Q∗,ℓ1) Cℓ1,ℓ2 diag⁡(K∗,ℓ2)A^{\ell_1,\ell_2}_{i,j} =Q_{i,\ell_1}(W_{i-j})_{\ell_1,\ell_2}K_{j,\ell_2}, \qquad A^{\ell_1,\ell_2} =\operatorname{diag}(Q_{*,\ell_1})\,C^{\ell_1,\ell_2}\,\operatorname{diag}(K_{*,\ell_2})Ai,jℓ1​,ℓ2​​=Qi,ℓ1​​(Wi−j​)ℓ1​,ℓ2​​Kj,ℓ2​​,Aℓ1​,ℓ2​=diag(Q∗,ℓ1​​)Cℓ1​,ℓ2​diag(K∗,ℓ2​​)

with

Ci,jℓ1,ℓ2=(Wi−j)ℓ1,ℓ2C^{\ell_1,\ell_2}_{i,j}=(W_{i-j})_{\ell_1,\ell_2}Ci,jℓ1​,ℓ2​​=(Wi−j​)ℓ1​,ℓ2​​

Intuition: the Q column rescales the rows, the K column rescales the columns, and the middle matrix only depends on the relative offset. That middle piece is Toeplitz.

Symbols:

  • A^{\ell_1,\ell_2}: the contribution from one active coordinate pair
  • Q_{*,\ell_1}: the \ell_1-th column of Q
  • K_{*,\ell_2}: the \ell_2-th column of K
  • C^{\ell_1,\ell_2}: the Toeplitz core built from the offset-dependent entry of W_{i-j}
  • \ell_1,\ell_2: indices of the active support coordinate

Read-aloud meaning: one coordinate slice of the score matrix is a Toeplitz matrix wrapped by two diagonal scalings.

Why it matters: FFT can multiply a Toeplitz matrix with a vector in O(n log n) time after the usual circulant embedding trick. The diagonal factors are cheap, so the whole slice stays fast.

The support size is only O(d), so the whole linear attention matrix is just the sum of a manageable number of these slices. That is why the linear case already looks promising: the offset structure survives decomposition.

Putting the exponential back in

Now the paper turns the exponential back on. This is where the polynomial method returns, but in a more structured form than in earlier fast-attention work.

The exponential is approximated on the bounded score range by a low-degree polynomial.

Formula card: polynomial approximation of the exponential

Purpose: replace the expensive exponential with a controlled low-degree polynomial.

Equation:

∣P(x)−ex∣<ϵfor x∈[0,B]\lvert P(x)-e^x\rvert<\epsilon \qquad \text{for } x\in[0,B]∣P(x)−ex∣<ϵfor x∈[0,B]

Intuition: if every score stays inside a bounded interval, then a carefully chosen polynomial can mimic the exponential closely enough for the final attention output to stay accurate.

Symbols:

  • P: the approximating polynomial
  • x: a score value fed into the exponential
  • B: the score bound coming from the bounded-entry assumption
  • \epsilon: the allowed approximation error

Read-aloud meaning: on a bounded input range, a polynomial can stand in for the exponential with arbitrarily small error.

Why it matters: the degree stays low enough in the regime B=o(\sqrt{\log n}) and \epsilon=1/\mathrm{poly}(n) that the expansion produces only subpolynomially many structured terms.

That is the bridge from “one structured score matrix” to “a sum of many structured score matrices.”

The key closure property is what makes this work: when the polynomial is expanded into monomials, each monomial becomes an entrywise product of rescaled Toeplitz pieces, and the paper proves that this Hadamard product stays in the same rescaled-Toeplitz family. So the polynomial does not break the FFT-friendly structure. It preserves it.

This is the real innovation. Earlier polynomial-method attention algorithms used low-degree polynomials to argue that the whole matrix is approximately low rank. Here, the polynomial is used to keep the matrix inside a different structured family instead.

Why the runtime stays almost linear

Once the polynomial expansion is in place, the proof becomes a counting argument plus fast matrix-vector products.

The paper’s final computation has three stages:

  1. Approximate A\mathbf{1}_n so the normalization matrix D can be recovered.
  2. Approximate AV, one column of V at a time.
  3. Combine the approximation of D^{-1} with the approximation of AV to get D^{-1}AV.

Each structured term costs O(n log n) to multiply by a vector. The number of terms coming from the polynomial expansion is only n^{o(1)} in the regime the paper studies. Since d=O(log n), the extra factor from the coordinate support also stays mild.

That is how the full algorithm reaches n^{1+o(1)} time.

The running time claim is therefore not magic. It is the product of three small facts:

  • the score matrices are sparse in coordinate support,
  • the exponential can be replaced by a low-degree polynomial on a bounded range,
  • Toeplitz-like pieces can be multiplied by FFT in nearly linear time.

The theorem that falls out is sharp: when B=o(\sqrt{\log n}), the algorithm is almost linear; when B grows to \Theta(\sqrt{\log n}), the lower bound kicks in.

The lower bound says the boundary is real

The paper does not only give an upper bound. It also shows that the bounded-entry assumption is necessary in the fine-grained sense.

Under SETH, once the entry bound reaches B=\Theta(\sqrt{\log n}), there is no truly subquadratic algorithm for the generalized problem. The proof is short because the generalized RoPE problem contains classical attention as a special case: set all the W_i matrices to the identity, and the position machinery disappears.

So the lower bound is not an artifact of the new proof technique. It is the same hardness barrier that already appears in ordinary attention, carried over to the RoPE setting.

That is why the theorem pair matters together. The upper bound says RoPE can still be fast in the bounded regime. The lower bound says that regime is close to the best possible one.

What to remember

If you want one mental hook, keep this:

RoPE does not destroy structure. It changes the structure from low rank to offset-based Toeplitz geometry.

From there, the method is easy to remember:

  • split the score matrix into sparse coordinate slices,
  • recognize each slice as a rescaled Toeplitz matrix,
  • approximate the exponential with a polynomial,
  • keep the resulting monomials inside the same structured family,
  • multiply each piece with FFT,
  • then normalize the rows.

The main limitations are equally clear. The paper is purely theoretical, so it does not report empirical speedups on real RoPE kernels. It also works under bounded-entry assumptions and focuses on d=O(log n), so it does not automatically settle the practical behavior of large modern models with wider hidden states.

Even with those caveats, the paper makes a strong conceptual point: when relative-position structure breaks low rank, do not force the old approximation tool. Find the preserved structure and build the algorithm around that instead.
