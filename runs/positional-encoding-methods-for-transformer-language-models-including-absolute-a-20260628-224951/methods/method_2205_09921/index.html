KERPLE: Kernelized Relative Positional Embedding for Length Extrapolation

KERPLE starts from a practical failure mode: a Transformer is trained on a manageable context length, then later gets asked to read far beyond it. If the position mechanism only knows how to behave inside the training window, the model can look sharp at 512 tokens and fall apart at 4k or 16k. KERPLE’s answer is not to add a bigger position table. It is to learn a distance curve and use that curve directly inside attention.

That is the key mental model to keep in mind:

token content similarity + learned distance bias -> softmax -> attention weights

The paper’s Figure 1 makes that idea concrete. It shows a tiny per-head kernel with just a few learnable scalars, and the dashed future row is the extrapolation step that appears only at test time. The model is not memorizing absolute positions; it is reusing the same distance rule when the sequence gets longer.

Look at the image as a training-to-inference story. During training, the kernel only sees short relative distances. At inference, the same kernel is evaluated on longer gaps, and the attention pattern extends naturally instead of needing a new table of position IDs. That is the whole point of the method.

The attention score gets one extra ingredient

The most important thing KERPLE changes is the pre-softmax attention score. The content-based query-key match is still there, but a distance-shaped bias is added before normalization.

Purpose: show where relative position enters the attention calculation.

Equation:

am,n=exp⁡((qm⊤kn+k~(m,n))/d)∑iexp⁡((qm⊤ki+k~(m,i))/d)a_{m,n}=\frac{\exp\left((\mathbf{q}_m^\top\mathbf{k}_n+\tilde{k}(m,n))/\sqrt{d}\right)}{\sum_i \exp\left((\mathbf{q}_m^\top\mathbf{k}_i+\tilde{k}(m,i))/\sqrt{d}\right)}am,n​=∑i​exp((qm⊤​ki​+k~(m,i))/d​)exp((qm⊤​kn​+k~(m,n))/d​)​

Intuition: attention is still comparing query and key content, but now each pair also gets a learned distance bonus or penalty before Softmax turns the row into probabilities.

Symbols:

  • a_{m,n}: attention weight from position m to position n.
  • \mathbf{q}_m: query vector at position m.
  • \mathbf{k}_n: key vector at position n.
  • \tilde{k}(m,n): relative positional kernel bias between positions m and n.
  • d: hidden dimension used for the standard attention scaling.
  • m, n: token positions.

Read-aloud meaning: the model scores each token pair by combining content similarity with a learned distance term, then normalizes the row with Softmax.

Why it matters: this is the interface where all of the paper’s theory lands. If you understand this line, you understand where KERPLE lives inside the Transformer.

Why kernels are useful here

KERPLE uses kernels because relative position is really a question about how similarity changes with distance. A kernel gives you a principled curve instead of a hand-written table. The paper focuses on conditionally positive definite, or CPD, kernels because many distance-shaped functions fit that class even when they are not ordinary positive definite kernels yet.

The important part is not the abstract definition. The important part is the behavior: a CPD kernel can describe a distance-like relation, and then the paper shows that a large enough constant shift can turn it into a PD kernel. That sounds technical, but Softmax makes the shift almost free in practice because row-wise normalization ignores shared offsets.

Purpose: explain why a CPD kernel can still be used as an attention bias.

Equation:

exp⁡(xi)∑jexp⁡(xj)=exp⁡(xi+c)∑jexp⁡(xj+c)\frac{\exp(x_i)}{\sum_j \exp(x_j)}=\frac{\exp(x_i+c)}{\sum_j \exp(x_j+c)}∑j​exp(xj​)exp(xi​)​=∑j​exp(xj​+c)exp(xi​+c)​

Intuition: if the same constant is added to every logit in a row, the relative ranking does not change, so Softmax returns the same attention weights.

Symbols:

  • x_i: the unnormalized logit for key i.
  • c: any constant added to every logit in that row.
  • \exp(\cdot): the exponential function used by Softmax.

Read-aloud meaning: shifting every score by the same amount does not change the normalized attention distribution.

Why it matters: the paper can design the positional bias for its shape, then hide the required constant offset inside Softmax instead of explicitly tracking it.

This is why the theory feels useful rather than ornamental. KERPLE is not asking attention to become something different. It is asking attention to keep doing its usual inner-product work while the positional part becomes a better-behaved kernel curve.

The two curves that matter in practice

The paper tries a family of CPD curves, but the practical story is very simple. The winner is a monotonic decay curve that is easy to learn and easy to extrapolate. The two main choices are a power curve and a logarithmic curve.

Purpose: describe the two practical relative-bias shapes used by KERPLE.

Equation:

k~(m,n)=−r1∣m−n∣r2\tilde{k}(m,n)=-r_1 |m-n|^{r_2}k~(m,n)=−r1​∣m−n∣r2​

Intuition: this is a distance penalty that grows like a power law. It can make far-away tokens drop quickly.

Symbols:

  • r_1 > 0: controls the strength of the penalty.
  • r_2: controls how sharply the penalty grows with distance, with the paper using 0 < r_2 \le 2.
  • |m-n|: the absolute distance between the two token positions.

Read-aloud meaning: the farther apart two tokens are, the more negative the bias becomes, and the increase follows a learnable power curve.

Why it matters: the power curve gives a clean monotonic prior, but it can fade too aggressively when the sequence gets long.

The logarithmic version keeps the same basic idea, but the decay is gentler.

Purpose: describe the slower-decaying curve that becomes KERPLE’s best practical choice.

Equation:

k~(m,n)=−r1log⁡(1+r2∣m−n∣)\tilde{k}(m,n)=-r_1 \log(1+r_2 |m-n|)k~(m,n)=−r1​log(1+r2​∣m−n∣)

Intuition: the log curve still penalizes distant tokens, but it does so more slowly, so the model keeps a useful window open farther into the sequence.

Symbols:

  • r_1 > 0: controls the overall penalty strength.
  • r_2 > 0: controls how quickly the curve starts bending.
  • \log(1+\cdot): the logarithm, which grows slowly for large inputs.

Read-aloud meaning: distance still hurts attention, but the penalty grows gently enough that long-range tokens are not crushed immediately.

Why it matters: this is the sweet spot the paper finds in practice. It extrapolates well, trains quickly, and stays simple enough to fit in a few parameters per head.

The nice thing about this design is that it does not need a huge parameter table. The main log variant uses only a small number of learnable scalars per head, shared across layers. The paper also explores more complicated kernel shapes, but the extra flexibility gives only marginal gains or worse speed. In other words, the simple curve is not a compromise on principle; it is the best engineering choice the authors found.

What the learned curves look like

The paper’s learned-kernel plots make the behavior much easier to trust. The log variant does not force every head to collapse into a tiny local window. Some heads stay relatively flat for longer, which means they can still pay attention to distant evidence.

The main thing to notice is the shape, not the exact y-values. In the log version, the curves bend slowly, so some heads keep useful signal farther away. The power version and ALiBi drop more aggressively. That is why the log variant can behave like a longer-range attention rule without becoming a giant absolute-position table.

This also matches the paper’s position-wise perplexity analysis. The log model stays below a fixed 512-token window baseline for much of the long evaluation range, which means it is using more than just local context. It is not simply pretending to be a short window model. It is actually extracting information farther away.

How this relates to ALiBi and T5

KERPLE is easiest to understand as a family of relative-bias rules rather than a completely new Transformer. ALiBi is a special case inside that family: it uses a simple linear distance penalty. T5 lives nearby conceptually because it also maps relative distance into a compact bias, but it does so through log-binned buckets.

That comparison is useful because it shows what KERPLE is not doing. It is not inventing a new attention backbone. It is not changing the token content path. It is only changing the shape of the distance curve that gets added to attention logits.

The log KERPLE variant ends up with a nice tradeoff:

  • fewer learned parameters than T5’s bucketed bias,
  • smoother distance behavior than a bucket table,
  • better long-context extrapolation than the standard baselines in the paper.

The one thing it does not do is express non-monotonic distance relations very well. Once you decide on a monotonic log curve, you are saying that farther generally means less attention. That is a sensible prior for language modeling, but it is still a prior.

What the experiments are really saying

The experimental pattern is consistent across the three datasets the paper uses: OpenWebText2, GitHub, and ArXiv. The models are trained on 512-token sequences and then tested far beyond that length, all the way to 16,384 tokens.

The broad takeaway is that the logarithmic variant is the practical winner. It is faster than the power variant, faster than T5 on the GitHub setup, and generally the strongest extrapolator at long lengths among the relative-bias baselines the paper compares against. On ArXiv at length 16,384, the log variant reaches 5.07 perplexity, compared with 5.78 for T5 and 6.25 for Rotary. On GitHub at the same length, it reaches 2.75 perplexity, compared with 3.02 for T5 and 3.35 for Rotary.

The short-length story is a little more nuanced. T5 can sometimes look as good or slightly better at shorter extrapolation lengths, which is a reminder that KERPLE is optimized for graceful long-range behavior rather than for winning every narrow-context comparison. That is the right tradeoff to care about if the real problem is context-length extrapolation.

Where the method stops

KERPLE is a better relative bias, not a magic way to make attention cheap. The attention pattern is still quadratic in sequence length. The paper also leans on monotonic distance decay, so it does not directly model positional relations that bounce up and down with distance. And the benefit can weaken in some short-length or larger-model settings, which is a normal warning sign for any learned bias curve.

That limitation is actually useful to remember. KERPLE is strongest when the task wants a controlled notion of distance and the model has to survive context lengths far beyond training. It is less compelling if you need a more structured, non-monotonic positional rule.

What to remember

If you want one sentence, keep this:

KERPLE learns a small, shift-friendly distance curve and adds it to attention, so the model can extrapolate to longer contexts without giving up the normal inner-product view of attention.

Three details are worth keeping in your head:

  • The bias is a kernel curve over relative distance, not a per-position lookup table.
  • Softmax absorbs the constant shift, so the kernel can be turned into an attention-friendly bias.
  • The logarithmic curve is the practical sweet spot because it keeps far tokens visible longer than the power curve.
