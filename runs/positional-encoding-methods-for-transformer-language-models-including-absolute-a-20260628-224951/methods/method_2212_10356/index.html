A Length-Extrapolatable Transformer

The paper’s core claim is simple but easy to miss: long-context behavior is not just about whether a positional encoding is absolute or relative. What really matters is the receptive field the model actually learns to use. If the training length covers that useful region, the model can stretch to longer inputs. If it does not, the extra tokens become either ignored or dangerous.

That is why this paper is interesting. It does not just propose another positional trick. It turns length extrapolation into a question about attention coverage.

You can keep the whole story in one pipeline:

token positions -> attention score bias -> empirical receptive field -> stable or unstable long-context prediction

The paper first diagnoses ALiBi and windowed attention, then measures the model’s empirical receptive field with gradients, and finally turns sinusoidal structure into a new relative bias called Sandwich.

On ArXiv at 2048 tokens, Sandwich gets 5.02 perplexity while ALiBi reaches 5.58. That is not just a small benchmark bump. It is evidence that a better bias shape can make longer context usable without learnable positional parameters.

ALiBi is a soft window

ALiBi looks like a simple linear penalty on distant tokens, but the paper’s experiments show that this is basically a soft version of windowed attention. When the slope is large, distant tokens get pushed down so hard that the head behaves almost as if it only had a local window. When the slope is too small, the bias is too loose and long-range attention becomes unstable.

That is the first useful reframing in the paper: the decisive factor is not slope diversity by itself. It is how much actual distance the bias allows the model to use.

Formula card: ALiBi as a soft window

Purpose: turn relative distance into a head-specific recency penalty.

Equation:

lmn={⟨qm,kn⟩−αh(m−n),m≥n−∞,otherwisel_{mn} = \begin{cases} \langle \mathbf{q}_m, \mathbf{k}_n \rangle - \alpha_h (m-n), & m \ge n \\ -\infty, & \text{otherwise} \end{cases}lmn​={⟨qm​,kn​⟩−αh​(m−n),−∞,​m≥notherwise​

Intuition: the farther back a key token is, the more score it loses. Large slopes make attention sharply local, so ALiBi starts to resemble a window.

Symbols:

  • l_{mn}: the pre-softmax attention logit from query position m to key position n.
  • \mathbf{q}_m: query vector at position m.
  • \mathbf{k}_n: key vector at position n.
  • \alpha_h: the head-specific slope; in the paper’s schedule it is set from 1 / 2^h.
  • m, n: token positions.

Read-aloud meaning: attention to a farther-back token is penalized more strongly.

Why it matters: this is why ALiBi can extrapolate at all. It quietly creates a local bias pattern, and that local pattern is what the model actually learns to rely on.

Windowed attention makes this idea explicit. Instead of a soft penalty, it sets a hard cutoff. The paper shows that the two behave similarly once you ask a more basic question: how wide is the region that actually influences the next prediction?

The real diagnostic is the receptive field

The paper borrows a receptive-field idea from convolutional models and applies it to attention. The point is not theoretical reach. It is actual usage.

If a model could, in principle, attend across a huge range but in practice only uses the recent slice, then the effective context is much smaller than the architecture suggests. That gap is exactly what the gradient analysis measures.

The trick is very practical: compute the gradient of the next-token prediction with respect to each input embedding, normalize the gradient mass across positions, and then accumulate it from the newest token backward. The first position that pushes the cumulative mass above 99% is taken as the empirical receptive field.

Formula card: measuring the empirical receptive field

Purpose: estimate how much of the context actually contributes to the next-token prediction.

Equation:

sm=∥gm∥2∑n=1Lex∥gn∥2,cm=∑n=mLexsn,ERF=min⁡{m∣cm>0.99}s_m = \frac{\lVert \mathbf{g}_m \rVert_2}{\sum_{n=1}^{L_{ex}} \lVert \mathbf{g}_n \rVert_2}, \qquad c_m = \sum_{n=m}^{L_{ex}} s_n, \qquad \mathrm{ERF} = \min\{ m \mid c_m > 0.99 \}sm​=∑n=1Lex​​∥gn​∥2​∥gm​∥2​​,cm​=n=m∑Lex​​sn​,ERF=min{m∣cm​>0.99}

Intuition: first measure how much gradient each token gets, then ask how far back you need to go before almost all of the influence is covered.

Symbols:

  • \mathbf{g}_m: gradient of the next-token prediction with respect to the input embedding at position m.
  • s_m: normalized share of gradient mass at position m.
  • c_m: cumulative gradient mass from position m to the end of the sequence.
  • L_{ex}: evaluation length.
  • \mathrm{ERF}: empirical receptive field.

Read-aloud meaning: the empirical receptive field is the shortest recent prefix that contains 99% of the gradient influence.

Why it matters: this is the paper’s main explanation for why some length-extrapolatable models work and others fail. If training length does not cover the useful receptive field, the model is forced to use untrained long-range interactions.

This is also where the paper makes a subtle but important point. A model does not need to use all of its theoretical receptive field to extrapolate. It only needs the part it really depends on to be covered during training.

That is why the paper can explain both stable plateaus and sharp explosions. If extra context is effectively ignored, perplexity stays flat. If the model starts using long-range interactions it never learned safely, perplexity blows up.

Sinusoidal and Rotary fail this test in a very visible way. Their gradient mass does not stay nicely bounded inside the training window, so when the context grows they end up relying on positions the model never truly learned to handle. The result is not graceful extrapolation but brittle boundary behavior.

Sandwich starts from sinusoidal positions and keeps only the useful part

The clever move in Sandwich is to take sinusoidal positional structure and remove the parts that make it awkward for extrapolation. The paper keeps the position-position inner product, because that collapses to a clean function of relative distance. In other words, absolute indices disappear and only the gap between tokens remains.

Formula card: turning sinusoidal structure into a relative bias

Purpose: convert sinusoidal position features into a distance-only attention bias.

Equation:

pm⊤pn=∑i=1dˉ/2sin⁡ ⁣(m100002i/dˉ)sin⁡ ⁣(n100002i/dˉ)+cos⁡ ⁣(m100002i/dˉ)cos⁡ ⁣(n100002i/dˉ)=∑i=1dˉ/2cos⁡ ⁣(m−n100002i/dˉ)\mathbf{p}_m^\top \mathbf{p}_n = \sum_{i=1}^{\bar d/2} \sin\!\left(\frac{m}{10000^{2i/\bar d}}\right) \sin\!\left(\frac{n}{10000^{2i/\bar d}}\right) + \cos\!\left(\frac{m}{10000^{2i/\bar d}}\right) \cos\!\left(\frac{n}{10000^{2i/\bar d}}\right) = \sum_{i=1}^{\bar d/2} \cos\!\left(\frac{m-n}{10000^{2i/\bar d}}\right)pm⊤​pn​=i=1∑dˉ/2​sin(100002i/dˉm​)sin(100002i/dˉn​)+cos(100002i/dˉm​)cos(100002i/dˉn​)=i=1∑dˉ/2​cos(100002i/dˉm−n​)

Intuition: the sine and cosine cross-terms collapse through the trig identity, so the score depends on relative offset instead of absolute position.

Symbols:

  • \mathbf{p}_m, \mathbf{p}_n: sinusoidal position feature vectors at positions m and n.
  • \bar d: positional feature size used by Sandwich.
  • i: index over sinusoidal feature pairs.
  • m-n: the relative distance between the two token positions.

Read-aloud meaning: comparing two sinusoidal position vectors produces a sum of cosine terms over their distance gap.

Why it matters: this is the core transformation. It turns a familiar absolute-style positional construction into a relative bias that can be used beyond the training length.

The paper then centers and rescales that raw similarity so it behaves like an attention bias instead of a raw similarity score.

Formula card: head-wise compression of Sandwich

Purpose: shape the cosine-based similarity into a usable attention bias for each head.

Equation:

bmn(h)=pm⊤pn−dˉ/2hb_{mn}^{(h)} = \frac{\mathbf{p}_m^\top \mathbf{p}_n - \bar d / 2}{h}bmn(h)​=hpm⊤​pn​−dˉ/2​

Intuition: subtract the maximum self-match so the bias is centered, then divide by a head-specific compression ratio so different heads can see different decay strengths.

Symbols:

  • b_{mn}^{(h)}: final positional bias used by head h.
  • \mathbf{p}_m^\top \mathbf{p}_n: raw cosine-sum similarity from the previous formula.
  • \bar d / 2: the maximum value at zero distance.
  • h: a head-specific compression ratio borrowed from the ALiBi-style head schedule.

Read-aloud meaning: turn the raw sinusoidal similarity into a centered, head-scaled distance bias.

Why it matters: this is what gives Sandwich its shape. The bias is still distance-aware, but it is no longer brittle or tied to absolute indices.

The role of \bar d is easy to misunderstand, so the paper makes it a separate hyperparameter on purpose. It is not the model hidden size. It is a shape knob. Larger \bar d makes the bias more window-like and strengthens the recency preference.

What to notice here is not just the peak near the most recent tokens. It is the tail. Sandwich still favors recent context, but it does not turn distant tokens into total dead space. The far context is flattened into a shared region instead of being treated as a brittle set of unique positions.

That is why the paper compares Sandwich with a smoothed log fit, KERPLE, and T5. They all share the same broad shape: a fast drop near the present, then a flatter distant band. The exact formulas differ, but the bias pattern is similar. The model can average over distant tokens instead of trying to memorize each one separately.

This averaging idea is the paper’s main explanation for why Sandwich can actually use tokens beyond L_tr.

If a bias decays too fast, far tokens are effectively masked out. If it decays too slowly, the empirical receptive field becomes so large that training no longer covers it. Sandwich sits in the middle. It gives the model enough long-range mass to benefit from older context, but the far tokens are smooth enough to behave like denoised history rather than new brittle exceptions.

That also explains the paper’s limit case. Very long contexts can start to reintroduce noise once the averaging effect saturates. So the method is not saying “more context is always better.” It is saying “better-shaped context can stay useful longer.”

Why this is different from just using a bigger window

It is tempting to think Sandwich is just a fancy windowed attention variant. It is not.

A hard window ignores everything outside its cutoff. Sandwich keeps a soft tail. That tail matters because it lets the model use distant information in a controlled way. The distant tokens are not memorized individually, but they are still part of the bias pattern the model learns during training.

That is why the paper says Sandwich is the first parameter-free relative positional embedding in this setting that truly uses information beyond the training length. The long-context benefit is not coming from extra learned parameters. It is coming from the bias shape itself.

The evidence is strongest on ArXiv and GitHub, where Sandwich stays competitive at the training length and then improves or holds up better once the test length grows. On OpenWebText2, the margin is smaller, which is a good reminder that the method is still tuned for language-model perplexity rather than every possible sequence task.

What to remember

The cleanest way to remember this paper is:

  • Length extrapolation is a receptive-field problem, not just a naming problem for positional embeddings.
  • ALiBi works because it behaves like a soft window.
  • A model extrapolates when the training length covers the receptive field it really uses.
  • Sandwich gets its relative bias by collapsing sinusoidal position vectors into a cosine-of-distance sum.
  • The head-wise scaling and the \bar d knob turn that raw distance signal into a usable, log-like decay.

The main limitation is still recency bias. Sandwich, KERPLE, and T5 all favor nearby tokens, which is helpful for ordinary text but not enough for tasks where every token matters equally, like parity-style problems. Very long contexts can also reintroduce noise after the smoothing effect saturates.

So the paper’s real lesson is narrower and more useful than a generic “longer is better” slogan. A positional method extrapolates when it gives the model a stable distance pattern that training has already covered or can safely generalize. Sandwich is a neat example of that principle.
