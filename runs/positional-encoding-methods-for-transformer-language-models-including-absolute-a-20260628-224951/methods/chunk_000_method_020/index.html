Megalodon: Efficient Long-Context LLM Pretraining and Inference with Unlimited Context Length

Megalodon is easiest to understand as a long-context hybrid block rather than a brand-new attention replacement. It keeps chunk-wise attention for local token mixing, but adds a richer recurrent memory path so information can keep flowing after the chunk boundary. The rest of the method is about making that hybrid stack stable enough to train at 7B scale without losing the long-range signal.

Mega gave this architecture its skeleton, but Megalodon tightens the memory, normalization, and residual routing so chunked attention can stay useful when the sequence gets much longer. The positional story is still RoPE; the novelty is in how the memory and normalization work around that backbone.

CEMA gives the model a memory lane

Megalodon starts from Mega’s damped EMA and asks a simple question: what if the memory could do more than fade?

A plain exponential moving average is good at carrying a slowly changing summary forward in time. It is not as good at representing richer temporal structure, because it can only decay. CEMA pushes the memory into the complex plane, so each feature can both decay and rotate. That rotation is the useful part: it lets the hidden state keep track of phase-like patterns instead of collapsing everything into a single real-valued trend.

Formula card: CEMA recurrence

Purpose: carry contextual information across time with a memory state that can decay and rotate.

Equation:

ht(j)=αj(cos⁡θj+isin⁡θj)⊙ut(j)+(1−αj⊙δj)(cos⁡θj+isin⁡θj)⊙ht−1(j),yt,j=Re(ηjTht(j))h_t^{(j)}=\alpha_j(\cos\theta_j+i\sin\theta_j)\odot u_t^{(j)}+(1-\alpha_j\odot\delta_j)(\cos\theta_j+i\sin\theta_j)\odot h_{t-1}^{(j)},\quad y_{t,j}=\mathrm{Re}(\eta_j^T h_t^{(j)})ht(j)​=αj​(cosθj​+isinθj​)⊙ut(j)​+(1−αj​⊙δj​)(cosθj​+isinθj​)⊙ht−1(j)​,yt,j​=Re(ηjT​ht(j)​)

Intuition: each feature keeps a decaying memory, but the memory is also rotated in the complex plane before it is read out. That gives the state more expressive temporal behavior than a purely real moving average.

Symbols:

  • h_t^{(j)}: memory state for feature j at timestep t
  • u_t^{(j)}: expanded input for feature j at timestep t
  • alpha_j: factor that controls how much new input enters the state
  • delta_j: damping factor on the carried state
  • theta_j: complex phase angle for feature j
  • eta_j: complex projection back to the output space
  • y_{t,j}: output feature read from the memory state
  • Re: real-part operator
  • i: imaginary unit
  • odot: elementwise multiplication

Read-aloud meaning: “Mix the current input with a rotated and damped version of the previous memory, then read out the real part.”

Why it matters: this is the path that keeps long-range information alive when attention is only computed inside chunks. Without a stronger memory lane, chunk boundaries would hide dependencies instead of carrying them forward.

The paper also spaces the complex phases across the hidden width so the memory units do not all learn the same temporal pattern. That small detail is what makes CEMA behave like a richer state update instead of just a decorated EMA.

Normalization has to stay causal

Once the memory path becomes stronger, the next problem is stability. Normalization is supposed to make optimization easier, but a standard per-token LayerNorm only looks inside a single token’s feature vector. It does nothing to control the statistics that evolve over time.

That is why Megalodon introduces Timestep Normalization. It borrows the spirit of GroupNorm, but makes the statistic computation causal: when the model is at timestep t, it may use the current token and all previous tokens, but never future ones. Figure 2 makes that visible.

What to notice in the figure is the region being averaged. The normalization window grows only along the past side of the sequence. That is exactly what autoregressive modeling needs. If the normalization looked ahead, it would leak future information into the current token representation.

Formula card: causal timestep normalization

Purpose: normalize sequence features over time without breaking causality.

Equation:

μt=1t dg∑i=1t∑j=1dgxi,j,σt2=1t dg∑i=1t∑j=1dg(xi,j−μt)2\mu_t=\frac{1}{t\,d_g}\sum_{i=1}^{t}\sum_{j=1}^{d_g}x_{i,j},\qquad \sigma_t^2=\frac{1}{t\,d_g}\sum_{i=1}^{t}\sum_{j=1}^{d_g}(x_{i,j}-\mu_t)^2μt​=tdg​1​i=1∑t​j=1∑dg​​xi,j​,σt2​=tdg​1​i=1∑t​j=1∑dg​​(xi,j​−μt​)2

Intuition: compute mean and variance from the current token and all earlier tokens in a feature group, then use those running statistics to normalize the present.

Symbols:

  • x_{i,j}: scalar feature j of token i inside one group
  • t: current timestep
  • d_g: number of features in one group
  • mu_t: cumulative mean up to timestep t
  • sigma_t^2: cumulative variance up to timestep t

Read-aloud meaning: “Average the past and present tokens in a feature group, and use that running average and variance to normalize the current step.”

Why it matters: this is the sequence-aware analogue of GroupNorm. It keeps the model from leaking future statistics, while still giving the memory-and-attention stack a stable scale along the time axis.

The paper only uses TimestepNorm before attention. Before the feed-forward layer it switches back to LayerNorm, partly because LayerNorm is cheaper and partly because attention has already mixed information across tokens, so feature-wise normalization is enough.

Attention only works if the score scale is under control

A stronger memory lane helps, but it can also make the attention block unstable if the query and key activations become too large or too irregular. Megalodon answers that by normalizing the shared representation before it becomes queries and keys.

Think of this as a calibration step. The memory path produces a contextual stream. That stream is projected into a shared space, normalized, and then split into query and key vectors with learned scale and shift parameters. Once those vectors are normalized, the dot products are much less likely to saturate or explode, so softmax can do its job more reliably. The paper also drops the old SiLU nonlinearity in this path, because normalization has already introduced enough nonlinearity for the score computation.

Formula card: normalized attention

Purpose: make the attention score path more stable by normalizing the contextual representation before queries and keys are formed.

Equation:

X′=CEMA(X)X'=\mathrm{CEMA}(X)X′=CEMA(X) Z=X′Wz+bz,Z′=Z∥Z∥Z=X'W_z+b_z,\qquad Z'=\frac{Z}{\|Z\|}Z=X′Wz​+bz​,Z′=∥Z∥Z​ Q=κq⊙Z′+μq,K=κk⊙Z′+μkQ=\kappa_q\odot Z'+\mu_q,\qquad K=\kappa_k\odot Z'+\mu_kQ=κq​⊙Z′+μq​,K=κk​⊙Z′+μk​ O=softmax(QKT)VO=\mathrm{softmax}(QK^T)VO=softmax(QKT)V

Intuition: first compress the contextual memory into a shared representation, then normalize it so query and key magnitudes stay controlled, then let softmax mix the values.

Symbols:

  • X: input token sequence
  • X': contextual sequence after CEMA
  • W_z, b_z: projection parameters for the shared representation
  • Z: projected shared representation
  • Z': normalized shared representation
  • Q: query matrix
  • K: key matrix
  • V: value matrix carrying the token content stream
  • kappa_q, kappa_k: learnable scale vectors for queries and keys
  • mu_q, mu_k: learnable shift vectors for queries and keys
  • O: attention output
  • ||Z||: norm used to normalize each token representation

Read-aloud meaning: “Turn the memory stream into normalized queries and keys, then use softmax attention to combine the values.”

Why it matters: the method is not trying to replace attention. It is trying to make attention behave well after the long-memory path has already done part of the work. Normalized scores are what keep chunked attention usable at scale.

The paper also reports that once this normalized path is in place, softmax is the safest default attention function across the tasks and modalities they checked.

The block stops using the old residual trick

The last stability fix is easy to overlook, but it matters a lot when the model gets deep. Standard pre-norm Transformer blocks keep adding the input and the sublayer outputs on top of each other. Over many layers, that can make the residual stream grow in range and variance. Mega used an update gate to soften that problem, but Megalodon removes the gate and instead rearranges the residual path.

What to notice here is the routing. The attention branch sees the normalized input and returns a first intermediate state. The feed-forward branch then sees a normalized version of that intermediate state, but the original input X is reused as the residual connection again. That is why the paper calls it a two-hop residual: the same source signal takes two controlled hops through the block instead of one long residual chain that keeps accumulating baggage.

Formula card: two-hop residual block

Purpose: keep deep training stable by reusing the original block input as the residual for both sublayers.

Equation:

Y^=Attention(Norm(X))+X,Y=FFN(Norm(Y^))+X\hat{Y}=\mathrm{Attention}(\mathrm{Norm}(X))+X,\qquad Y=\mathrm{FFN}(\mathrm{Norm}(\hat{Y}))+XY^=Attention(Norm(X))+X,Y=FFN(Norm(Y^))+X

Intuition: let attention refine the input once, then let the feed-forward layer refine the original signal again after normalization, instead of chaining residual growth through the whole block.

Symbols:

  • X: block input
  • \hat{Y}: intermediate output after the attention sublayer
  • Y: final block output
  • Attention: chunk-wise attention with normalized queries and keys
  • FFN: feed-forward network
  • Norm: normalization layer, using TimestepNorm before attention and LayerNorm before the FFN

Read-aloud meaning: “Apply normalized attention to the input, then apply a normalized feed-forward layer, while keeping the original input as the residual path.”

Why it matters: this is the part that makes the hybrid block train like a real large model instead of a fragile recurrence with attention attached. It keeps residual growth under control and removes the need for the extra update gate.

There is a second practical reason for the split. Megalodon only uses TimestepNorm before attention; the FFN still uses LayerNorm. After attention, each token already contains a mixture of information from other tokens, so feature-wise normalization is enough, and it is cheaper than running another sequence-aware normalization pass.

Why this scales better than a plain long Transformer

Once the block is stable, chunking becomes a useful engineering choice instead of a compromise. The model can process long sequences in fixed-size chunks, keep a persistent CEMA state across chunk boundaries, and only communicate a small amount of information between parallel workers.

That is where the paper’s extra parallelism comes from. In addition to data, tensor, and pipeline parallelism, Megalodon can parallelize along the sequence-chunk axis. The only cross-device state that really needs to move is the last CEMA hidden state and the cumulative statistics for TimestepNorm. Everything else can stay local to the chunk.

With asynchronous communication, that extra axis can hide most of its overhead behind local computation. In other words, the model does not have to pay full quadratic attention over the entire sequence; it spends local attention where the detail matters and uses CEMA to preserve long-range summaries cheaply.

What to remember

Megalodon is a good example of a general long-context lesson: long-range capability does not come from one clever positional trick alone. It comes from making the whole sequence mixer stable enough that a compact memory path can actually help.

The four pieces to keep in mind are:

  • CEMA gives the model a memory lane that can decay and rotate.
  • TimestepNorm makes sequence normalization causal.
  • Normalized attention keeps query-key scores in a safe range.
  • Two-hop residuals stop deep residual growth from taking over.

The tradeoff is real. The paper reports a small short-context slowdown because the model pays for CEMA and extra normalization, and it is still using RoPE rather than replacing positional handling entirely. But when the context gets long, the architecture has a much better chance of keeping signal alive without paying full attention cost.
