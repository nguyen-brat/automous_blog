HoPE: Hyperbolic Rotary Positional Encoding for Stable Long-Range Dependency Modeling in Large Language Models

HoPE is what happens when you stop asking a rotary positional encoding to behave like a circle. RoPE is elegant, but its distance signal is periodic, so far-away tokens can come back around with a familiar phase. HoPE changes the geometry. It replaces the circular part with a hyperbolic 2D transform and then adds exponential damping so distance fades in a more predictable way.

That is the key idea to keep in mind. HoPE does not rewrite the Transformer. It changes the positional shape that enters the query-key dot product, so the model still gets relative position information, but the distance response becomes smooth instead of oscillatory.

The attention curve the paper is trying to change

The Attention score trend comparison is the easiest way to see why the paper exists.

What to notice:

  • RoPE does not fall cleanly as distance grows. It ripples.
  • That ripple changes with query position and head size, which is a sign that the problem is structural, not a one-off artifact.
  • HoPE is trying to keep relative-position structure while making the decay feel monotone and less noisy.

The paper’s complaint is not that RoPE is useless. It is that RoPE can make distance behave like a wave, and waves are the wrong shape if you want far tokens to fade away consistently.

The hyperbolic block at the center

HoPE still works the RoPE way at the pipeline level: token embeddings are projected into query and key vectors first, and only then does the positional transform act on them. The difference is the transform itself. Instead of sine and cosine, HoPE uses hyperbolic sine and cosine, borrowed from the shape of a Lorentz boost.

You do not need the special-relativity story to use the method. The useful part is simpler: hyperbolic mixing is not circular mixing. A circular rotation can wrap back around. A hyperbolic transform pushes the representation in a way that can support decay instead of repetition.

Formula card: Hyperbolic rotation matrix

Purpose: define the core 2D positional transform used by HoPE.

Equation:

B(θ,m)=(cosh⁡(mθ)sinh⁡(mθ)sinh⁡(mθ)cosh⁡(mθ))B(\theta,m)= \begin{pmatrix} \cosh(m\theta) & \sinh(m\theta) \\ \sinh(m\theta) & \cosh(m\theta) \end{pmatrix}B(θ,m)=(cosh(mθ)sinh(mθ)​sinh(mθ)cosh(mθ)​)

Intuition: each 2D query or key slice is mixed by a position-dependent hyperbolic block, so position changes the geometry without using periodic sine-wave rotation.

Symbols:

  • B(\theta,m): the position-dependent hyperbolic transform for one 2D feature pair.
  • \theta: the per-block scale that controls how strongly position changes the transform.
  • m: the token position index.
  • \cosh, \sinh: hyperbolic cosine and sine.

Read-aloud meaning: position m applies a hyperbolic mix to each 2D feature pair.

Why it matters: this is the paper’s replacement for RoPE’s circular rotation block, and it is the geometric move that makes monotone decay possible.

Because the full embedding is split into 2D pairs, this same block is stacked across the hidden state rather than applied as one giant matrix. That keeps the implementation close to RoPE: same Transformer plumbing, different positional geometry.

The sign choice also matters. HoPE uses opposite-sign transforms for queries and keys so the relative offset survives in the dot product instead of getting washed out by two unrelated absolute transforms.

Why damping has to come with it

Hyperbolic mixing on its own goes the wrong way at long distances. The hyperbolic functions grow, and growth is the opposite of the locality prior we want in attention. The paper therefore adds exponential damping around the query and key transforms.

Formula card: Damped query-key encoding

Purpose: combine the hyperbolic positional transform with an explicit distance penalty so far tokens do not dominate the score.

Equation:

fq(xm,m)=e−mθ′B(θ,m)Wprojqxm,fk(xn,n)=enθ′B′(θ,n)Wprojkxnf_q(x_m,m)=e^{-m\theta'}B(\theta,m)W_{\mathrm{proj}}^q x_m,\quad f_k(x_n,n)=e^{n\theta'}B'(\theta,n)W_{\mathrm{proj}}^k x_nfq​(xm​,m)=e−mθ′B(θ,m)Wprojq​xm​,fk​(xn​,n)=enθ′B′(θ,n)Wprojk​xn​

Intuition: the query and key still go through learned projections, but the exponential factors keep the hyperbolic growth from turning into runaway long-range scores.

Symbols:

  • f_q(x_m,m): the position-aware query vector at token position m.
  • f_k(x_n,n): the position-aware key vector at token position n.
  • x_m, x_n: token embeddings before positional encoding.
  • W_{\mathrm{proj}}^q, W_{\mathrm{proj}}^k: learned query and key projection matrices.
  • B(\theta,m): the hyperbolic block used on the query side.
  • B'(\theta,n): the opposite-sign hyperbolic block used on the key side.
  • \theta': the damping coefficient.

Read-aloud meaning: project the token into query and key space, apply hyperbolic position mixing, and then rescale both sides so distance contributes a controlled decay.

Why it matters: without the damping term, the hyperbolic transform would grow with distance instead of fading, which would break the locality behavior attention is supposed to learn.

The paper makes that point explicit with an asymptotic decay argument.

Formula card: Long-range decay guarantee

Purpose: show why HoPE can still decay with distance even though the underlying transform is hyperbolic.

Equation:

lim⁡∣m−n∣→∞e−∣m−n∣θ′cosh⁡(∣m−n∣θi)∝e−∣m−n∣(θ′−θi)\lim_{|m-n|\to\infty}e^{-|m-n|\theta'}\cosh(|m-n|\theta_i)\propto e^{-|m-n|(\theta'-\theta_i)}∣m−n∣→∞lim​e−∣m−n∣θ′cosh(∣m−n∣θi​)∝e−∣m−n∣(θ′−θi​)

Intuition: hyperbolic growth is exponential, so a stronger exponential damping term can overpower it and turn the long-range score into a monotone decay.

Symbols:

  • m, n: token positions.
  • |m-n|: token distance.
  • \theta_i: the scale for the i-th 2D block.
  • \theta': the global damping coefficient.

Read-aloud meaning: at large distances, the damped hyperbolic score falls roughly like an exponential whenever the damping coefficient is larger than the block scale.

Why it matters: this is the mathematical reason HoPE can preserve relative position while still preferring nearby tokens.

The appendix also argues that this does not destroy positional discrimination. In plain language, HoPE is not just a forget gate. It still lets the model distinguish relative offsets; it just makes the distance response behave more sensibly.

What the curves look like once the fix is in

The paper’s method is easier to trust once you look at the attention curves side by side.

What to notice:

  • RoPE still has ripples, especially once distance grows.
  • Alibi is monotone, but it looks like a hand-crafted bias rather than a geometric encoding.
  • HoPE is the middle ground: it falls smoothly without the same oscillatory noise.

The head-size panels matter too. The same qualitative story shows up in both narrow and wider heads, so the effect is not just a quirk of one feature width.

That is the core design win. HoPE is not trying to make attention flat. It is trying to make distance behave in a way the model can use consistently.

The damping knob is real

HoPE is not parameter-free, and the paper is honest about that. The scaling or damping coefficient needs care. Too little damping and the hyperbolic growth leaks through. Too much damping and the positional signal gets flattened before it can help.

The ablation experiment makes that tradeoff visible.

What to notice:

  • Tiny damping values let perplexity blow up fast as length grows.
  • Huge damping values also do badly because the signal becomes too distorted or too weak.
  • Moderate settings are the sweet spot.

That is why the paper lists coefficient tuning as one of the method’s practical limits. HoPE works because the damping is strong enough to control the hyperbolic mix, but not so strong that it erases the position signal entirely.

Where it helps on real long-context tasks

The SCROLLS downstream results are the practical test. The setup is a train-short, test-long regime: pretrain at 1024 tokens, probe much longer contexts on PG19 and arXiv, then fine-tune at 8192 on SCROLLS.

The anchor results line up with the theory. At length 4096, HoPE lowers PG19 perplexity from 88.59 with RoPE to 60.03, and arXiv perplexity from 62.78 to 40.10. On SCROLLS, HoPE leads 4 of 7 tasks, and on NarrativeQA it improves Rouge-L by 1.57 over RoPE. The same kind of gain also survives the BiPE-style retrofit experiments, where BiPE-HoPE stays ahead of BiPE-RoPE at longer lengths.

There is an important caveat, though. HoPE is not a universal winner on every corpus. Alibi can still post lower perplexity on arXiv, which makes sense if that dataset rewards a very strong local-bias prior. So the method is better thought of as a better geometric default for long-range structure, not as a strict win on every text distribution.

What HoPE changes conceptually

The big lesson is not “hyperbolic math is cool.” The lesson is that the distance law should live inside the attention kernel itself.

RoPE makes distance periodic, which can let far positions feel familiar again. Alibi makes distance decay by fiat, which is simpler but less geometric. HoPE changes the kernel so the score naturally falls with distance while still preserving relative position structure. That is why it belongs in the generalized rotary and phase family: it keeps the rotary spirit, but it changes the geometry underneath it.

The open questions are natural ones. How does the same idea behave in much larger modern LLMs? Should each head or layer learn its own damping schedule? Does the hyperbolic design help when the model has to fuse text with audio or vision? The paper does not answer those yet, so the method’s scope is still text-only.

What to remember

HoPE is easiest to remember as “rotary encoding, but with a geometry that actually prefers decay.”

The useful mental model is simple:

  • RoPE wraps distance around a circle.
  • HoPE replaces that circle with a hyperbolic block.
  • Damping turns the block into a monotone distance response instead of an expanding one.

So the method’s real contribution is not a new Transformer skeleton. It is a new shape for the positional kernel, one that makes long-range attention smoother and less noisy without throwing away relative position information.
