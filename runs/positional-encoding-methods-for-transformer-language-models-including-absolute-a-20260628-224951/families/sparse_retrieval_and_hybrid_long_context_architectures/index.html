How Long-Context Language Models Avoid Full Attention

The hard part of long-context modeling is not merely “making the window bigger.” It is deciding which past information deserves expensive computation, which information can be compressed, and which position signals still matter after the sequence becomes too large for dense attention. That is why the common theme here is not a single algorithm. It is a shared interface problem between retrieval, memory, sparsity, and positional structure.

Once context becomes large, full attention starts charging quadratic cost in both compute and memory. Even if hardware can tolerate that for short prompts, decoding latency, KV-cache growth, and long-range retrieval quality become practical bottlenecks. The methods in this group attack different parts of that failure: some search for a sparse mask before attention is computed, some replace global pairwise interaction with a memory path, some retrieve chunks or blocks on demand, and some train the model so the surviving context carries forward enough history.

That compact pipeline hides several different choices. A model may score individual tokens, chunks, blocks, or special routing tokens. It may construct a dynamic sparse mask, a local-plus-global pattern, or a retrieval shortlist. Then it spends full attention only where the earlier stage says attention is worth paying for.

In practice, that pipeline often sits beside a second path that carries history more cheaply. Megalodon uses a recurrent complex EMA memory to move information across chunks. Hybrid Attention Transformer with Gated FIFO Memory keeps a fixed-size memory path beside local and global attention. MiniCPM-SALA alternates sparse and linear layers so only some layers pay for precise retrieval.

Why the problem is hard

Quadratic cost is only the first failure mode. The deeper issue is that long context forces a trade between reach and precision.

If attention stays dense, inference becomes slow and memory-heavy. If attention becomes too local, the model forgets distant evidence. If retrieval becomes too selective, it misses middle-context facts or cross-chunk reasoning. If memory becomes too compressed, the model preserves gist but loses exact token-level detail. And if positional encoding is stretched naively, the model may technically see a longer sequence while still behaving as if distant tokens are out of distribution.

This is why the family should not be reduced to “sparse attention.” Some papers mainly fight quadratic attention cost. Others fight quadratic context scaling during training. Others focus on long-range retention, long-context retrieval efficiency, or attention dispersion. The bottlenecks are related, but not identical.

The main design axes

The design axes in the input show where these architectures actually differ:

  • hardware_strategy: block approximation
  • mask_type: dynamic sparse
  • refresh_policy: cached periodic mask refresh
  • local_mixing: chunk-wise softmax attention
  • memory_path: complex EMA
  • normalization_scope: causal timestep plus feature norm

The first axis asks whether sparsity is only theoretical or can be mapped onto hardware-friendly blocks. The second and third ask whether the mask is fixed or query-dependent, and whether it is recomputed every step or refreshed from cached structure. The fourth, fifth, and sixth axes belong to hybrid models: if global dense attention is removed, what local mixer remains, what memory path substitutes for long-range aggregation, and what normalization keeps the resulting system stable?

Other methods in the group expose parallel axes even when they are not named in the synthesis block. Extended Transformer Construction uses routing tokens and relative labels. LongHeads retrieves chunks head by head. Landmark Attention turns memory blocks into retrievable units through landmark tokens. Adaptive-Scalable alpha-entmax changes the attention normalizer itself. SWAT changes training so window-limited inference has a better memory pathway later.

Selecting before paying

One of the clearest ideas in this area is that the model should search before it aggregates. HiP Attention is a direct example: it estimates which keys matter for a query through hierarchical pruning, then performs sparse attention only on the surviving positions.

Formula card: Top-k sparse attention mask

Purpose: turn dense attention into a masked sparse computation.

Intuition: first estimate which query-key matches are worth keeping, then normalize only over that shortlist instead of over the full context.

Symbols: Q\mathbf{Q}Q is the query matrix, K\mathbf{K}K is the key matrix, V\mathbf{V}V is the value matrix, M\mathbf{M}M is the binary mask that keeps the top-scoring positions, and O^\widehat{\mathbf{O}}O is the sparse attention output.

Read-aloud meaning: compute attention scores, keep only the strongest positions, and aggregate values through that sparse pattern.

Why it matters: this is the core move behind training-free sparse serving methods. The model does not need to abandon pretrained attention behavior; it only needs a reliable way to prune low-value interactions before paying dense cost.

The same basic lesson appears at larger units than tokens. LongHeads retrieves chunks instead of individual keys, while Landmark Attention retrieves blocks through learned landmark gates. In both cases, the point is similar: do a cheap search over summaries or routing surrogates, then reopen only a small part of the past for exact computation.

When sparse access is not enough

Sparse retrieval solves only part of long-context scaling. Some tasks need the model to carry forward information continuously, even when no explicit retrieval event happens. That is where hybrid memory paths become useful.

Megalodon illustrates the idea well. It keeps chunk-wise softmax attention for local mixing, but adds a recurrent memory path so information can propagate beyond the chunk boundary without reconstructing full pairwise attention. Its central claim is not “replace attention entirely.” It is “stabilize cheaper sequence mixing well enough that long context remains trainable.”

Formula card: CEMA recurrence

Purpose: carry information across time through a richer decaying memory than a standard moving average.

Intuition: the memory state both fades and rotates, so it can represent long-range patterns that a purely real decay would blur away.

Symbols: ht(j)h_t^{(j)}ht(j)​ is the memory state for feature jjj at time ttt, αj\alpha_jαj​ controls how much new input enters, δj\delta_jδj​ is a damping factor on the carried memory, θj\theta_jθj​ sets the complex rotation angle, ut(j)u_t^{(j)}ut(j)​ is the current input feature, ηj\eta_jηj​ projects the state back to output space, and yt,jy_{t,j}yt,j​ is the resulting contextual feature.

Read-aloud meaning: update each feature’s memory by mixing the current input with a rotated, decayed version of the previous state, then read out a real-valued contextual signal.

Why it matters: this is the memory path that lets Megalodon preserve long-range structure while keeping local attention bounded.

The same architectural instinct appears in Hybrid Attention Transformer with Gated FIFO Memory: local detail, global access, and persistent memory are treated as different jobs. Instead of forcing one attention pattern to do all three, the model keeps separate paths and learns how much to trust each.

Purpose: blend multiple context pathways instead of committing to one attention style.

Why it matters: hybrid long-context models work when they separate roles cleanly rather than pretending one pathway can be simultaneously cheap, precise, and persistent.

Retrieval depends on positional treatment too

Long context is not only a routing problem. It is also a positional calibration problem.

mGTE long-context multilingual retrieval pipeline makes this visible from the encoder side: it pairs RoPE-based long-context scaling with a staged pretraining schedule, then uses hybrid dense-sparse retrieval and reranking on top. The lesson is that longer context often requires both positional redesign and a training curriculum.

MiniCPM-SALA pushes that idea further by applying positional treatment selectively. Its hybrid sparse-linear stack keeps RoPE in linear layers but removes RoPE from sparse layers. That is a useful reminder that not every attention module benefits from the same positional bias once context becomes extreme.

Extended Transformer Construction reaches a similar destination from another direction. It uses a two-stream setup where most tokens stay local while a small set of global tokens route information across distant regions, and it relies on relative labels rather than absolute positions. The design says: long-range communication is easier when the model gets explicit routing structure instead of trying to recover it from dense pairwise scores alone.

Training the memory pathway

Some long-context methods aim less at inference-time retrieval and more at representation discipline during training. SWAT is a clean example. If inference will use a sliding window, then training should force token states to relay history before older tokens disappear. That is why SWAT changes the attention normalizer and positional bias during training rather than only swapping kernels at deployment.

Adaptive-Scalable alpha-entmax (ASEntmax) targets a related issue from a different angle: as context grows, softmax can spread mass too broadly and long-range signals become diffuse. Sparse attention helps, but fixed sparsity is brittle. ASEntmax therefore adapts sharpness with sequence length so the model stays selective without becoming overly narrow when many tokens matter.

The shared takeaway is important: efficient long-context behavior is rarely achieved by inference tricks alone. If the eventual system will throw away old tokens, compress history, or rely on sparse retrieval, training often has to prepare representations for that regime.

What the evidence supports

The compact input gives a few strong anchor points and only limited detail for some newer hybrids, so the evidence should be read as illustrative rather than exhaustive.

  • HiP Attention reports strong efficiency gains, including about 3.01x prompt speedup and 36.60x decoding-latency speedup on LLaMA2-13B with only a small perplexity increase.
  • On LLaMA2-7B MMLU, HiP at k=1024 is reported at 43.08%, slightly above vanilla’s 42.76%, which matters because it suggests sparse selection need not automatically hurt quality.
  • In vLLM decoding, HiP is reported to reach up to 1.94x lower per-batch latency and up to 3.30x higher throughput at larger batch sizes.
  • Megalodon reports lower training loss than Llama2-7B at 2T training tokens and a monotonic perplexity improvement as evaluation context grows from 4K to 2M, which supports the claim that the recurrent memory path scales with context instead of degrading immediately.

For methods such as the gated-memory hybrid and some retrieval pipelines, the provided input gives fewer concrete numbers. That is enough to support the conceptual role they play here, but not enough to make this chapter a benchmark-driven comparison.

What to remember

Long-context architecture is not one idea. It is a set of decisions about when to search, what unit to retrieve, what history to compress, and where positional information still helps. Some systems preserve global reach by pruning token interactions. Some retrieve chunks or blocks through summaries. Some carry forward a persistent memory state. Some train the model so local windows can still preserve old information. Some split positional treatment across layer types instead of assuming one encoding should govern everything.

The durable lesson is simple: once full attention stops being affordable, the model must become deliberate about which past information stays exact, which becomes summarized, and which is ignored.
