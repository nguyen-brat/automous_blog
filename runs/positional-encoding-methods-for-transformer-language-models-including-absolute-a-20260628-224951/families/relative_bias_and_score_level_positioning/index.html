How Relative Attention Bias Shapes Long-Context Transformers

Long-context transformers often fail for a simple reason: attention scores know which tokens match in content, but they do not automatically know how distance should matter in a particular situation. If the model always discounts faraway tokens the same way, it misses useful long-range evidence. If it treats every faraway token as equally reachable, attention becomes noisy, expensive, or both.

This subtopic is about fixing that interface. The common theme is not a single algorithm. It is a shared interface problem inside attention: how to inject position information directly into pre-softmax scores so the model can stay order-aware, remain stable beyond training length, and still retrieve distant information when it matters.

Where position enters the attention pipeline

These methods operate in the same narrow region of the transformer:

Instead of attaching position only as an embedding on token inputs, they modify the score matrix that decides who attends to whom. That move matters because the attention logits are where the model decides whether a distant token should be softly discouraged, strongly ignored, or explicitly pulled back into view.

At a high level, score-level positioning asks four questions:

  1. What distance pattern should attention prefer by default?
  2. Should that pattern be fixed, learned as a curve, or adapted from the current content?
  3. Can the bias remain useful beyond the training window?
  4. Can the same positional effect be executed efficiently at long lengths?

Why the problem is hard

A fixed long-range bias is attractive because it is simple. Methods like Attention with Linear Biases show that even a hand-designed distance penalty can make short-trained models run on longer contexts. But fixed bias also hard-codes one answer to the question “how much should distance matter?” That is often too rigid.

Several bottlenecks show up repeatedly:

  • Static long-range bias: the model keeps the same distance preference even when the content strongly suggests that a far token is important.
  • Quadratic long-context attention: even a good bias still lives inside dense all-pairs attention unless it can be approximated or re-executed more cheaply.
  • Long-context generalization: a bias that works inside the training window can break once attention reaches token pairs the model never had to use during training.
  • Long-range retrieval failure: perplexity can stay reasonable while the model still fails at the concrete act of finding a distant needle in context.

Those are related, but not identical. A method can help extrapolation without reducing cost. Another can reduce cost without improving retrieval. That is why this area is best understood through design axes rather than a single family recipe.

The main design axes

The embedded family synthesis points to a few recurring choices.

Fixed curves versus adaptive corrections

One option is to define a distance-shaped bias once and reuse it everywhere. KERPLE does this with simple kernel-shaped curves over relative distance, while Attention with Linear Biases uses head-specific linear penalties. These methods make a strong prior explicit: nearer tokens usually deserve more weight.

Another option is to let the bias react to the current sequence. CAPE and CABLE push in that direction. They keep the bias additive, but stop treating distance as a fixed ruler.

Which base bias to start from

Some methods build on a known bias source such as ALiBi, KERPLE, or FIRE, then add adaptation on top. That choice matters because the base curve defines the starting locality pattern before any learned correction appears.

What signal should control the bias

A score-level positional method needs to decide what it trusts. CAPE conditions on query-key similarity, which means the model can adjust positional pressure based on what the tokens are already saying semantically. CABLE conditions on token hidden states and turns them into a cumulative bias signal, so different tokens can stretch or relax the effective distance scale.

Head coupling and granularity

Some approaches keep each head on a simple independent schedule. Others process heads jointly, which can make the bias richer but also increases compute and memory cost.

Exact execution versus structured approximation

A relative bias may encode a useful inductive bias but still inherit quadratic attention cost. Positional LSH is important here because it reframes the bias itself as something that can be approximated by sampled local masks.

When a fixed distance prior is enough

The foundational lesson from Attention with Linear Biases is that position information does not have to live in token embeddings. A head-specific linear penalty directly on attention scores can already preserve causal order and encourage useful recency bias.

That is conceptually important because it relocates position from “token decoration” to “routing pressure.” The model no longer asks “what absolute slot is this token in?” It asks “given this pair of positions, how much should distance change the score?”

KERPLE extends the same idea by replacing a hard-coded line with a learnable distance curve. Its key lesson is that extrapolation depends heavily on the shape of decay. If the bias fades too aggressively, distant evidence disappears. If it fades too weakly, attention becomes diffuse. The practical attraction is that a simple monotonic log-shaped kernel often lands in a useful middle ground.

The deeper point is that these methods treat relative position as an inductive bias over score formation, not as a learned table of absolute slots. That tends to age better when context length grows.

Why extrapolation is really a receptive-field problem

A Length-Extrapolatable Transformer adds an important perspective: long-context failure is not only about whether a bias is relative or absolute. It is also about the empirical receptive field the model actually learns to use.

If training only covers a short useful range, then very distant tokens at test time arrive as effectively untrained interactions. A stable long-context method therefore needs a bias that either keeps those distant interactions safely weak or introduces them through a smooth pattern the model can generalize from.

This is why recency bias remains so common even in successful long-context methods. It is not just conservatism. It is a way of controlling which new interactions appear when the window grows.

The same lens helps explain why Improve Transformer Models with Better Relative Position Embeddings matters. Its stronger relative score terms show that order modeling improves when distance directly modulates token-pair interactions rather than merely tagging tokens with standalone position vectors.

When fixed bias becomes too rigid

Static curves solve one problem and create another. Sometimes far tokens should stay suppressed; sometimes they are the entire point of the task. That is where adaptive methods become useful.

CAPE starts from a base relative bias, then learns how to correct or replace it using the current attention similarity pattern. The intuition is simple: if two tokens already look semantically relevant, the model should be allowed to weaken a default distance penalty. If they do not, the same penalty can remain strong.

CABLE reaches a similar destination by a different route. Instead of modifying a base bias with an MLP, it predicts token-conditioned cumulative position signals and uses their differences as pairwise attention offsets. That makes the effective slope depend on the input itself.

Making long-context bias cheaper to execute

Even a good score-level bias can be trapped inside quadratic attention. Positional LSH tackles that bottleneck by asking a different question: can a relative bias be reinterpreted as an average over simpler local computations?

Its answer is conceptually elegant. Instead of applying one dense distance-biased attention matrix, it samples several contiguous local masks whose average approximates the same positional preference. The point is not merely approximation for its own sake. It reveals that some biases already encode a latent local structure.

This is useful because it separates two ideas that are often mixed together:

  • the bias pattern you want,
  • the computation pattern you can afford.

If a positional prior can be decomposed into sampled local routing patterns, then efficient execution becomes a property of the bias representation, not only of the attention kernel.

Position as a prior over where attention should look

Bayesian Attention Mechanism highlights another useful viewpoint: positional encoding can be treated as a prior over relative positions rather than a formatting trick on token vectors.

That framing matters because retrieval tasks expose a weakness in many long-context claims. A model may preserve decent language modeling quality while still failing to recover a distant key fact. A prior-based view makes that failure legible: the model’s positional prior may simply place too little mass on the relevant far region.

This is also why retrieval-oriented heads can look different from locally optimized language-model heads. A bias that is ideal for next-token fluency is not automatically ideal for passkey-style search over very distant context.

What evidence actually supports these ideas

The compact input gives only selective evidence, but the pattern is still clear.

  • CAPE improves over its static base biases on Arxiv, Books3, and CHE settings, and a highlighted result shows CAPE-Kerple reaching 5.00 perplexity at evaluation length 8192 versus 31.93 for Kerple when training length is 128.
  • CABLE is reported to beat ALiBi far beyond training length, including 15.41 perplexity versus 17.46 on a GPT-2 Medium setting at length 15360.
  • BAM is reported to maintain perfect passkey retrieval up to 32k tokens in its cited setup, which supports the claim that score-level positional priors affect retrieval behavior, not only perplexity.
  • Positional LSH reports shrinking approximation error as more masks are averaged, supporting the idea that a distance bias can be executed through sampled local structure.

The evidence should be read cautiously. These numbers do not prove one universal winner. They do show that score-level positional design changes concrete long-context behavior in predictable ways.

Open questions that still matter

The hardest unresolved issue is how to balance three goals at once: faithful long-range retrieval, stable local language modeling, and affordable execution. Methods in this family usually get two of the three more easily than all three.

Other open questions follow naturally:

  • How adaptive should positional bias become before its compute overhead cancels the benefit?
  • Which bias shapes transfer best from pretraining to instruction-tuned models?
  • When does a retrieval-friendly prior hurt ordinary local modeling?
  • Which structured biases admit efficient approximations like sampled masks without losing the original inductive bias?

What to remember

Relative bias and score-level positioning methods succeed when they treat position as a decision rule inside attention, not as a static label attached to tokens. The most durable lesson is that long-context performance depends on how the bias shapes access: what stays local, what can surface from far away, and which new interactions appear safely when the context window grows.

Some methods encode that rule as a fixed curve. Some learn the curve. Some adapt it from content. Some reinterpret it as efficient local computation. But they are all trying to solve the same interface problem: how to make attention distance-aware without making it distance-trapped.
