How Transformers Encode Position Without Rotary Embeddings

Position handling looks like a detail until a Transformer leaves its comfort zone. A model trained on one context length may stumble on longer sequences, an efficient attention variant may lose positional expressivity, and a cached long-context decoder may accidentally store position in places that cannot be safely reused. The common theme here is not a single algorithm. It is a shared interface problem: how positional information should enter the model so order is preserved without breaking extrapolation, efficiency, or cacheability.

This family shows three different answers to that interface problem. One answer rewrites positions into structured sequences that can generalize beyond the seen range. Another injects position directly into attention scores instead of mixing it into token embeddings. A third keeps absolute positions but moves them into the query-key interaction so cached states stay reusable. Together they map out the design space of non-rotary positional representations.

The Recurring Setup

That text pipeline matters because the hardest decisions happen before the softmax. A positional method must decide:

  • what the position representation unit is,
  • where the representation enters the attention computation,
  • which structural priors are enforced during training,
  • and whether the resulting hidden states can still be reused at inference time.

The first visual is useful because it makes one point concrete: position does not have to start as a fixed lookup row. It can be formatted, encoded, and regularized like a small structured input.

Why Position Becomes a Bottleneck

There are three bottlenecks in this family, and each one constrains a different part of the Transformer.

When Length Extrapolation Fails

A learned lookup table is comfortable inside the training span and brittle outside it. Even when positions are represented continuously, the model still needs some notion of which unseen indices should stay close and which should separate. Otherwise, the representation may overfit to the spelling of a position ID instead of its geometric meaning.

That is the lesson illustrated by SeqPE. It treats a position like a short structured string rather than a single opaque integer. The point is not the digit format by itself. The deeper idea is that if raw position IDs do not extrapolate, the model may need a representation whose internal structure survives beyond the largest seen index.

When Attention Rank Becomes Too Weak

Another failure mode appears even before extrapolation. If position is only added to token embeddings at the input, the attention heads may receive an order signal that is too entangled with content and too limited for richer positional interactions. In that view, the bottleneck is not only the embedding table. It is the rank and placement of positional information inside attention itself.

That is the interface problem illustrated by DIET: keep content-based matching, but add positional structure directly to each head’s attention logits.

When Caches Carry the Wrong Position

Long-context decoding introduces a more practical constraint. If a model reuses cached hidden states, those cached values cannot be stamped with the wrong absolute location. Once position is baked into reusable states, moving them into a new decoding step can make old tokens behave as if they are still at their previous coordinates.

That is the bottleneck illustrated by Shortformer with Position-Infused Attention. The useful principle is narrow but durable: let position influence who attends to whom, while keeping the cached message itself as reusable as possible.

The Main Design Axes

The embedded inputs support six design axes, and they are enough to organize this topic without turning it into a paper catalog.

Representation Unit

One option is to encode a position as a digit sequence. This gives the model reusable local structure such as place value and dimension identity. SeqPE uses this route so the same encoder can handle 1D token indices and 2D image positions.

Another option is to leave the position as an index and inject its effect later, for example as a learned term in the attention logits. DIET and Shortformer both stay closer to this second style, but they use it for different reasons: one to improve expressivity, the other to preserve cacheability.

Injection Site

Where the signal enters the network changes what can be reused and what can be expressed.

  • Input-side addition is simple, but it tightly couples content and position.
  • Per-head logit injection gives each head its own structural map.
  • Query-key-only injection preserves the causal attention interface while avoiding position-coded reusable values.

This is why “position embedding” is too narrow a phrase for the whole topic. The more important question is where order enters the computation.

Geometric Prior

If a model should extrapolate, it needs some prior about which unseen positions ought to behave similarly. SeqPE makes that prior explicit through contrastive Euclidean alignment: nearby true positions should receive similar embeddings even when their digit strings differ.

Dimensionality

The family also shows that position design is not only about long text. SeqPE explicitly targets both 1D and 2D positions, which matters because many handcrafted schemes must be redesigned when moving from token sequences to grids.

Stability Under Distribution Shift

Training tricks are not secondary here. The compact input explicitly notes that shifted teacher-student distillation can hurt on its own and becomes useful only when paired with the distance regularizer. That is a reminder that out-of-distribution stability is part of the representation design, not just an optimization detail.

SeqPE’s distance regularizer is a good example of how a positional method can teach the model what “nearby” should mean before longer contexts ever appear.

Intuition: Pick one sampled position that is truly closest to ppp, then train the encoder so its embedding is the most similar among the sampled candidates.

  • Lδ\mathcal{L}_{\delta}Lδ​: the distance-regularization loss.
  • ep\mathbf{e}_pep​: the learned embedding of position ppp.

Why it matters: The compact input flags a concrete failure case: without this term, lexically similar numbers such as 100 and 1000 can end up too close even when their true positions should not. The regularizer pushes the encoder toward geometric consistency rather than string memorization.

A Formula for Position as an Attention Bias

DIET teaches a different lesson: content matching, positional structure, and segment structure can be separated cleanly inside a single attention logit.

Purpose: Show how a head scores token jjj for token iii using standard content attention plus separate positional and segment terms.

Equation:

Intuition: First compute the normal content score. Then add one learned score for the pair of positions and another learned score for the pair of segments.

Symbols:

  • Ai,jABSA_{i,j}^{\mathrm{ABS}}Ai,jABS​: the final attention logit from token iii to token jjj in the absolute-position variant.
  • XXX: the matrix of token hidden states.
  • WQ,WKW_Q, W_KWQ​,WK​: the query and key projection matrices used for content attention.
  • PQ,PKP_Q, P_KPQ​,PK​: learned positional matrices whose interaction creates a position-specific score.
  • ES(S(i),S(j))E_S(S(i),S(j))ES​(S(i),S(j)): the learned segment bias between the segments containing tokens iii and jjj.
  • ddd: the scaling dimension used in attention normalization.

Read-aloud meaning: “Score jjj for iii by combining content similarity with an explicit positional bias and an explicit segment bias.”

Why it matters: This formulation makes the design choice visible. Position is not smeared into token vectors once at the input. It becomes a separate structural term that each attention head can use directly.

Three Representative Mechanisms

Because not_a_shared_mechanism is true in the input, this topic should not be taught as if all methods implement the same positional algorithm. They do not. What they share is the same interface pressure.

Structured Indices for Unseen Lengths

SeqPE is the clearest example of treating positions as structured inputs. It converts each position into left-padded digit tokens, marks digit place and dimension identity, then encodes that short sequence with a lightweight Transformer. This design preserves a few key invariants from the input packet: nearby positions should remain nearby, digit place should align, dimensions should stay distinguishable, and the scheme should still work for unseen larger indices.

The important lesson is broader than digit tokens. When a position representation must extrapolate across length or dimensionality, a model can benefit from a compositional format rather than a flat index.

Per-Head Structural Maps

DIET represents another design move: keep token hidden states focused on content, and let each head receive its own positional bias map during score construction. This is especially relevant when attention rank is the bottleneck. If the head needs richer structural interactions than input-added absolute embeddings can support, direct logit injection gives a cleaner place to express them.

This also explains why the family includes both absolute and relative styles under the same conceptual umbrella. The shared lesson is not “absolute is best” or “relative is best.” It is that position often works better when it shapes pairwise attention decisions directly.

Position-Light Cached Messages

Shortformer focuses on the inference side. It keeps absolute positions, but applies them to queries and keys instead of mixing them into reusable token states. That solves a specific systems problem: cached blocks can be reused across later decoding steps because the stored values are not irreversibly tied to their old absolute locations.

Its training schedule reinforces the same idea. Rather than brute-forcing the largest possible working window from the start, it trains short first and long later. In this view, positional design and sequence-length curriculum are linked: once attention can use position without contaminating the cache, longer context becomes cheaper to exploit.

What the Evidence Supports

The evidence in the compact input is enough to anchor the concepts without turning this page into a benchmark sheet.

  • SeqPE is the strongest anchor for length extrapolation. On Wikitext-103, it reaches 18.95 average perplexity across 512 to 16K contexts, compared with ALiBi at 19.54.
  • On long-context RULER-SQuAD, SeqPE is also the anchor for robustness beyond the training range, posting 12.34 average perplexity and 13.9 average EM versus ALiBi’s 36.06 and 11.3.
  • The SeqPE ablation is conceptually important: distance regularization alone helps sharply, while out-of-distribution distillation alone hurts, which supports the claim that collapse risk appears when geometric structure is missing.
  • DIET is the anchor for attention-side injection. The compact input reports that all per-head position encoding schemes outperform input-added absolute embeddings on GLUE and XTREME.
  • Shortformer is the anchor for cacheable long-context attention. Its reported gains support the systems argument that better position placement can outperform simply making the training subsequences longer.

That evidence is intentionally sparse, but it is enough to support the design claims made above.

What to Remember

Non-rotary positional design is not one fallback bucket for “everything that is not RoPE.” It is a set of concrete answers to a recurring question: what should a Transformer preserve about position, and where should that information live so the model remains expressive, stable, and reusable?

If extrapolation is the problem, give position a representation with internal structure. If attention expressivity is the problem, bias the attention scores directly. If caching is the problem, keep position out of reusable value-like states. The durable takeaway is that positional representations work best when they are matched to the exact interface bottleneck they need to relieve.
