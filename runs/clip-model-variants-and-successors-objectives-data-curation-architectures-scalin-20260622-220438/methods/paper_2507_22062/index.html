Meta CLIP 2: Scaling CLIP to Worldwide Image-Text Data

Meta CLIP 2 asks a simple question with a hard answer: what happens if CLIP stops treating the web as mostly English and starts treating it as worldwide? The paper’s claim is that multilingual degradation is not inevitable. It appears when the data curation rules, balancing thresholds, and training scale are still tuned for English. Once those pieces are made language-aware, worldwide image-text data can improve English and non-English performance at the same time.

The method is deliberately conservative. It keeps the familiar CLIP dual-encoder setup and avoids translation, distillation, or custom multimodal architecture changes. The real change is in the recipe around the model: build multilingual metadata, curate each language separately, then give the model enough capacity and enough seen pairs to absorb the extra diversity.

Why the usual CLIP recipe breaks

The paper’s starting point is the “curse of multilinguality”. If you simply add non-English pairs to an English CLIP recipe, English performance can fall instead of rise. That sounds backwards until you think about what the model is actually seeing. The new data is not just more data. It is a different mixture of languages, concept frequencies, and caption styles. If you keep the old English-only curation logic, the balance shifts in a way the model was never designed to handle.

Figure 1 makes that paradox visible.

What to notice is the turning point. On a smaller backbone, worldwide data still acts like interference. On a larger backbone, the same data becomes a resource. The paper’s main point is not that multilingual data is always good or always bad. It is that the outcome depends on whether the curation and the model scale are large enough to give every language room to matter.

What to notice here is that the paper does not treat curation and training as separate concerns. The data pipeline and the model scale together. That matters because worldwide data changes both the quality of the pairs and the amount of useful supervision the model can absorb.

Building metadata for the whole world

The first missing ingredient in worldwide CLIP training is metadata. Meta CLIP already showed that CLIP improves when raw web text is aligned to a structured vocabulary of visual concepts instead of being filtered by a black box. Meta CLIP 2 keeps that idea, but expands it from English to 300-plus languages.

The key move is to maintain separate metadata by language. That sounds like a bookkeeping detail, but it is actually the foundation of the method. A word string can mean different things in different languages, and even when the surface form looks similar, the local concept distribution is not the same. Treating every language as if it shared one global concept pool would mix those distributions together and make the matching less trustworthy.

The metadata is built from the same broad source families as before, just widened to worldwide scope:

  • multilingual WordNet synsets,
  • Wikipedia unigrams and bigrams across hundreds of languages,
  • Wikipedia page titles ranked by traffic,
  • local tokenizers for scripts that do not use spaces cleanly.

That last part matters more than it first appears. For languages like Chinese, Japanese, Thai, and others with compact writing systems, you cannot just split on spaces and hope the words survive. The paper uses community-built tokenizers so the metadata still reflects meaningful units rather than broken character fragments. In other words, the curation system has to respect the language before it can count it.

The important intuition is simple: if you want the model to learn from native-language alt-text, you first need a concept inventory that is native enough to recognize those captions as meaningful.

Curating each language on its own terms

Once the metadata exists, the paper does not throw all languages into one matching pool. It first identifies the language of each alt-text, then matches that text against the metadata for that same language. That language isolation is the heart of the curation algorithm.

The reason is easy to see with a toy example. Suppose one language has a huge amount of data and another has much less. If you apply one global balance threshold, the bigger language can still overwhelm the smaller one, and the smaller one may end up dominated by common head concepts. The result looks multilingual on the surface, but the long tail of meaningful concepts gets flattened.

Meta CLIP 2 fixes that by keeping the balance logic language-specific. The paper first uses the English setup to infer a target tail proportion, then derives a separate threshold for each language so the same rough head-versus-tail ratio is preserved everywhere. Tail concepts are always kept. Head concepts are downsampled. The only thing that changes by language is how aggressive the downsampling should be.

That is the subtle but crucial shift. The method is not saying every language should have the same number of examples. It is saying every language should preserve the same qualitative balance between common and rare concepts. That is a much better target for multilingual curation because it respects the fact that language corpora are not equally large.

A useful way to think about it is that the paper rebalances meaning, not just counts. The model should still see cats, cars, and people, but it should not see them so often that they drown out the rest of the world.

Why native alt-text matters

Another important design choice is what the paper does not do. It does not translate everything into English first, and it does not rely on a teacher model to distill English labels back onto the world. That means the supervision comes from native alt-text, not from synthetic translations.

This matters because translation can smooth away local naming, local context, and local bias. Native captions preserve how people in a region actually describe images. They also preserve concepts that may never survive a translation pipeline cleanly. The paper’s broader claim is that you do not have to choose between multilingual coverage and clean supervision if you build the curation recipe around native language data from the start.

The result is a more faithful training set. It is still noisy, because the web is noisy, but it is noisy in the languages people actually used.

Keeping training scale in sync with the data

Curation alone does not solve the curse of multilinguality. The training recipe also has to change.

The first change is seen-pair scaling. If you add a large non-English corpus but keep the old training budget, English examples get seen less often than before. That can quietly damage English performance even if the curated data is good. Meta CLIP 2 avoids that by increasing the batch size and the total number of seen pairs so the English portion does not get squeezed out as the worldwide pool grows.

In the paper’s setup, this is a 2.3x scale-up in global batch and seen pairs. The point is not the exact multiplier. The point is that the training budget should grow with the data budget. Otherwise, adding languages just redistributes attention instead of increasing usable supervision.

The second change is model capacity. The paper finds that ViT-L/14 is still too small to fully absorb the worldwide setting, while ViT-H/14 is the point where the multilingual curse finally breaks. That is an important lesson. If the model is too narrow, it cannot keep both English and non-English structure in view at the same time. A larger backbone gives the representation room to hold more variation without collapsing it into interference.

So the training side has a clear rule: if the dataset gets wider, the model needs to get wider too.

On the text side, the paper swaps in a multilingual tokenizer, and among the options they test, XLM-V performs best. That is a small change compared with the rest of the recipe, but it is still important because the tokenizer is the first place where the model learns to segment language into usable units. If that unitization is weak, the rest of the pipeline starts from a bad parse.

How the system stays feasible

Worldwide curation is more expensive than English-only curation, so the paper also pays attention to implementation. It uses the Aho-Corasick algorithm for fast substring matching, which is a trie-based way to search many metadata strings at once instead of checking them one by one. That makes million-scale metadata practical.

The matcher is built per language and loaded lazily, so each shard only keeps the languages it actually needs in memory. Counts are also memory-mapped so the balancing step does not blow up RAM when many languages are present. These choices do not change the idea of the method, but they are what make the idea runnable on billions of pairs instead of just describable in a paper.

The pipeline also keeps the usual safety and leakage protections: NSFW filtering, face and biometric removal, and deduplication against ImageNet evaluation overlap. That is important because a worldwide dataset only means something if the evaluation does not accidentally leak into it.

What the experiments say

The headline result is that the full recipe breaks the multilingual curse rather than just softening it. The ViT-H/14 worldwide model reaches 81.3 percent zero-shot ImageNet, which is above the English-only counterpart at 80.5 percent. It also reports 50.2 percent on Babel-ImageNet, 57.4 percent on CVQA, and 64.3 percent on XM3600 image-to-text retrieval.

The ablations tell the more interesting story. Using one shared threshold or merging all metadata into a single pool is worse than language-aware curation. The model improves when metadata is isolated by language, and it improves again when the threshold is adjusted per language instead of reused from English. That is strong evidence that the balance logic, not just the presence of non-English data, is doing real work.

The paper also shows that the worldwide gain is not free. On smaller capacity or weaker scale settings, the curse still shows up. That is why the method matters as a recipe rather than a single trick. The gain comes from the combination of metadata, curation, and capacity, not from any one piece alone.

What this means beyond the numbers

The larger lesson is that multilingual data is not inherently poisonous to CLIP. It becomes harmful when the pipeline treats all languages as if they were the same distribution and then asks a smaller model to absorb the result. Meta CLIP 2 argues that the fix is to treat each language as its own text distribution, preserve native supervision, and rebalance rare concepts so they are not drowned out by dominant ones.

That is why the paper is useful even if you only care about the method. It gives a practical recipe for worldwide dual-encoder pretraining:

  • keep native alt-text,
  • build language-specific metadata,
  • balance each language separately,
  • scale the training budget with the data,
  • use enough model capacity to hold the added diversity.

The method does not rely on a flashy new architecture. It relies on making the data pipeline honest about language differences.

Limits to keep in mind

The paper is strong, but it is not claiming the evaluation problem is solved. The benchmarks are still biased toward North America and Western Europe, and some multilingual benchmarks may still carry dataset bias or leakage risk. Worldwide curation is also more expensive than English-only curation, which is why the implementation tricks matter.

There is also a deeper open question: how far can this recipe scale beyond the language set studied here, and how much of the gain can be recovered by smaller models without the move to ViT-H/14? The paper points in a clear direction, but it also shows that capacity and data balance have to be pushed together.

What to remember

Meta CLIP 2 is easiest to remember as a three-part adjustment to CLIP’s usual English-first recipe.

First, give the model worldwide metadata so non-English alt-text can be matched without translation. Second, balance each language separately so rare concepts survive instead of being flattened by a single global threshold. Third, scale training capacity and seen pairs so the extra languages reinforce English instead of competing with it.

If you keep one sentence in mind, make it this: worldwide CLIP works when each language is treated as its own distribution, but all of them are still allowed to teach the same model.
