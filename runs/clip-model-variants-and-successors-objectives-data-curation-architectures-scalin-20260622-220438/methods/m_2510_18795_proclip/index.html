ProCLIP: Progressive Alignment for LLM-based Embedders in CLIP

ProCLIP is easiest to understand as a careful bridge between two pretrained systems that were never naturally designed to meet. On one side is CLIP, which already knows how to build a useful shared vision-language space. On the other side is a large language model embedder, which can understand longer and more varied text but does not automatically speak CLIP’s geometric language. The paper’s core warning is simple: if you swap the text tower and immediately start contrastive training, the model may gain language coverage while losing the alignment structure that made CLIP useful in the first place.

So ProCLIP changes the order of operations. It does not jump straight into full image-text alignment. It first teaches the new text side to inherit the old CLIP text space, and only then does it open up the full dual-encoder training loop. While that second stage is running, the vision encoder is also protected by a self-distillation signal so it does not drift away from its pretrained behavior.

The figure is worth reading as a story, not as a diagram. The first block is about inheritance, not alignment. The second block is about alignment, but with a memory of the old model still attached to the image tower. That is the big idea of the paper: a direct swap is risky, while a staged bridge is much safer.

Why direct replacement is fragile

The problem ProCLIP attacks is not that LLMs are bad at language. It is that a good language model is not automatically a good retrieval embedding. CLIP’s original text tower learned a very specific habit: compress a sentence into a vector that lives in the same space as the image embedding. That habit matters. If you throw away that geometry and let the new text tower learn everything from scratch, the model can become better at language while becoming worse at zero-shot transfer.

That failure mode shows up in a very practical way. The new text side may produce richer semantics, but the image tower suddenly has to meet a representation that has no prior relationship to its own space. The contrastive loss then has to do two jobs at once: invent the new alignment and preserve useful pretrained structure. That is a hard ask when the alignment data is far smaller than the data used to build the original CLIP model.

ProCLIP treats that as a curriculum problem. First stabilize the text side. Then do the expensive joint tuning. Do not try to solve both at the same time.

Stage 1: teach the LLM to speak CLIP

The first stage is representation inheritance. The LLM-based text encoder is used to produce text embeddings, and a small adapter learns how to map those embeddings into CLIP’s text space. The important detail is that this stage is text-only. The image tower is not part of the learning signal yet, because the immediate task is not cross-modal matching. The immediate task is translation: make the new text representation look like the old CLIP text representation.

That translation happens in two complementary ways. Instance semantic alignment says that each adapted LLM embedding should land close to the corresponding CLIP text embedding for the same sentence. Think of that as pointwise imitation: one sentence in, one target vector out.

But pointwise imitation alone can still distort the shape of the space. Two sentences can each land near their own targets while the broader neighborhood map gets bent out of shape. That is why the paper also uses structure alignment. Instead of matching only single examples, it matches the pairwise relationships inside a batch. If two captions are near each other in CLIP space, they should stay near each other after the adapter. If they are far apart, they should remain far apart. In plain language, the adapter should inherit both the vocabulary and the geography of CLIP’s text space.

That second part is the key to remembering the method. ProCLIP is not just copying vectors. It is copying the shape of the space.

This is also why the paper can scale the alignment dataset without turning the first stage into a noisy end-to-end retraining run. Once the adapter has a CLIP-shaped target, more data helps it refine the bridge rather than invent one from scratch.

Stage 2: align images and texts without erasing the vision tower

Once the text side has learned CLIP’s dialect, ProCLIP moves to the second stage. Now the projected LLM embeddings and the image embeddings are trained with the usual image-text contrastive objective. This is the part that actually closes the cross-modal loop.

The difference from direct replacement is that the model no longer starts from a blind swap. The text side already knows where it should live, so the contrastive gradients can focus on fine-tuning the joint space instead of discovering the entire geometry from zero.

The second stage is also where the paper protects the vision encoder from catastrophic forgetting. It does this with an exponential moving average teacher, which is just a slowly updated copy of the image encoder. The current image encoder is trained not only against the text side, but also against that stable teacher. That matters because the teacher acts like a memory of the image tower’s recent past. Without that anchor, the image encoder can drift too far while chasing the new text representation.

Figure 1 makes that point visible.

What to notice here is not the exact value of the similarity score. What matters is the pattern: ProCLIP keeps more of the pretrained visual structure than the direct-swap baseline. That is exactly what the self-distillation term is supposed to do. It does not freeze the image encoder completely, because the model still needs to adapt. It just stops the model from erasing the useful geometry that CLIP had already learned.

The regularizer is mirrored on the image side using the same two ideas as Stage 1. Each current image embedding is pulled toward its EMA teacher, and the relationships among images in a batch are also kept stable. That combination is what makes the vision tower adapt without becoming a different model.

The paper’s ablations make one practical point very clear: the regularization weight needs to stay small. Too little regularization and the image tower drifts. Too much and the model stops aligning well. The reported sweet spot is tiny, which is a good reminder that self-distillation is a stabilizer, not the main objective.

Why the two stages fit together

The real novelty in ProCLIP is not any single loss. It is the ordering. Stage 1 makes the new text encoder legible to the old CLIP space. Stage 2 then fine-tunes the pair while protecting the image encoder from overcorrection. That sequencing matters because it separates two very different problems:

  1. make the new text side look compatible with CLIP,
  2. improve the shared space without destroying the old one.

If those two goals are mixed together too early, the model spends its gradients on translation and repair at the same time. ProCLIP avoids that conflict by treating alignment as a curriculum. The first stage is the warm start. The second stage is the real joint tuning.

That is why the method is good at the exact things the paper cares about. It can take advantage of the broader language coverage of an LLM-based embedder, but it does not pay for that with the usual collapse in zero-shot transfer.

What changes in practice

The downstream story matches the design story. ProCLIP improves zero-shot classification substantially over LLM2CLIP, and the gains carry over across short-text retrieval, long-text retrieval, multilingual retrieval, and fine-grained matching. The method is especially convincing because the improvements are stable across model scales rather than a one-off win on a single backbone.

That said, the paper is honest about the tradeoff. The second stage is more expensive because the vision encoder is unfrozen and the self-distillation teacher has to be maintained online. The method adds roughly 35 percent computational overhead. In other words, ProCLIP buys safer alignment, but it does not buy it for free.

There is also a boundary to where the gains show up. The method helps the dual-encoder retrieval story most clearly. Some downstream MLLM benchmarks move only modestly, which suggests that preserving CLIP-style transfer is not the same thing as optimizing every larger multimodal stack that might reuse the encoder later.

That is not a weakness of the core idea. It is just the right reminder that this paper is about controlled migration of a dual encoder, not about solving every multimodal adaptation problem at once.

What to remember

If you want one sentence to keep in your head, it is this: ProCLIP upgrades CLIP by first teaching an LLM-based text embedder to inherit CLIP’s geometry, then letting image-text contrastive learning refine the joint space while self-distillation keeps the vision tower from forgetting.

That simple ordering is the whole method. Preserve the old space first, then align the new one, and only then let the model adapt more aggressively. ProCLIP is a good example of a general rule for pretrained systems: when you replace one tower with a stronger encoder, the safest path is usually not a direct swap. It is a staged handoff.
