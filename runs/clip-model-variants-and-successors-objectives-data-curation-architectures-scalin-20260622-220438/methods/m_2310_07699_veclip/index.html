VeCLIP: Improving CLIP Training via Visual-enriched Captions

VeCLIP starts from a very practical diagnosis: the CLIP objective is not the main problem, the web caption often is. Alt-text can be vague, noisy, or only loosely connected to the image, so CLIP ends up learning from text that misses the most useful visual facts. VeCLIP keeps CLIP’s training recipe intact and focuses on the text side instead. It builds a better caption from the image itself, then uses that better caption as an extra view during pretraining.

The easiest way to think about it is as a two-step repair job. First, recover missing visual content from the image. Then, rewrite the caption so that content is expressed in normal language without drifting away from the original pair.

What to notice in the figure is the division of labor. The image is used to surface concrete visual details. The original alt-text is not thrown away, but it is no longer trusted to carry the whole meaning by itself. The final caption, VeCap, is a fusion of what the image shows and what the web caption already knew.

Why the image has to speak first

If you try to rewrite a weak web caption without looking at the image, you usually only get a better-shaped version of the same weak caption. That is the key limitation VeCLIP is trying to avoid. A language model can polish syntax, but it cannot reliably invent image-specific facts that were never in the caption to begin with.

So the paper uses an image-grounded captioning step first. A multimodal captioner sees only the image and extracts a short description of what is actually there. That image-only step matters because it prevents the noisy alt-text from steering the rewrite back toward generic or irrelevant wording. In other words, the image supplies missing facts, and the captioner turns those facts into a compact textual clue.

The next step is fusion. An open-source language model reads both the original alt-text and the image-derived clue, then merges them into one cleaner sentence. The paper treats that as a strong text augmentation: it is not just swapping synonyms, it is reorganizing the sentence so the visible attributes come first and the final caption reads more like a human description of the picture.

This is also why VeCLIP is more scalable than a manual curation pipeline. The rewrite step is designed for batch processing, so the method can be applied to very large web corpora instead of only small hand-picked datasets.

VeCap is not just a prettier rewrite

The name VeCap hides the most important design choice in the paper: the final caption must stay image-grounded. That means the rewrite is allowed to clean up grammar, reorder attributes, and combine overlapping phrases, but it should not wander into new semantic territory.

The paper is careful about this because multimodal generators can hallucinate. If the image captioner invents something that is not visible, the fusion step has to pull it back toward the original text and the actual image content. That is why the fusion stage is useful by itself. It gives the model one more chance to keep the caption faithful before that caption becomes training data.

There are also practical failure cases. Very long alt-text can slow the rewrite pipeline, and problematic text can trigger safety refusals. The paper handles that by keeping the pipeline conservative when needed, for example by rewriting only the generated caption in unsafe cases or trimming long inputs so the batch process stays manageable. Those details are not glamorous, but they are what make the method usable at web scale.

Mixed caption sampling keeps the text distribution broad

This is where VeCLIP becomes more than a caption-cleaning recipe.

If every example is replaced by the same style of rewritten text, the dataset starts to look too uniform. That can help retrieval, because the model sees richer descriptions, but it can also narrow the text distribution enough to hurt zero-shot classification. The paper sees exactly that tension: VeCap alone can be very strong for retrieval, yet the model can become less comfortable with the short, generic prompts used at inference time.

The fix is mixed caption sampling. For each image-text pair, the training text is sampled from either the original alt-text or the rewritten VeCap. The CLIP contrastive loss stays the same; only the text view changes. This means the model keeps seeing the messy web phrasing sometimes and the cleaner image-grounded phrasing other times.

That small choice does a lot of work.

  • The rewritten caption improves image-text alignment.
  • The original alt-text preserves diversity and keeps the text space from collapsing into one style.
  • The alternating mix makes the method feel like augmentation rather than replacement.

Seen this way, VeCLIP is not trying to pick a winner between old and new captions. It is training the model to live with both.

Training still looks like CLIP

One of the cleanest parts of the method is that the learning objective does not change. VeCLIP keeps the CLIP contrastive loss unchanged and leaves the rest of the dual-encoder recipe alone. The image encoder still learns to match the paired text, and mismatched pairs still serve as negatives.

That matters because the paper is not trying to turn CLIP into a more complicated multitask system. It is trying to make CLIP’s supervision better without paying for a more complex objective. The gain comes from better captions, not from adding more loss terms or more model heads.

This is also why the method is easy to remember: the model architecture stays familiar, the loss stays familiar, and the only real change is the text stream that feeds the loss.

What the gains are telling you

The strongest effect shows up on retrieval. That fits the method well. Retrieval rewards captions that preserve fine-grained visible details, so a visually enriched caption gives the model more useful matching clues than a bare web alt-text string. The paper reports up to a 25.2% gain on COCO and Flickr30k retrieval in the 12M setting, and it also shows that the approach remains data-efficient when compared with much larger vanilla CLIP and ALIGN pretraining runs.

The classification story is more delicate. A very detailed caption can drift away from the simple prompts used for zero-shot ImageNet-style evaluation. That is why the mixed sampling step is not an afterthought. It is the mechanism that keeps the rewritten captions from dominating the training distribution.

So the overall pattern is easy to remember:

  • better captions help the model line up images and text more precisely,
  • original alt-text keeps the language distribution broad enough for zero-shot prompting,
  • the mix of the two gives a better balance than either one alone.

The main limitation

The paper is honest about the remaining rough edge: caption quality is still not easy to score cleanly at scale. VeCLIP can make the training text more informative, but measuring how good those generated captions really are is still somewhat underdefined. The authors also note that if the rewritten captions become too uniform or too visually overloaded, zero-shot classification can lag behind the original alt-text baseline.

That limitation is important because it explains the design philosophy of the whole method. VeCLIP is not trying to eliminate the web caption. It is trying to improve it just enough to make CLIP learn better, while preserving enough of the original data texture that the model does not overfit to one caption style.

What to remember

VeCLIP is easiest to remember as “image-grounded rewriting plus mixed training.”

First, recover missing visual facts from the image. Then fuse those facts with the original web caption into VeCap. Finally, train CLIP with mixed caption sampling so the model alternates between the richer rewrite and the original alt-text. The CLIP contrastive loss stays the same the whole time.

That is the core lesson of the paper. If the text supervision is noisy, do not just polish it. Use the image to make the text say something better, then keep enough of the original text around that the model still learns to speak the web’s language at inference time.
