Cross-Modal Retrieval Meets Inference: Improving Zero-Shot Classification with Cross-Modal Retrieval

X-MoRe is a CLIP inference trick before it is anything else. It does not change the backbone, it does not add adapters, and it does not ask the model to relearn its class space. Instead, it asks a more practical question: if a frozen CLIP model is uncertain about an image, can we give it a second opinion at test time from a large external image-text memory?

The paper’s answer is yes. X-MoRe retrieves related image-text pairs, turns the retrieved captions into a second class prediction, and then blends that text-based prediction with the original image-based prediction. The blend is not fixed. It is weighted by which branch looks more reliable on the current example.

The whole method is easy to keep in your head as a pipeline:

That is the real idea in one sentence: CLIP can be stronger at inference if it is allowed to consult external multimodal memory and then trust whichever modality seems more dependable for the current sample.

Why the paper takes this route

The paper starts from a familiar problem. Plain CLIP zero-shot classification is convenient, but it can be brittle. A prompt like “a photo of a class name” often gives a good first guess, yet the prediction can still wobble when the image is ambiguous or out of distribution. Fine-tuning can help on seen classes, but it can also damage the very zero-shot behavior that made CLIP attractive in the first place.

X-MoRe avoids that tradeoff by refusing to retrain the model. The encoders stay frozen, and the only extra ingredient is an external image-text database at inference time. In the paper, that memory comes from a large public image-text corpus. The method is therefore a retrieval-and-ensemble story, not a new training recipe.

That design choice matters because it keeps the original CLIP interface intact. You still start from a query image and class names. You still get a final class distribution. The difference is that the prediction is no longer based on the query image alone. It is guided by nearby examples and by a confidence estimate that says which branch should matter more.

The retrieval step is the heart of the method

The first stage is not just “find similar images.” It is a two-stage retrieval process that uses images first and captions second.

What to notice in this figure is the handoff between the two stages. The query image first searches for nearby image-text pairs in the external pool. That gives a coarse candidate set. Then the method compares the query image directly with the captions of those candidates and keeps the captions that are most useful for the query.

That second step is important. Image similarity and caption usefulness are related, but they are not the same thing. A visually similar neighbor may have a caption that is not very helpful, while a slightly less similar neighbor can carry a caption that names exactly the right concept. If you stop after image retrieval, you are still trusting the image ranking to do a text task. X-MoRe does not make that mistake.

The paper’s default setting keeps a pool of 128 candidate image-text pairs and then fine-retrieves 16 captions from that pool. The broad pool gives the method room to look around, while the smaller set of captions keeps the final text signal focused. That balance is part of the method’s strength. Too few captions and the text branch becomes noisy or overconfident. Too many and the retrieved language starts to drift away from the query.

Turning retrieved captions into a second prediction

Once the top captions are selected, X-MoRe treats them as a second source of evidence for the class label.

The text branch works by encoding each retrieved caption with CLIP’s text encoder and comparing the resulting representation with the class prompts. That produces a class distribution from text, just as the original query image produces a class distribution from vision.

The retrieved captions are then averaged into one text-side prediction. This averaging step is a small but important stabilizer. A single caption can be misleading, especially if it accidentally mirrors the class label too directly. Several captions are safer because they spread the risk across multiple nearby examples.

This is also why the paper reports that 16 captions is a strong setting. The text branch needs enough evidence to be useful, but not so much that it becomes a blur of weakly related descriptions. The best caption count is the one that gives the text branch context without turning it into noise.

Think of the text branch as a second witness. One witness can be overconfident and wrong. A small group of witnesses is usually less dramatic, but more trustworthy.

Confidence is what makes the ensemble work

The ensemble is not a plain average. X-MoRe asks how certain each modality looks before combining them.

The confidence signal is based on entropy, which in plain language means “how sharp or flat the prediction distribution looks.” If a branch puts most of its mass on one class, it looks confident. If it spreads probability across many classes, it looks uncertain. So low entropy means high confidence, and high entropy means low confidence.

That is a good first signal, but the two branches do not naturally live on the same scale. The image branch and the text branch can have very different confidence ranges, so raw entropy alone is not enough. X-MoRe therefore rescales each modality separately using the range of confidence values it has seen so far for that modality.

The point of that rescaling is easy to miss, but it is central. The method is not trying to say that image confidence and text confidence are directly comparable in absolute terms. It is saying that each modality should be judged relative to its own history. A confidence value that is high for the image branch and a confidence value that is high for the text branch should both count as “trust this branch more than usual,” even if their raw numbers do not match.

Once those modality-specific confidences are normalized, the final prediction is just a weighted blend of the image distribution and the text distribution. If the image branch looks clearer, it gets more weight. If the retrieved captions look cleaner, they get more weight. The method is trying to make the better branch speak louder on each individual example instead of forcing both branches to contribute equally.

That is the key intuition to remember: X-MoRe is not a search-and-average method. It is a search-and-trust method.

Why the direct text retrieval matters

One of the paper’s most useful ablations is also one of the simplest. The authors compare direct text retrieval with indirect text retrieval. Direct retrieval means comparing the query image with candidate captions in the shared embedding space. Indirect retrieval means retrieving images first and then inheriting their captions without the extra text-level check.

Direct retrieval wins. The reason is straightforward once you think about it. An image can be the right neighbor while its caption is the wrong evidence. The text branch should not be limited by the image ranking if the goal is to help classification through language. X-MoRe’s second-stage text search fixes exactly that mismatch.

The paper also shows that using multiple captions helps prevent overconfident wrong text predictions. That fits the intuition above. A single retrieved caption can be too sharp for its own good, especially because the text encoder shares structure with the class prompts. Multiple captions reduce that brittleness.

What the experiments actually say

The reported results support the method’s central claim: retrieval-backed inference can improve zero-shot CLIP without extra training.

With CLIP-RN50, X-MoRe reaches a 61.04 average accuracy across the 10 zero-shot classification datasets used in the paper. That is better than plain CLIP, caption ensemble, CALIP, and TPT in the reported comparison. With CLIP-ViT-B/16, X-MoRe reaches 65.57 average accuracy on the same benchmark family.

The pattern is more interesting than the headline number. X-MoRe helps most when the image branch and the text branch are both reasonably reliable. In that setting, weighting them by confidence gives the model a better chance to pick the right side for each sample. When one branch is much weaker than the other, the ensemble can drag performance down instead of pulling it up. The paper sees that effect more clearly in the ViT-B/16 setting, where the image branch already dominates more strongly.

The domain-generalization results tell the same story. X-MoRe can improve robustness on shifted ImageNet variants, but the gain is not uniform across every backbone and every shift. That is not a flaw in the method so much as a reminder of what it depends on: the external memory has to supply useful evidence, and the two branches have to be close enough in quality that confidence weighting is meaningful.

The limits are part of the method

X-MoRe is elegant, but it is not free.

It depends on an external image-text database at inference time, which means retrieval latency and system complexity are built into the pipeline. It also assumes that the retrieved memory is actually useful for the query. If the pool is weak or badly matched to the target domain, the extra captions may not help.

The method is also most appealing when the two modalities are comparably strong. If the image branch is already much better than the text branch, or if the retrieved captions are consistently noisy, the weighted blend can become a liability. In other words, the confidence mechanism is helpful, but it is not magic. It still depends on the quality of the evidence each branch can produce.

What to remember

X-MoRe is best remembered as CLIP plus a retrieval-backed second opinion.

The model keeps the encoders frozen, uses the query image to retrieve related image-text pairs, re-reads the best captions directly against the image, and then blends image and text predictions according to how confident each branch looks. The retrieval stage gives the method extra context, and the confidence stage stops that context from being treated as equally trustworthy in every case.

If you want the shortest possible summary, it is this: X-MoRe does not teach CLIP a new representation. It gives CLIP access to external multimodal memory at test time and then lets confidence decide which signal deserves more weight.
