CLIP's biases in multi-object scenarios: encoder analysis, origins, and impacts

CLIP is usually presented as a clean image-text matcher: encode an image, encode a caption, compare the two, and choose the closest pair. This paper asks what happens when that tidy setup runs into a crowded scene. The answer is not that CLIP becomes random or useless. It is subtler than that. In multi-object settings, CLIP often behaves as if one object deserves most of the representation. On the text side, that object is usually the one mentioned first. On the image side, it is usually the one that takes up the most space.

Figure 1 gives the whole arc at a glance: build a controlled benchmark, probe the encoders directly, trace the bias back to training data and contrastive learning, and then see how the bias changes downstream matching behavior.

What matters here is the flow. The paper does not jump straight to a new model. It first builds a measurement setup that can separate order effects from size effects. That choice is what makes the rest of the analysis useful.

Start with a controlled scene

The benchmark is the first important idea. ComCO is built from COCO objects, but the scenes are rendered in Blender so the authors can control object count, location, and size. That lets them ask very specific questions: if two captions describe the same objects but in a different order, does the text tower react? If two objects are both present but one is much larger, does the image tower react? Those are the right questions because they isolate the two biases instead of blending them together.

The paper also uses SimCO, which replaces real objects with simple shapes. That matters because it shows the effect is not just about natural object categories, texture, or scene realism. Even when the objects are reduced to clean geometric forms, the same broad pattern still appears.

The method can be read as a simple probe pipeline:

The key idea is that the model is not being asked to solve a hard open-ended task. It is being asked a narrower question: which single object does this mixed embedding behave as if it prefers? That makes the bias measurable instead of anecdotal.

Figure 2 is the heart of the measurement strategy. TOR asks whether a multi-object caption embedding is closest to the single-object caption for the first, second, third, or later object. IOR asks the same question for images by comparing a multi-object image embedding against single-object images. The appendix repeats the same logic with single-layer classifiers, which is useful because it strips away the nearest-neighbor retrieval step and checks whether the object information is linearly readable from the embedding itself.

What the probes reveal

The retrieval results are very consistent. In TOR, the text encoder tends to collapse toward the first mentioned object. On ComCO, CLIP LAION reaches 63.96% for the first object but only 3.76% for the fourth. Similar descending curves show up across other CLIP variants, SigLIP, and NegCLIP. The exact numbers shift, but the shape does not: earlier objects are easier to recover than later ones.

That matters because it says the text embedding is not preserving every object evenly. It is compressing the caption around the earliest item, which means a semantically correct reordering can still change the embedding a lot. The same skew survives on longer caption templates, so this is not just a short-sentence artifact. On real COCO, the same direction appears too, which is why the authors treat ComCO as a microscope rather than a toy.

The appendix classification probes make the same point from another angle. If a simple classifier can read the first object more easily than the rest, then the bias is already inside the representation, not just in the retrieval protocol.

The image side behaves differently but just as clearly. In IOR, the image encoder prefers the larger object. On ComCO, CLIP openAI gets 87.88% for the large object in the two-object setting but only 12.12% for the small one. Again, the pattern persists across model families. The object that takes up more visual real estate is the one the embedding seems to preserve best.

Here the classification probes are especially reassuring. If a classifier can recover the large object much more easily than the small ones from the same embedding, then the size bias is not an artifact of the nearest reference image. It is encoded in the representation itself.

Why the image side leans toward size

Figure 3 gives the simplest explanation for the image encoder bias.

The intuition is almost built into the architecture. A ViT-style image encoder breaks the image into patches. A large object covers more patches, so it naturally contributes more evidence to the CLS token. That means the encoder is not just seeing the object more often; it is also aggregating more tokens from it. Figure 3 shows exactly that behavior: the CLS token spends more attention on the large object than on the smaller ones.

So the size bias is not mysterious. It is the kind of bias you would expect when a patch-based encoder must compress a scene into one global vector. Larger objects produce louder visual signals, and the CLS token listens.

Why the text side leans toward order

The text side is more interesting, because architecture is only part of the story. Causal attention can make the first token especially influential, but that alone does not explain why a bidirectional text encoder variant still shows a similar preference. The paper’s argument is that training is doing most of the work here.

Figure 4 ties together the three pieces of evidence.

First, larger objects have more influence on text embeddings when they are mentioned early. That makes sense if the text tower is learning from paired image-text data where large objects are visually dominant. Second, the LAION caption audit shows that large objects are often mentioned earlier in the caption. So the data itself already ties visual prominence to early mention. Third, the bias strengthens as training progresses, which is what you would expect if the effect accumulates through contrastive learning rather than appearing from nowhere.

The paper also sketches a useful intuition for why this can happen even when the text representation is incomplete. Contrastive training does not require the text encoder to preserve every detail in the caption. It only needs enough shared content to win against the negative examples in the batch. That means a text encoder can settle on a representation that captures the obvious, easy-to-match parts of the scene while quietly dropping some of the rest. In a multi-object caption, the first and most salient object is a natural place for that shortcut to form.

This is a sketch rather than a full mechanistic proof, but it explains the observed pattern well. The model can look excellent on the contrastive loss and still be incomplete in the way it represents ordered content.

When the bias helps, and when it hurts

The downstream matching experiment makes the whole story concrete.

The two scenarios differ only in where the important object appears. In the first scenario, the correct caption puts the largest object first, while the incorrect caption puts a nonexistent object first. That arrangement accidentally aligns with the model’s bias, so the bias helps separate correct from incorrect pairs. In the second scenario, the correct caption moves the largest object to the end, while the incorrect caption delays the mismatch to the end as well. Now the same bias becomes a liability.

That is why the second scenario is the revealing one. On ComCO, CLIP openAI drops to 52.23% there, while a simple split-caption aggregation trick lifts it to 88.56%. The same pattern also appears on COCO, where natural scenes are messier but the order sensitivity is still visible. The lesson is not that rephrasing should always fail. It is that a model with a positional bias can look strong in one caption order and brittle in another, even when the underlying meaning is unchanged.

The split-caption fix is a good example of the paper’s pragmatism. If one object order is dominating the embedding, then separate the caption into object-level subcaptions and combine the resulting embeddings instead of trusting the original sequence alone. That helps a lot on this benchmark, but the authors are careful to present it as a heuristic, not a universal solution.

What this analysis does not settle

The paper’s evidence is strong, but it is still a diagnosis paper, not a full causal proof.

ComCO and SimCO are synthetic controls. That is a strength because they isolate order and size, but it is also a limit because real scenes contain more clutter, more ambiguity, and more varied language. The LAION and COCO audits help bridge that gap, yet they do not eliminate it.

The causal story about training is also partly indirect. The attention patterns, caption statistics, and training-stage curves all point in the same direction, but they do not give a mechanistic theorem that proves exactly how the bias is formed in every model.

And the mitigation is not general-purpose yet. Splitting captions is useful here, but it is tailored to this controlled setting. The paper itself treats it as a promising direction, not a finished remedy.

What to remember

The main lesson is easy to keep in your head: CLIP-style dual encoders do not necessarily preserve every object in a multi-object scene equally. The text tower often leans toward the first object, the image tower often leans toward the largest object, and contrastive training plus caption statistics make those shortcuts more likely.

If you remember one thing, remember the shape of the failure: the model does not need to be completely wrong to be compositionally brittle. It only needs to preserve the easiest object too well and the rest too weakly. That is enough to make reordering a caption or changing object scale alter the embedding in ways that matter for retrieval, matching, and downstream generation.
