UMG-CLIP: A Unified Multi-Granularity Learning Framework for Vision-Language Foundation Models

UMG-CLIP is CLIP with a better sense of “where” as well as “what”. Standard CLIP-style training is very good at saying that an image and a text description belong together, but that global match can hide the local evidence that makes the statement true. A model can recognize that a photo is about a dog, a chair, or a street scene without learning which region, patch, or pixel actually supports that idea. UMG-CLIP tries to fix that by training one dual encoder on multiple granularities of pseudo supervision at the same time.

The practical goal is easy to remember: do not only ask the model to align whole images with whole captions. Also ask it to align regions with region captions, regions with region tags, and image-level tags with image-level captions. Once the model has learned to see those smaller correspondences, dense tasks such as segmentation and open-vocabulary recognition become much less of a stretch.

Why global-only CLIP hits a wall

The problem UMG-CLIP is solving is not that CLIP is weak overall. It is that a single global image vector is often too coarse for the kinds of questions dense vision tasks ask. If the text says “the chair beside the table” or “the small object in the corner”, the model has to preserve enough spatial detail to know which part of the scene matters. If all of that detail gets averaged into one global embedding too early, the model can still do well on broad classification but miss the local evidence that matters for grounding and segmentation.

That is why the paper starts by changing the data, not just the architecture. Instead of depending on the weak local structure hidden inside ordinary image-text pairs, it manufactures explicit multi-granularity supervision. The model is then trained to treat local correspondence as a first-class skill rather than a downstream afterthought.

Building UMG-41M: the data is already multi-granular

The heart of the paper is the dataset pipeline. Figure 2 is the clearest overview of what the authors are doing.

What to notice here is that every branch starts from the same image, but each branch extracts a different kind of supervision. The image-level branch uses a tag model like RAM and captioners such as BLIP2 or Shikra, then filters captions with EVA-CLIP scoring. The region-level branch uses detectors like ViTDet to propose boxes and class labels, then uses GPT4ROI or Shikra to write captions for those boxes. The pixel-level branch adds masks, using SAM for foreground and Mask DINO for background, with box jittering used as a quality filter for unstable masks.

The important thing is not the exact zoo of annotators. The important thing is the structure they create. Instead of one noisy caption per image, UMG-CLIP gets a much richer training example: global tags, global captions, region tags, region captions, and pixel-level masks. The paper scales this up across six public datasets and ends up with UMG-41M, which is on the order of 41 million images and 389 million regions.

That scale matters because it changes what the model can learn. A standard web caption mostly tells the model the gist of the scene. Multi-granularity pseudo labels tell it which part of the scene supports which phrase.

The resulting annotations are easier to understand in one concrete example.

If Figure 2 is the factory, Figure 3 is the finished training sample. One image now carries several different teaching signals at once. There are global tags and captions for the whole scene, plus masks that separate foreground from background. That is exactly the kind of supervision a dense model needs if it is going to reason about local objects instead of only the scene as a whole.

How one dual encoder uses all that supervision

Once the dataset exists, the model itself is still recognizably CLIP-like. Figure 4 shows the basic flow.

The image encoder produces a global class token plus a grid of visual tokens. Region boxes are then turned into region embeddings with RoIAlign over those visual tokens, so the model can compare text not only with the whole image but also with a specific region inside the image. The text encoder processes image-level tags and captions as well as region-level tags and captions, and the whole system is initialized from EVA-CLIP weights.

The training objective is a weighted combination of two parts. One part handles image-level alignment, and the other handles region-level alignment. The image-level part matches image tags and image captions against the global image representation. The region-level part matches region tags and region captions against the region embeddings. The paper sets the two weights equally, so the model is asked to care about global semantics and local semantics at the same time.

There is also a useful loss choice hidden in the details. Captions are trained with a contrastive objective, which makes sense because captions are distinct descriptions that should pull the right pairs together and push wrong pairs apart. Tags are trained with a softer smooth-L1 style loss because different images and regions can legitimately share the same or overlapping tags. That avoids over-punishing semantic overlap when the label space is messy.

This is the core training idea in plain English: the model should learn to agree with the right whole-image description, but it should also learn to agree with the right description of a box or region.

Making high-resolution training affordable

The first obvious problem with region supervision is memory. Fine-grained pretraining works best at higher resolution, but higher resolution means more visual tokens and a much heavier transformer stack. UMG-CLIP does not solve that by randomly dropping patches. Instead, it clusters redundant visual tokens and keeps only representative tokens moving through the transformer.

That distinction matters. Random masking, like the kind used in FLIP, is fine when the goal is just to speed up global matching. UMG-CLIP needs more than that, because region-level learning depends on preserving spatial and semantic detail. So the paper clusters tokens based on similarity and distance, keeps the representative tokens, and only expands the merged positions back to full resolution at the end. In other words, it compresses redundant areas without throwing away the structure that local supervision needs.

The paper says this reduces 75 percent of the visual tokens during pretraining. A good mental model is: merge near-duplicates, do not blindly delete them. The goal is to save memory while still keeping enough local information for region-text alignment to work.

After pretraining, keep the backbone mostly frozen

UMG-CLIP is not trying to become a giant fully finetuned vision system for every downstream task. The authors lean on the fact that the backbone already learned general multi-granularity behavior, then use parameter-efficient tuning to adapt it to specific tasks.

The adaptation recipe is deliberately lightweight. The pre-trained backbone is frozen, and small learnable Convpass modules are added in parallel with the normal attention and MLP layers. Those modules use a simple 1x1, 3x3, 1x1 convolution stack with GELU activations, and their contribution is scaled down so they nudge the representation instead of overwhelming it.

That design is practical for two reasons. First, it preserves the general knowledge already stored in the pre-trained backbone. Second, it makes downstream adaptation cheaper and more stable than full finetuning. For global tasks like classification and retrieval, the model mainly needs a light touch. For dense tasks like panoptic and semantic segmentation, it can reuse the same general backbone but attach task-specific decoders and a denser adaptation setup.

This is also where the extra pixel-level data becomes useful. The paper extends the annotation pipeline to a dense adaptation dataset called BigDet Panoptic, which gives the downstream segmentation stage much better supervision than the usual COCO-only recipe.

Why the representation ends up more local

The best qualitative check in the paper is the token-text correspondence visualized in Figure 5.

What to notice here is the difference in concentration. The EVA-CLIP baseline tends to spread attention or similarity over a broader, noisier area, while UMG-CLIP tightens the response around the object that actually matches the text. The red and green comparisons are not about prettier heatmaps. They are showing that the representation has become more discriminative at the region level.

That is the reason the same backbone can improve both global and dense tasks. If the tokens know which local pieces belong to which concept, classification still works, retrieval gets cleaner, and segmentation has a much better starting point.

What the paper shows in practice

The empirical story is consistent with the design. UMG-CLIP-L/14 reaches 72.1 average zero-shot classification accuracy, slightly ahead of EVA-02-CLIP-L/14, and UMG-CLIP-E/14 goes higher still at 74.6. Retrieval improves as well, which is a good sign that the extra local supervision is not destroying the shared embedding space. On dense tasks, the biggest gains show up when the model is adapted with the larger BigDet Panoptic data, which is exactly what you would expect if the pretraining really did improve spatial correspondence.

The deeper takeaway is more important than any single benchmark. UMG-CLIP shows that better granularity in the data and the loss can make CLIP-like models behave more like true visual reasoners without abandoning the dual-encoder setup.

The tradeoff the paper accepts

The cost is obvious and the paper says so directly. The annotation pipeline is cumbersome, it depends on several external models, and the pseudo-labels can still be improved. That is the price of getting local supervision at scale without paying for manual annotation on every example.

So the method is best thought of as a systems-aware upgrade to CLIP: spend the complexity budget in automated data curation and token handling, then keep downstream adaptation cheap. If you want CLIP to work beyond global matching, this is the kind of tradeoff that makes sense.

What to remember

UMG-CLIP is a three-step story. First, build a large pseudo-labeled dataset that exposes image-level, region-level, and pixel-level structure. Second, train one CLIP-like dual encoder on both global and regional image-text matching, while using token clustering to keep high-resolution training affordable. Third, adapt the frozen backbone with PET instead of full finetuning.

The short memory hook is this: CLIP learns the scene, UMG-CLIP learns the scene and the pieces inside it. That is why it stays competitive on global recognition while becoming much more useful for dense understanding.
