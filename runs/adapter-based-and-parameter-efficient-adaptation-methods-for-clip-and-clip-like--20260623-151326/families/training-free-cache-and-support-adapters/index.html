Few-Shot CLIP Adaptation with Training-Free Caches and Support Memories

CLIP is already good at recognizing new categories from text alone, but few-shot adaptation asks for something more precise: use a tiny labeled support set to become better on a new task without paying the full cost of retraining. That sounds easy until the support set is only a handful of examples, the target distribution is slightly off from CLIP’s pretraining world, and any learned adapter can overfit before it becomes useful.

Show how support features and labels become adapter weights and retrieved logits. That is why this subtopic matters. The common theme is not a single algorithm. It is a shared interface problem: how should a frozen vision-language model use a tiny support set at inference time? Some methods treat the support set as a searchable memory, some improve that memory so it better matches the target domain, and some regularize the correction so local neighbors do not distort the decision boundary too aggressively.

The Basic Pipeline

The recurring setup in this group is:

The key decision is to leave CLIP mostly untouched. A test image is encoded with the frozen visual encoder, compared against stored support examples, and then the retrieved evidence is fused with CLIP’s original zero-shot prediction. Instead of learning a deep task-specific head, the system asks a smaller question: can the support set itself act like an adapter?

This framing is attractive because it preserves what CLIP already knows. The text classifier still provides a semantic prior, while the support cache injects task-local evidence. In practice, that means adaptation becomes a problem of memory construction, retrieval, and fusion rather than end-to-end retraining.

Why This Is Hard

The difficulty comes from three different bottlenecks.

First, few-shot adaptation cost is unforgiving. If a method needs long optimization, it loses the main advantage of starting from CLIP. With only a few labeled images, expensive tuning also increases the risk of learning noise instead of signal.

Second, the support set may not actually match the target distribution. A tiny image-only cache can miss style, context, or instance-level cues that matter at test time. If the stored support examples do not look like the incoming queries, retrieval becomes brittle no matter how elegant the fusion rule is.

Third, nearest-neighbor style correction is often too local. A support cache can help when the query lands near good examples, but it can also amplify accidental neighbors, sparse coverage, or boundary artifacts. This is where local bias and overfitting enter, even when the method is nominally training-free.

The Main Design Axes

These methods differ along a few concrete axes supported by the input.

Where adaptation comes from

One choice is whether the extra task knowledge lives entirely in a support cache. In Tip-Adapter and Tip-Adapter / Tip-Adapter-F, the support set becomes a memory of instance-level keys paired with labels. The model does not learn a large new module; it retrieves from stored examples.

How support evidence is fused with CLIP

Another choice is fusion style. The dominant pattern here is residual logit fusion: let the cache vote, let CLIP vote, and combine them instead of replacing one with the other. This is important because the zero-shot prior still carries broad semantic structure, especially when the support set is noisy or tiny.

What counts as support memory

Support memory does not have to be image-only. CapS-Adapter pushes this axis further by caching both image and text-side support features, using caption-guided synthetic support to better align the memory with the target domain.

Whether anything is trained at all

Some methods stay fully training-free, while others allow a very small amount of tuning. The interesting compromise is key-only adaptation: keep the labels fixed, preserve the cache structure, and update only the support keys when extra accuracy is worth a few epochs.

How local the correction should be

Pure retrieval is flexible, but it can become too myopic. ProKeR treats this as a modeling problem rather than just a retrieval heuristic, adding a globally anchored correction so the support set bends CLIP’s decision function without fully replacing it.

When a Cache Becomes an Adapter

The cleanest way to understand the training-free cache idea is to view each support example as a stored hint. A test image looks for nearby hints, those hints cast weighted votes over labels, and CLIP’s original text-based prediction remains in the loop.

Why it matters: This is the core reason cache-based adapters can stay lightweight. The support set acts like a task-specific correction layer without requiring a deep trainable head.

This is the main lesson taught by Tip-Adapter. The support set is not just supervision for later training; it is the adapter itself. The method’s contribution is conceptual as much as practical: if the pretrained embedding space already organizes class evidence well, direct retrieval may be enough to capture much of the few-shot gain.

The evidence in the compact input supports that framing. Tip-Adapter is reported to surpass zero-shot CLIP on ImageNet across 1, 2, 4, 8, and 16 shots, and Tip-Adapter-F surpasses CLIP-Adapter while using 20 fine-tuning epochs instead of 200. The important point is not a single score. It is that a strong cache initialization removes most of the adaptation burden.

Support Sets Fail Before Retrieval Fails

A cache can only be as good as the support examples it stores. If the few-shot images are generic, sparse, or stylistically mismatched to the test set, retrieval will faithfully return the wrong kind of evidence. This is the support-set distribution mismatch problem.

CapS-Adapter addresses that bottleneck by changing what goes into memory. Instead of relying only on a small image support set, it uses target-sample captioning to build caption-based prompts, synthesizes support images from those prompts, and caches both the generated image features and the text-side features. The goal is to make the memory look and sound more like the target domain before inference even starts.

Why it matters: This changes the adaptation problem from “retrieve the nearest image support example” to “retrieve the support evidence that best matches the target domain across modalities.”

The compact evidence is enough to justify why this matters. CapS-Adapter reports an average accuracy of 64.94% across 19 datasets, improving 2.19% over the prior leading training-free method. More importantly for the concept, the method shows that better support construction can matter as much as better fusion.

Why Purely Local Retrieval Can Mislead

Once support retrieval works, a new problem appears: local similarity is not the same thing as a well-behaved decision rule. A query may land near a few misleading examples, or the support set may be too sparse to define a stable boundary. In high-dimensional few-shot settings, this can make retrieval-based correction brittle.

ProKeR addresses this by reinterpreting cache adaptation as kernel regression. That move matters because it reveals a limitation in plain cache voting: it behaves like a local estimator. ProKeR keeps the training-free spirit, but it anchors the correction to the zero-shot CLIP predictor through a regularized kernel view. Conceptually, the support set is still providing local evidence, yet the base model remains a global prior that smooths the correction.

This is a useful mental upgrade. Instead of thinking “the cache overrules CLIP when neighbors agree,” think “the cache bends CLIP’s function near the support data, but regularization prevents the bend from becoming arbitrary.” The compact input reports an average gain of 3.94% over Tip-Adapter across 11 datasets, which is exactly the kind of evidence that supports the local-versus-global design axis.

What the Evidence Actually Says

The strongest evidence here is narrow but consistent.

  • Tip-Adapter shows that a training-free support cache can already beat zero-shot CLIP across standard few-shot ImageNet settings.
  • Tip-Adapter-F shows that if extra tuning is allowed, tuning only the cache keys can outperform heavier adapter training with far fewer epochs.
  • CapS-Adapter shows that support-set quality is not a side issue; multimodal, target-matched support memories can improve training-free adaptation further.
  • ProKeR shows that better inductive bias matters after retrieval starts working; purely local corrections can be upgraded through globally anchored kernel regularization.

That is enough to support the big picture. The field is moving from “learn a small adapter” toward “engineer the interface between frozen embeddings and tiny task memories.”

What to Remember

Training-free CLIP adaptation works when the pretrained model already provides a strong semantic geometry and the support set is used carefully. The real question is not whether to add an adapter block. It is how to turn a tiny support set into a reliable correction signal.

Three durable lessons follow.

First, a support cache can replace most of the training burden when the embedding space is already organized well. Second, better support memories often come from better support construction, not just better similarity formulas. Third, retrieval alone is not enough; once adaptation depends on local evidence, regularization and global priors become essential.

The common theme is therefore broader than any one method: few-shot adaptation for CLIP becomes easier when support examples are treated as structured memory, harder when that memory is mismatched, and safer when local corrections remain anchored to the base model.
