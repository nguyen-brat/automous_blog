Keeping CLIP Prompt Tuning Stable Under Weak Supervision

CLIP is useful because it starts with broad visual-language knowledge before any downstream adaptation begins. The catch is that prompt tuning can erase that advantage very quickly. With only a few labels, missing images, pseudolabel noise, or distribution shift, the trainable prompt can learn shortcuts faster than it learns the task. The common theme in this subtopic is not a single shared algorithm. It is a shared interface problem: how do you adapt a frozen CLIP-style model when the adaptation signal is weaker or less reliable than the pretrained model itself?

That question matters because prompt tuning is supposed to be the safe version of adaptation. It changes only a tiny parameter set, keeps the encoders frozen, and should preserve most of the backbone’s prior knowledge. But small trainable scope does not automatically mean stable learning. The methods in this group show that the real problem is deciding what the prompt should trust: scarce labels, zero-shot priors, text descriptions, pseudolabels, or a carefully constrained mix of them.

The Adaptation Loop

The recurring setup is simple on paper:

Most methods here keep both CLIP encoders frozen and train only prompt vectors or closely related prompt-side parameters. That means nearly all of the design freedom is packed into a narrow interface layer:

  • what supervision is available,
  • how prompt updates are computed,
  • how much deviation from zero-shot behavior is allowed,
  • when stability is enforced: before adaptation, during optimization, or across multiple refinement rounds.

Some methods change where supervision comes from rather than how gradients are filtered. TaI-DPT, for example, learns prompts from text descriptions and then deploys them on images, leaning on CLIP’s pretrained cross-modal alignment instead of labeled image examples.

Where the Interface Fails

The backbone usually is not the weak part. The weak part is the adaptation signal.

In few-shot tuning, the prompt can overreact to tiny labeled sets and drift away from useful zero-shot behavior. In label-scarce settings, the model may have to bootstrap from its own predictions, which risks reinforcing its early mistakes. In noisy-label settings, a bad label can produce a large update that fights a strong pretrained prior. In missing-image settings, the bottleneck is even earlier: the model has to adapt without the standard image-label pairs that prompt tuning usually assumes.

So the central issue is reliability mismatch. Pretrained alignment is broad but indirect. Downstream supervision is direct but often weak. These methods differ mainly in how they manage that mismatch.

When the Backbone Should Say No

One important design axis is whether the downstream task gradient gets unconditional authority. Several methods here say it should not.

ProGrad is the clearest example. It uses zero-shot CLIP predictions as a knowledge anchor and checks whether the few-shot task gradient agrees with that anchor. If the two gradients conflict, the update is filtered before it reaches the prompt parameters.

Why it matters: this turns regularization into a concrete geometric rule. The prompt is allowed to adapt, but not by breaking the alignment that made CLIP useful in the first place.

Double-Softmax Prompt Tuning follows the same conservative instinct from a different angle. Instead of explicit gradient projection, it changes the loss so that very confident predictions that clash with noisy labels produce much smaller updates. Robust Prompt Tuning reaches a related outcome by freezing class-name semantics and tuning only shared soft prompt tokens, so noise has less room to rewrite the label space itself.

When Labels Are Missing, Borrow Another Signal

Weak supervision does not always mean bad labels. Sometimes it means not enough labels, or no labeled images at all.

TaI-DPT teaches an important lesson here. Because CLIP already aligns image and text embeddings, descriptive text can act as surrogate supervision. The method learns separate global and local prompts from text features, then applies those prompts back to images at test time. The interesting idea is not just “use text.” It is that a vision-language model sometimes lets adaptation move to the cheaper modality without unfreezing the backbone.

GRIP handles a different scarcity regime. Instead of treating pseudolabels as a one-time substitute for labels, it treats them as an iterative curriculum. The model starts from its safest top-KKK guesses, learns prompts, relabels the target pool with the improved model, and expands coverage over multiple rounds. The key mechanism is not a single loss trick. It is controlled growth of supervision so the prompt does not learn from the noisiest unlabeled examples too early.

These two methods illustrate a shared lesson: when direct supervision is thin, the prompt should absorb structure from whichever auxiliary signal preserves class semantics best, whether that signal is language or the model’s own most reliable predictions.

How to Spend a Tiny Prompt Budget

Another design axis appears after supervision is stabilized: what should a prompt-only adapter actually look like?

ECO treats prompt capacity as a budget that can be split across multiple short prompts instead of one long prompt. That makes prompt diversity the main resource. PRE spends capacity differently. Rather than multiplying prompts, it reparameterizes them through a lightweight encoder so prompt tokens interact before the frozen text encoder reads them. One strategy buys diversity. The other buys structure.

FPP shifts the question one step earlier. If test-time prompt tuning is unstable, maybe the prompt is starting from a brittle region. FPP therefore pretrains the initial prompt toward a flatter region while preserving semantic alignment, so later test-time updates begin from a safer basin.

These methods are not just small architectural variations. They answer a practical question that shows up often in CLIP adaptation: if you refuse to fine-tune the encoders, where should the limited adaptation capacity go?

What the Evidence Actually Supports

The compact input gives the strongest direct evidence for ProGrad, so the safest claim is also the narrowest one: alignment-aware prompt regularization improves few-shot CLIP adaptation in practice.

  • ProGrad raises average few-shot accuracy from 73.49 to 74.28 at 16 shots.
  • At 1 shot, ProGrad reaches 62.61 versus 59.46 for CoOp and 58.77 for zero-shot CLIP.
  • For base-to-new generalization, ProGrad reports a 69.06 harmonic mean versus 65.58 for CoOp.
  • In domain generalization, ProGrad beats CoOp on ImageNet source and all four shifted targets.
  • TaI-DPT shows that text-only prompt learning can materially improve over a zero-shot CLIP baseline in multi-label recognition, with 88.3 mAP on VOC2007 versus 77.3 for the zero-shot baseline under its dual-grained prompting setup.

That does not prove one universal recipe. It supports a more useful principle: prompt tuning works better when the update path is filtered, anchored, or restructured so the adaptation signal cannot easily wipe out pretrained alignment.

What to Remember

Supervised prompt optimization for CLIP is not mainly about inventing better prompt tokens. It is about deciding which supervision deserves trust at the prompt interface.

If labels are scarce, borrow stability from zero-shot predictions or iterative pseudolabel refinement. If labeled images are missing, lean on the text side of the shared embedding space. If labels are noisy, constrain the update so a strong frozen prior is harder to overwrite. If prompt capacity is limited, spend it on diversity, structure, or initialization quality instead of assuming a single soft prompt is enough.

The durable takeaway is that parameter-efficient adaptation succeeds when it stays humble. The backbone already knows a lot. The prompt should learn to steer it, not argue with it.
