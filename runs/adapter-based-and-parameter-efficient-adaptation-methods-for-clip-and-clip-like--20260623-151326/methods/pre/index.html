Prompt Learning with Reparameterization Encoder

PRE is a small architectural change with a very specific goal: soft prompt tokens should not be treated like isolated knobs. They should be allowed to interact, form a short sequence, and then hand a cleaned-up version of that sequence to CLIP’s frozen text encoder. CoOp learns prompt vectors directly. PRE inserts a trainable reparameterization step between those vectors and the text encoder so the prompt can absorb domain-specific structure before CLIP turns it into text features.

At a glance, the data flow is simple:

learnable prompt tokens -> prompt encoder -> reparameterized prompt tokens -> class token appended -> frozen CLIP text encoder -> image-text similarity -> class prediction

What to notice in the figure is the extra block in the middle. PRE does not replace CLIP’s text branch, and it does not train a new classifier head. It gives the prompt tokens one lightweight place to coordinate with each other before the frozen encoder reads them. That is the whole trick.

Why direct soft prompts can overfit

The paper starts from a familiar few-shot problem. CoOp replaces hand-written templates with learnable context vectors, which is already much better than searching for the perfect sentence by hand. But those learned vectors are optimized directly against a tiny base-class training set. With so little data, they can become too specific to the seen classes and too brittle for novel classes.

The intuition in PRE is that prompt tokens should not be independent. A good context token for one position often depends on the other tokens around it. If you let the prompt tokens talk to each other first, the final prompt can carry more of the domain structure that few-shot data reveals, instead of just memorizing a few base-class boundaries.

The prompt is rewritten before CLIP reads it

The important move is not to change the frozen CLIP backbone. The change happens one step earlier. The learnable prompt tokens are first passed through a trainable prompt encoder, and only then are they concatenated with the class token and sent into the text encoder. The output is still a standard CLIP-compatible text feature, but it is produced from a prompt sequence that has already been contextually adjusted.

Purpose: show that PRE rewrites the whole learned prompt into a new sequence before the class token is attached.

Equation:

V~=[F(v1),F(v2),…,F(vM)]\tilde{\mathbf{V}} = [\mathcal{F}(\mathbf{v}_1), \mathcal{F}(\mathbf{v}_2), \ldots, \mathcal{F}(\mathbf{v}_M)]V~=[F(v1​),F(v2​),…,F(vM​)]

Intuition: each prompt token keeps its position, but its value is now informed by a trainable encoder instead of being a raw independently learned vector.

Symbols:

  • V\mathbf{V}V: the original set of learnable soft prompt tokens.
  • vi\mathbf{v}_ivi​: the iii-th prompt token embedding.

Read-aloud meaning: the prompt encoder turns the original soft prompt tokens into a new, context-aware prompt sequence.

Why it matters: CLIP still receives a prompt of the same basic shape, so the method stays lightweight, but the content of that prompt is now shaped by a trainable sequence model.

The paper writes the reparameterization as a residual update.

Formula card: residual prompt update

Purpose: keep the learned prompt editable without erasing what the original prompt already knows.

Equation:

F(vi)=φ(vi)+vi\mathcal{F}(\mathbf{v}_i)=\varphi(\mathbf{v}_i)+\mathbf{v}_iF(vi​)=φ(vi​)+vi​

Intuition: the encoder proposes a correction, and the residual path preserves the original token as an anchor.

Symbols:

  • vi\mathbf{v}_ivi​: the original iii-th prompt token.
  • φ\varphiφ: the trainable prompt encoder that produces the learned correction.
  • F\mathcal{F}F: the final transformed token used by PRE.

Read-aloud meaning: each prompt token becomes its original value plus a learned adjustment.

Why it matters: the residual path is what lets PRE adapt the prompt while keeping CLIP’s pretrained semantics in play. Without it, the prompt encoder has to do all the work itself, which makes the system easier to damage in few-shot training.

The paper instantiates φ\varphiφ as a one-layer BiLSTM, so the learned correction is context-aware in both directions. That is why Figure 2 matters.

What to notice in the BiLSTM diagram is that the prompt encoder is not a deep tower of layers. It is a shallow sequence model. The forward pass and backward pass let prompt tokens influence one another in both directions, so the correction for a token can depend on the rest of the prompt, not just on that token alone.

This is the key reason BiLSTM is a better fit than a plain MLP. An MLP can transform each token, but it does not naturally model the prompt as a sequence with dependencies. The paper’s ablation shows that the BiLSTM is the strongest option among the tested encoders, while the MLP tends to lose more Base-class accuracy. The transformer encoder follows a similar trend but sits slightly below BiLSTM.

In plain language, PRE asks: if the prompt is really a short sentence, why should we tune its words independently?

From the revised prompt to the class score

Once the prompt encoder has rewritten the context tokens, PRE attaches the class token and sends the whole prompt through CLIP’s frozen text encoder. That produces a class-specific text feature. The image branch is still frozen too. So at inference time, the model is still doing the same thing CLIP always does: compare image and text features in a shared space, then pick the closest class.

Intuition: score every class by image-text similarity, then normalize those scores into probabilities.

Symbols:

  • x\mathbf{x}x: the input image.
  • f\mathbf{f}f: the image feature from the frozen CLIP image encoder.
  • tir\mathbf{t}_i^rtir​: the class-specific text feature produced from the reparameterized prompt for class iii.

Read-aloud meaning: PRE predicts a class by comparing the image feature with each reparameterized class prompt and softmaxing the similarity scores.

Why it matters: the classifier rule itself does not change. PRE only improves the text features that feed into CLIP’s familiar matching process, which is why the method stays easy to slot into an existing CLIP pipeline.

Formula card: training objective

Purpose: train the prompt embeddings and the prompt encoder on few-shot base-class data.

Equation:

L=−∑c=1Cyclog⁡P(y=c∣x)\mathcal{L}=-\sum_{c=1}^{C} y_c \log P(y=c \mid \mathbf{x})L=−c=1∑C​yc​logP(y=c∣x)

Intuition: increase the probability of the correct class while reducing the probability of the wrong ones.

Symbols:

  • L\mathcal{L}L: the cross-entropy loss.
  • CCC: the number of classes.
  • ycy_cyc​: the target indicator for class ccc, with 111 for the true class and 000 for the others.
  • P(y=c∣x)P(y=c \mid \mathbf{x})P(y=c∣x): the predicted probability for class ccc from the previous scoring step.

Read-aloud meaning: the model learns by pushing the correct class probability up and the incorrect ones down.

Why it matters: gradients flow through the frozen text encoder during backpropagation, but only the prompt side learns new parameters. That keeps adaptation parameter-efficient and preserves CLIP’s pretrained backbone.

Why the residual path matters more than it looks

The ablation section makes an important point: the skip connection is not a cosmetic choice. When the residual path is removed, Base, New, and harmonic-mean performance all drop, regardless of whether the encoder is an MLP, a transformer, or a BiLSTM. That tells you the learned correction is useful only when it is allowed to blend with the original prompt embedding.

The residual path also explains why PRE does not collapse into a brittle prompt rewrite. It can preserve CLIP’s original semantic prior while still nudging the prompt toward the target domain. In few-shot settings, that balance is often the difference between a prompt that generalizes and one that memorizes.

What the numbers say

The paper’s main evidence matches the mechanism. In the 16-shot setting, PRE raises average New-class accuracy over CoOp from 67.14% to 71.90%, and it improves the harmonic mean by about 3 points. BiLSTM is the strongest tested prompt encoder, which fits the story above: prompt tokens benefit from a sequence model that can model their dependencies instead of treating them as separate vectors.

The gains are not unlimited, though. They shrink as shot count decreases, which makes sense because the prompt encoder needs enough data to learn useful token interactions. The paper also reports that PRE still trails zero-shot CLIP on New classes in 5 of 8 datasets. So PRE is best read as a strong few-shot adaptation method, not a universal replacement for CLIP’s original prior.

What to remember

  • PRE does not tune the prompt tokens directly and stop there. It rewrites them through a small trainable encoder first.
  • The residual connection is the safety rail. It lets the prompt learn new structure without throwing away CLIP’s pretrained semantics.
  • BiLSTM is the paper’s best encoder choice because it models prompt-token dependencies, not just token-wise transformations.
  • The method stays lightweight because CLIP’s image and text encoders remain frozen, and only the prompt side learns.
  • The tradeoff is real: the method helps most when you have enough few-shot data, and it does not beat zero-shot CLIP everywhere.
