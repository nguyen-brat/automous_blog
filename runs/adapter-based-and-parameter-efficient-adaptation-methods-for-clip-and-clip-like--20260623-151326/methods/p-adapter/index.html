p-Adapter: Graph-Calibrated Adapter Tuning for Vision-Language Models

p-Adapter starts from a practical suspicion: the attention block inside a vision-language transformer is not always connecting similar things. Query features and value features can sit in different clusters, so an edge can join two nodes that are meant to exchange information, not to be averaged into the same thing. That makes the attention graph heterophilic. If a vanilla adapter trusts the raw attention weights and then applies a standard bottleneck projection, it can smooth away the distinctions that matter. p-Adapter keeps the backbone frozen, leaves the ordinary adapter in place, and adds one extra step right after attention so the adapter sees a better mixed feature.

Start with the picture, not the algebra

The architecture figure is the cleanest place to see the method.

Read the figure from bottom to top. Self-attention or cross-attention happens first, then p-Adapter recalibrates the attention-derived edges, then the usual adapter projection runs. The right-hand zoom shows the internal order: attention output, feature-aware re-normalization, aggregation, down-projection, non-linearity, up-projection, and residual add. The important detail is not that the block is bigger. It is that the block changes the mixing rule before the adapter does its usual compression and expansion.

Notice two things in the diagram. First, p-Adapter sits on the attention side, not as a replacement for the whole transformer block. Second, the standard adapter after the feed-forward network is still there. p-Adapter is an extra corrective step, not a new backbone.

Why attention needs graph thinking

The paper’s main move is to treat adapter tuning after attention as graph message passing. That sounds abstract, but the intuition is simple.

Attention already gives you a connectivity pattern: some query token points more strongly to some value token, and weakly to others. If you read those weights as graph edges, then the projected query and value features become node features. The problem is that the raw attention matrix is not yet a clean graph adjacency. In self-attention it is not naturally symmetric, and in cross-attention it can even be rectangular because the number of query tokens and value tokens can differ.

So the paper uses an augmented graph trick. It stacks the projected query features above the projected value features, then connects the two halves with an off-diagonal attention matrix. In other words, it turns the original attention map into a bipartite graph where query nodes live on one side and value nodes live on the other. If there are 3 query tokens and 5 value tokens, the augmented graph has 8 nodes instead of forcing a 3-by-5 matrix into a graph operator that wants a square adjacency.

This is a bookkeeping trick, but it is a very useful one. Once attention becomes a square, symmetric graph, the adapter can be read as one graph propagation step followed by the usual adapter projection. The original attention behavior is not discarded; it is preserved on the query side of the augmented graph.

The edge weight is where the real fix happens

Vanilla graph convolution assumes a homophilic world: neighbors tend to look alike, so smoothing makes sense. p-Adapter is built for the opposite case. Here, a query node and a value node may be linked precisely because they are complementary, not because they should be blurred together. That is why the paper borrows the p-Laplacian idea from graph learning and uses feature differences to re-normalize attention edges.

The key idea is that the edge weight is no longer just “how much attention did the model assign?” It also becomes “how trustworthy is this edge given the two features it connects?” That is a stronger question for a heterophilic graph.

What the exponent p is really doing

The exponent ppp is the control knob for how aggressively the graph re-normalizes the attention edges. A fixed ppp works, but it is too blunt if every layer sees a different kind of attention geometry. The paper therefore makes ppp learnable, and in practice it learns a separate value per layer.

That choice is not cosmetic. The ablation shows that learnable ppp beats the fixed settings the authors tried, with an average score of 80.27 versus roughly 80.02 to 80.08 for the fixed variants. The gain is not huge on paper, but it is consistent, and consistency matters when the whole point is to fix the mixing rule rather than to add a lot of new capacity.

The paper’s p-Laplacian analysis explains why this is sensible. When ppp is smaller than 2, the operator becomes more sensitive to feature differences instead of blindly smoothing everything. That is exactly what you want if some attention edges join nodes that are intentionally far apart in feature space. A shared fixed exponent is therefore too rigid for a stack of transformer layers that do not all have the same attention structure.

The same logic also explains one of the simpler ablations: padding the augmented graph with zeros or random noise is worse than concatenating the real query features. If the normalization step is supposed to understand the difference between the two sides of the attention graph, fake filler does not help. The graph needs actual feature content so it can tell what kind of edge it is calibrating.

Follow the data flow through the block

Seen as a pipeline, p-Adapter does five things:

  1. It takes the frozen attention output from the backbone block.
  2. It builds an augmented bipartite graph by stacking query and value features.
  3. It reweights the attention edges with the feature-difference term from the p-Laplacian idea.
  4. It aggregates the features with a residual mix so the original signal is not lost.
  5. It runs the usual down-projection and up-projection adapter on the calibrated feature.

The output you pass onward is the query-side result, because that is the part the transformer expects. The value-side nodes are there to make the graph message passing meaningful, not to become a new downstream representation.

This is also why the method stays compatible with a standard adapter. p-Adapter does not replace the feed-forward adapter path; it only changes the attention-side mixing. That is a useful design choice because it keeps the model modular and preserves the original adapter behavior where the paper did not see a special gain from changing it.

The insertion ablation makes that point clearly. When the method is attached only after the feed-forward network, p-Adapter behaves like a plain adapter. The real improvement comes from the attention side, especially when the method is inserted after self-attention and cross-attention together.

What the experiments actually say

The paper’s results match the design story. With BLIP, p-Adapter tunes only 6.39% of the parameters and reaches 70.39 on VQA2.0, 79.40 on SNLI-VE, and 130.9 CIDEr on COCO Captions. On the same backbone, it beats the vanilla adapter by a clear margin. With mPLUG, the pattern repeats: p-Adapter reaches 71.36 on VQA2.0, slightly above full fine-tuning.

The exact benchmark numbers are not the main lesson. The lesson is that a small, attention-side correction can get surprisingly close to full fine-tuning when the real bottleneck is the mixing operator rather than the amount of trainable capacity.

There are also two limits worth keeping in mind. First, the paper validates on generative vision-language models such as BLIP and mPLUG, not on every CLIP-style backbone. So transfer to dual-encoder CLIP variants is still an open question. Second, the graph reformulation adds conceptual and implementation complexity compared with a plain adapter. p-Adapter pays that complexity cost because the authors believe the attention graph is the wrong shape for naive smoothing.

What to remember

p-Adapter is easiest to remember as a graph-aware correction for attention-side adapter tuning. The frozen backbone still does the heavy lifting, but the method refuses to trust raw attention weights as if every edge were a homophilic one. It first recalibrates those edges with feature-aware p-Laplacian normalization, then lets the ordinary adapter project the cleaned-up signal.

If you want the one-sentence version, it is this: when attention connects mismatched feature spaces, do not just add a small adapter on top of it, change the mixing rule that feeds the adapter.
