CLIP-Adapter: Better Vision-Language Models with Feature Adapters

CLIP-Adapter is easiest to understand as a refusal to overcorrect. CLIP already gives you a strong image-text alignment prior, so with only a handful of labels per class the safest move is not to rebuild the model or spend all your effort on prompt engineering. The paper instead learns a tiny residual correction on top of frozen CLIP features, then mixes that correction back into the original representation.

This first figure shows the core tradeoff. CoOp spends its effort on the prompt side. CLIP-Adapter keeps CLIP frozen and puts the task-specific change into a small feature adapter after the encoder. That is the whole philosophy of the paper: when the pretrained model is already good, learn a correction instead of rewriting the whole system.

Start from CLIP’s normal loop

To see why the method is interesting, it helps to remember how CLIP does classification in the first place. An image goes through the visual encoder and becomes a feature vector. A class name wrapped in a prompt goes through the text encoder and becomes the class weight for that label. Prediction then comes from how well the image feature matches each class weight.

What matters in this comparison is where the change happens. In plain CLIP, both encoders are frozen and the prompt is fixed. In CoOp, the prompt itself becomes trainable. In CLIP-Adapter, the encoders still stay frozen, but the feature that comes out of them is allowed to pass through a small adapter before classification. The paper is moving the learning problem from “change the prompt” to “correct the representation.”

The adapter is a correction term

The adapter itself is deliberately small. It is a two-layer bottleneck MLP: compress the feature, apply a nonlinearity, then expand it again. That narrow middle layer is not a cosmetic detail. It keeps the module from becoming a second backbone and forces it to learn a compact correction rather than memorize the few-shot set. The paper uses a hidden size of D/4, which is small enough to stay cheap but large enough to preserve useful semantics.

The residual blend

Purpose: Keep the pretrained CLIP feature as the anchor and add only a task-specific correction.

Equation:

f⋆=αAv(f)T+(1−α)ff^{\star}=\alpha A_v(f)^T+(1-\alpha)ff⋆=αAv​(f)T+(1−α)f

Intuition: The adapter proposes a correction; alpha decides how much to trust it. When alpha is 0, you recover zero-shot CLIP. When alpha is 1, you trust only the adapted feature, which can overfit.

Symbols: f is the original CLIP image feature. A_v(f) is the output of the visual adapter. A_v(f)^T is the same adapted feature written in the paper’s shape convention so it can be added back to f. alpha is the residual mixing ratio. f^\star is the blended feature used for classification.

Read-aloud meaning: Take the original feature, add a scaled adapter correction, and use the mixture as the final feature.

Why it matters: This is the mechanism that preserves CLIP’s zero-shot knowledge while still letting few-shot training move the representation toward the new task.

The text branch uses the same idea with beta, but the default variant in the paper adapts the visual branch only. That choice is not arbitrary. The experiments suggest that the image side usually needs the bigger correction, while the text side is already carrying useful class semantics through the prompt.

Why the design stays small and late

CLIP-Adapter’s most important design choices are all about not disturbing the backbone more than necessary.

The adapter sits after the final encoder layer. The paper tests earlier insertion points and finds that the end-of-encoder version is both cheaper and stronger. That makes sense if you think about what CLIP is already doing well. The lower layers build generic features; the final embedding is where the class separation actually matters. If the backbone is already strong, the safest place to adjust it is near the output.

The bottleneck size matters for the same reason. A tiny adapter keeps the update focused. It gives the model enough freedom to reshape the feature space for the new task, but not enough freedom to become a new model that overwrites the pretrained prior.

The residual ratio, alpha or beta, is the other safety valve. It decides how much to trust the adapter versus the frozen CLIP feature. Small values mean “mostly keep CLIP.” Larger values mean “lean harder on the new correction.” In the paper, the best ratio varies with the dataset: specialized, fine-grained tasks usually want more adaptation, while generic datasets need a lighter touch.

The paper also explores learning alpha and beta with a hypernetwork. That is a practical idea if you do not want to tune the balance by hand for every dataset. The core intuition stays the same either way: the pretrained feature should remain the default answer, not a nuisance to be discarded.

Training is straightforward. Freeze CLIP, train only the adapter weights, and use CLIP’s contrastive objective on the few-shot examples. Because the backbone never changes, the update stays cheap and avoids the slow full forward-backward path through every encoder layer.

What the experiments teach

The empirical story lines up with the design story.

Across eleven datasets and shot counts from one to sixteen, CLIP-Adapter beats zero-shot CLIP, linear-probe CLIP, and CoOp. The interesting part is not only the average gain. The method helps most when labels are scarce, which is exactly when full fine-tuning is most likely to overfit.

A few other patterns are especially useful to remember:

  • Visual-only adaptation is stronger than text-only adaptation, and using both together does not reliably help.
  • Final-layer insertion is better than earlier or dense insertion, which supports the idea that the correction should be made near the output geometry.
  • The method gives a strong cost-accuracy tradeoff. On ImageNet 16-shot, it improves accuracy over linear probing while using about half as many parameters, and it trains and runs much faster than CoOp because it does not backprop through the whole text encoder.
  • Gains are smaller on generic datasets where zero-shot CLIP is already strong. That is not a failure of the method. It is a reminder that if the pretrained prior is already close, there is less room to improve.

The residual ratio also reflects task type. The more the downstream data differs from CLIP’s pretraining distribution, the more the method benefits from trusting the adapter. When the downstream task is already close to CLIP’s world, the original feature should keep most of the vote.

What the paper leaves open

The main open problem is still the same one that appears in many adapter-style methods: how do you pick the residual ratio without tuning it per dataset? The paper shows that the right value changes with task shift, which is useful intuition but not a universal rule.

A second open question is why text and visual adapters can interfere when trained together. The paper suggests they may overlap in what they correct, but that is still only a partial explanation.

A third limitation is scope. The evidence is strongest for few-shot classification. It is plausible that the same residual idea transfers to broader vision-language tasks, but the paper does not settle that.

What to remember

CLIP-Adapter treats CLIP’s pretrained feature as a strong default answer and learns a small correction instead of replacing it. That simple shift explains almost everything about the method: the bottleneck size, the final-layer placement, the residual mix, the visual-first default, and the efficiency gains.

If you remember one sentence, make it this: when the pretrained encoder is already strong, adapt its output space with a small residual module and keep the original knowledge in the loop.
