CMAP: Cross-Modal Adaptive Prompting

CMAP is easiest to understand as a refusal to let CLIP adapt like a vision-only model. The paper keeps the CLIP backbone frozen, but it stops treating the image tower as the only place where adaptation decisions should happen. Instead, it uses the text side as a semantic map for task identity, combines visual and textual evidence to judge whether an input is trustworthy, and then opens or closes prompts in both encoders together.

What to notice in the figure is where the control lives. The frozen encoders are still the backbone, but the method inserts small trainable controls at every important decision point: task routing, confidence calibration, and symmetric gating. The snowflake icons in the paper mark the frozen CLIP pieces. The flame icons mark the trainable bits. CMAP’s whole argument is that the trainable bits should steer both modalities, not just the image branch.

Why the text side matters for routing

The routing problem sounds simple, but it is one of the easiest places to lose performance in task-incremental learning. The model sees a new image, but it does not know which task it belongs to. A common baseline is to fit a visual Gaussian for each task and then pick the task whose visual distribution best explains the image.

That works only as long as the visual statistics are stable and there is enough data to fit them well. In the data-scarce setting, or when tasks arrive in a different order, those fitted distributions can become brittle. CMAP takes a different route: it uses CLIP’s text embeddings as the task map. Those embeddings are frozen, semantically stable, and independent of the visual samples that happened to arrive first.

Think of it like this. A visual Gaussian is a sketch made from whatever examples were on the table during training. A text prototype is a semantic label map that already exists before training starts. CMAP prefers the map.

That last point is easy to miss but important. CMAP is not learning a richer classifier just to solve routing. It is taking advantage of a part of CLIP that already knows how to line up class names and images, and it uses that structure as the routing signal.

Confidence should listen to both views

After the model chooses a task, it still needs to decide how strongly to trust the sample. CMAP does not want to apply the same prompt strength to every image. A sample that sits comfortably inside the task distribution should get more prompting than one that looks ambiguous or out of distribution.

The paper’s answer is to give each class two kinds of evidence. One comes from the visual side. But instead of forcing each class into a single Gaussian, CMAP uses multiple visual centroids. The paper sets this to K=3K=3K=3, which is just enough to let a class have a few visual modes without turning the method into a full memory bank. The other comes from the text side: the image is compared directly with the class name embedding in CLIP space.

This matters because one class can show up in several visual forms. An aircraft, for example, can appear from the side, from the front, close up, or far away. A single center is too rigid for that. Multiple prototypes give the class a few places to live.

Formula Card: multi-prototype visual-textual confidence

Purpose: Score a class by combining its best visual prototype match with its image-text alignment.

Equation:

Read-aloud meaning: Measure class confidence from both the class’s visual neighborhoods and its text meaning, then average the two scores.

Why it matters: Visual-only confidence can be fooled by view changes or sparse training data, and textual-only confidence can be too coarse by itself. CMAP uses both because they capture different parts of the same decision.

Once class scores are available, CMAP does not trust a single top class score blindly. It averages the top five class scores inside the selected task to get a task-level confidence. That smooths out spiky false positives and makes the decision depend on the overall shape of the task’s candidate set, not on one lucky match. Then the model calibrates a task-specific upper and lower threshold from the training distribution. The thresholds are not fixed globals; they are the 80th and 20th percentiles of the task’s confidence values.

That calibration step is one of the paper’s most practical choices. It prevents the confidence logic from becoming a fragile global heuristic. Hard tasks can tolerate a different confidence scale than easy tasks, and CMAP lets that difference show up in the prompt weight.

The gates have to agree

The other big idea is that prompting should not happen on only one side of CLIP. Earlier methods often gated the image encoder while leaving the text encoder on a fixed prompt depth. That can work when the input is clean, but it creates a mismatch on out-of-distribution samples: the image side may decide to fall back toward zero-shot CLIP, while the text side still carries task-specific prompting. The two towers stop speaking the same language.

CMAP fixes that by adding symmetric gating on the text encoder. The text gate is conditioned on the batch-mean image feature, so the current mini-batch provides a shared summary of how trustworthy the input looks. In effect, the image and text branches read the same weather report before deciding whether to open the prompt valves.

Why it matters: If the image side is deciding to back off, the text side should back off too. CMAP keeps both encoders open or closed together, which preserves cross-modal coherence instead of letting one branch drift ahead of the other.

That symmetry is the real punchline. The paper is not just adding more prompt machinery. It is making the control machinery itself cross-modal.

What the full pipeline does

At test time, CMAP runs as a short sequence of decisions. First, the image is encoded with frozen CLIP features. Next, the sample is routed to a task using cosine similarity against the task’s frozen text prototype. Then the chosen task’s classes are scored using the multi-prototype visual-textual confidence. That confidence becomes a prompt weight through task-specific thresholds. Finally, the image and text encoders use synchronized gates and the class prediction is produced from cosine similarity between image and text features.

Training stays in the same low-budget regime as the rest of the family. The backbone is frozen, there is no memory buffer, and the extra trainable parameters are small. The paper reports about 2.5M trainable parameters in total. That is still tiny compared with full finetuning, but the extra pieces are doing real work: they are not just more weights, they are control signals.

What the evidence says

The strongest evidence is the pattern, not the table. On Order-I, CMAP reports 73.6% Transfer, 81.4% Average, and 88.9% Last. Removing text routing hurts Transfer the most, which fits the design: the biggest win comes from routing with text prototypes instead of visual Gaussians. Removing symmetric gating hurts Average and Last, which fits the second design claim: retention depends on keeping the two encoders in sync, not on prompting only one side.

The confidence ablation tells the same story. Visual-only confidence and text-only confidence both trail the equal-weight combination, which is exactly what you would hope if the two views are complementary rather than redundant. The paper’s message is not that text should replace vision. It is that routing, confidence, and gating should all use both modalities when the backbone itself is cross-modal.

Where to be careful

CMAP is a strong fit for classification-style multi-task incremental learning, but its assumptions are visible.

  • It depends on informative class names and stable text embeddings.
  • It is evaluated mainly on classification-style MTIL benchmarks, so the same recipe is less obviously tested for retrieval or other non-classification vision-language tasks.
  • It adds extra calibration and K-means prototype construction steps, so the pipeline is a little more involved than plain prompt tuning.

Those limits do not weaken the method, but they do define its boundary. CMAP is best read as a cross-modal control recipe for CLIP-style continual learning, not as a universal adapter for every vision-language workload.

What to remember

CMAP’s core lesson is simple:

  • use frozen text prototypes to choose the task,
  • use both visual prototypes and image-text alignment to judge confidence,
  • and gate both encoders together so CLIP stays coherent while it adapts.

If you want one sentence to carry around, make it this: CMAP treats text as the semantic map, then lets both encoders adapt together only when the input looks trustworthy enough to justify it.
