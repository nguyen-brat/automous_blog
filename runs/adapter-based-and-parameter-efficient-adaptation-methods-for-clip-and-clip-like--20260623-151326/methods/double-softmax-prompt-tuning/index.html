Double-Softmax Prompt Tuning (DSPT)

DSPT is a very small change with a very specific target: CLIP prompt tuning under noisy labels. The backbone stays frozen. Only the prompt tokens learn. Before cross-entropy sees the prediction, DSPT sends the logits through softmax twice. That extra pass turns the loss into a shock absorber. Highly confident mismatches calm down, while ambiguous samples still contribute useful updates.

The key idea is easier to see as a flow:

The paper is not trying to build a bigger adapter or a smarter sample selector. It is trying to stop a bad label from shouting louder than CLIP’s pretrained prior.

The overview figure makes that contrast concrete.

What to notice here is the difference between a normal prompt-tuning update and the DSPT update. In ordinary training, a sample whose label disagrees with the model’s confidence can create a big corrective push. DSPT bends that push down before it reaches the prompt parameters, so the mismatch sample stops dominating the batch.

Why ordinary prompt tuning breaks on noisy labels

The failure mode is not subtle. Imagine an image of a car that has been mislabeled as dog. CLIP may already be very sure the image looks like a car. Standard prompt tuning still treats the label as ground truth, so it keeps pushing the prompt toward dog and away from car. If enough such samples appear, the prompt can drift away from the pretrained alignment that made CLIP useful in the first place.

The ordinary cross-entropy gradient makes that problem easy to see.

Formula card: standard cross-entropy gradient

Purpose: show why a mislabeled example can produce a strong corrective signal on the logits.

Equation:

∂Lpt∂zi=pi−δy~i,p=softmax⁡(z)\frac{\partial \mathcal{L}_{\mathrm{pt}}}{\partial z_i}=p_i-\delta_{\tilde{y}i}, \qquad p=\operatorname{softmax}(\mathbf{z})∂zi​∂Lpt​​=pi​−δy~​i​,p=softmax(z)

Intuition: compare the model’s predicted probability for class iii with the noisy target indicator. If the model already puts almost no probability on the noisy label, the update still insists on increasing that label and decreasing the classes the model prefers.

Symbols:

  • Lpt\mathcal{L}_{\mathrm{pt}}Lpt​: the ordinary prompt-tuning loss.
  • z\mathbf{z}z: the logit vector over classes for one sample.
  • ziz_izi​: the logit for class iii.
  • pip_ipi​: the first-softmax probability for class iii.
  • y~\tilde{y}y~​: the noisy label.
  • δy~i\delta_{\tilde{y}i}δy~​i​: an indicator that is 1 when class iii equals the noisy label and 0 otherwise.

Read-aloud meaning: the update for each class is the model’s probability minus the label indicator.

Why it matters: the gradient does not know which labels are wrong. In CLIP prompt tuning, that means mislabeled samples can keep steering the prompt in a direction that fights the pretrained prior.

The important detail is not that one scalar term is huge. The important detail is that the update repeatedly points against the model’s best prior knowledge on samples that are likely wrong. That is how noisy labels become a training problem.

The actual trick: soften the prediction twice

DSPT changes only the map from logits to loss. There are no extra parameters, no auxiliary branch, and no threshold to tune. The prompt still sees the same frozen CLIP features. The only difference is that the model does not hand its raw probability vector straight to cross-entropy. It softens it once more first.

Formula card: double-softmax loss

Purpose: define the drop-in loss change that makes high-confidence noisy samples saturate instead of dominating the prompt update.

Equation:

Lours=−1n∑i=1nlog⁡(softmax⁡(softmax⁡(zi))y~i)\mathcal{L}_{\mathrm{ours}}=-\frac{1}{n}\sum_{i=1}^{n}\log\left(\operatorname{softmax}\left(\operatorname{softmax}(\mathbf{z}_i)\right)_{\tilde{y}_i}\right)Lours​=−n1​i=1∑n​log(softmax(softmax(zi​))y~​i​​)

Intuition: turn logits into probabilities, soften those probabilities again, and then score the noisy target with cross-entropy.

Symbols:

  • Lours\mathcal{L}_{\mathrm{ours}}Lours​: the DSPT training loss.
  • zi\mathbf{z}_izi​: the class-logit vector for sample iii.
  • y~i\tilde{y}_iy~​i​: the noisy label for sample iii.
  • nnn: the number of samples averaged in the loss.

Read-aloud meaning: for each sample, normalize the logits, normalize the normalized probabilities again, and then compute cross-entropy on the noisy label.

Why it matters: this second normalization is the whole method. It creates a conservative response curve without introducing a new network block or a hand-tuned clipping threshold.

This is why the paper frames DSPT as a loss-side intervention rather than a new architecture. The backbone stays frozen, the prompts still learn, and the loss itself becomes the gatekeeper.

The second softmax is doing more than “smoothing” in the generic sense. It compresses extremes. If the first softmax is already very sharp, the second softmax flattens the response enough that additional confidence does not keep producing larger and larger pressure on the prompt. If the sample is only ambiguous, the response is softened, but not erased.

That is the key distinction between DSPT and a blunt regularizer. It does not say “trust every label less.” It says “let very suspicious samples speak more quietly.”

When gradient vanishing becomes a noise filter

The paper’s theory turns that intuition into a clean limit statement. If the model is already extremely confident in some class y^\hat{y}y^​ and that class disagrees with the noisy label, the total gradient on the logits goes to zero.

Purpose: express the saturation zone where a highly confident mismatch stops steering the prompt.

Equation:

lim⁡py^→1, y^≠y~∑i∣∂Lours∂zi∣=0\lim_{\mathbf{p}_{\hat{y}}\to 1,\ \hat{y}\neq \tilde{y}} \sum_i \left| \frac{\partial \mathcal{L}_{\mathrm{ours}}}{\partial z_i} \right| = 0py^​​→1, y^​=y~​lim​i∑​​∂zi​∂Lours​​​=0

Intuition: when the model is already almost certain that the noisy label is wrong, the double-softmax loss makes that sample’s total gradient disappear.

Symbols:

  • p\mathbf{p}p: the first-softmax probability vector.
  • py^\mathbf{p}_{\hat{y}}py^​​: the probability assigned to the model’s most confident class.
  • y^\hat{y}y^​: the class the model is most confident in.
  • y~\tilde{y}y~​: the noisy label.
  • ziz_izi​: the logit for class iii.

Read-aloud meaning: if the model is nearly sure of one class and that class is not the noisy label, the total gradient becomes negligible.

Why it matters: this is the saturation zone in formal form. It explains why DSPT can mute mislabeled, high-confidence samples while still leaving room for informative, uncertain samples.

The paper’s gradient plots line up with that claim. On noisy Caltech101, the mismatched high-confidence samples are pushed close to zero gradient, while clean samples still keep usable updates. That is exactly what you want if CLIP already has a strong prior and the labels are the unreliable part.

The paper also proves that the loss stays inside a finite range and that the clean/noisy optima stay within bounded risk gaps under symmetric and asymmetric label noise. The exact constants are less important than the shape of the result: hard samples cannot blow up the objective, and noisy supervision cannot drag the optimum arbitrarily far away from the clean one.

Why this is simpler than clipping or selection

DSPT is easiest to appreciate when you compare it with the nearest alternatives.

LogitClip also tries to stop extreme samples, but it does so with a norm threshold. That threshold has to be chosen, and the paper shows that the right value can move around across datasets and noise rates. In other words, the method becomes another calibration problem.

NLPrompt takes a different route. It tries to separate likely clean and likely noisy samples, then handles them differently. That is more explicit, but also more machinery. DSPT avoids the split entirely. It does not decide which examples are clean. It just makes the suspicious ones much quieter.

That makes DSPT a good fit for the setting the paper cares about: a confident pretrained VLM that already has a useful prior, so the safest move is often conservative adaptation rather than aggressive correction.

The tradeoff is equally important. If the backbone is weak, or if training starts closer to scratch, this same smoothing can suppress useful learning too much. The paper is honest about that limitation. DSPT is designed for strong pretrained vision-language models, not for generic from-scratch training.

What the experiments actually support

The best summary is not a leaderboard table. It is a pattern.

Across nine datasets, DSPT usually has the best average accuracy in the tested noise settings. Two concrete anchors make the pattern easy to remember: on Caltech101 with 60 percent symmetric noise, DSPT reaches 95.31 percent versus 94.42 percent for NLPrompt; on DTD with 80 percent symmetric noise, DSPT reaches 55.50 percent, clearly ahead of CoOp and NLPrompt.

The gradient analysis tells the same story as the accuracy numbers. Clean samples continue to contribute, but mislabeled high-confidence samples stop producing the kind of spike that can overwrite the prompt. That is the central claim of the paper, and the experiments support it reasonably well.

The caveat is that DSPT is not always the winner under moderate noise. In some settings, NLPrompt can still be competitive or better. That is consistent with the method’s design: DSPT is conservative by construction, so if the pretrained prior is less trustworthy or the noise is not extreme, the extra flattening can be a little too cautious.

What to remember

DSPT is a loss-level gate for prompt tuning. It does not add another branch, and it does not try to discover noisy labels explicitly. It simply says that if CLIP is already very sure a label is wrong, that sample should stop steering the adapter so hard.

That is the mental model worth keeping:

  • freeze the backbone,
  • learn only the prompts,
  • apply softmax twice,
  • let suspiciously confident mismatches fade,
  • keep ambiguous samples in the learning loop.
