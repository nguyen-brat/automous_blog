Texts as Images in Prompt Tuning for Multi-Label Image Recognition

TaI-DPT starts from a very practical idea: CLIP already knows how to place text and images in the same embedding space, so why should prompt tuning always need labeled images? If a descriptive sentence already sits near the image it describes, then the sentence can play the role of a surrogate image during prompt learning. That is the core move in this paper. The method learns prompts from text descriptions only, then reuses those prompts on real images at test time.

That sounds abstract until you look at the flow end to end:

The pipeline figure makes the cross-modal trick concrete.

What to notice here is that the same learned prompts are used in both phases. During training, the prompts are matched against text features. During testing, the exact same prompts are matched against image features. CLIP’s alignment does the heavy lifting that makes this possible.

Turning captions into weak supervision

The paper does not try to mine random web text and hope for the best. It uses text sources that are already likely to mention objects: public image captions, such as MS-COCO, and localized narratives from object detection datasets, such as OpenImages. Only the text is used. No image content or labels are exposed to the prompt learner.

The text collection step has two requirements. First, the descriptions need enough scene detail to be useful. Second, the corpus needs to cover the target classes well enough that the prompts can learn discriminative cues for each class.

That is why the method uses a noun filter. It builds a synonym dictionary for the target class set, using WordNet and manual cleanup, then searches the descriptions for class names or close synonyms. A sentence like “a person holding a cellphone” should count for the phone class even if the exact dataset label is “cell phone”. Sentences that do not mention any target class are dropped.

The output is a binary pseudo-label vector for each retained description. If a description mentions several classes, several entries become positive. That is important because TaI-DPT is designed for multi-label recognition, not single-label classification.

This filtering is simple on purpose. It is reproducible and cheap, but it is also imperfect. Paraphrases, misspellings, and rare phrasings can slip through. The paper accepts that tradeoff because it wants a text-only training signal that is easy to reproduce, not a brittle text mining pipeline that only works once.

Why the prompt has two scales

Once the text data is prepared, the method learns a prompt in the usual CLIP style: a few trainable context tokens followed by the class name.

Formula card: the basic prompt template

Purpose: define a prompt as learnable context plus the class token.

Equation:

ti=[v1,v2,…,vM,si]\mathbf{t}_i = [\mathbf{v}_1, \mathbf{v}_2, \ldots, \mathbf{v}_M, \mathbf{s}_i]ti​=[v1​,v2​,…,vM​,si​]

Intuition: prepend a short trainable context to each class name so the model can learn how to phrase the class for CLIP.

Symbols:

  • ti\mathbf{t}_iti​: the prompt for class iii.
  • vj\mathbf{v}_jvj​: the jjjth learnable context token embedding.
  • MMM: the number of context tokens.
  • si\mathbf{s}_isi​: the token embedding of the class name for class iii.

Read-aloud meaning: build each class prompt by placing learned context tokens in front of the class name.

Why it matters: this is the smallest unit of prompt tuning in the paper. Everything else is built on top of this CLIP-style prompt form.

The paper then splits that prompt into two branches. The global branch learns a coarse class embedding from the whole sentence or whole image. The local branch learns a finer class embedding from token-level or patch-level evidence. Think of the global branch as a wide-angle lens and the local branch as a magnifying glass.

Formula card: dual-grained prompts

Purpose: create separate prompt sequences for coarse and fine-grained reasoning.

Equation:

tiG=[v1,v2,v3,…,vM,si],tiL=[v1′,v2′,v3′,…,vM′,si]\begin{split} \mathbf{t}^{G}_i &= [\mathbf{v}_1, \mathbf{v}_2, \mathbf{v}_3, \ldots, \mathbf{v}_M, \mathbf{s}_i], \\ \mathbf{t}^{L}_i &= [\mathbf{v}'_1, \mathbf{v}'_2, \mathbf{v}'_3, \ldots, \mathbf{v}'_M, \mathbf{s}_i] \end{split}tiG​tiL​​=[v1​,v2​,v3​,…,vM​,si​],=[v1′​,v2′​,v3′​,…,vM′​,si​]​

Intuition: learn one prompt branch for whole-scene matching and another prompt branch for local object cues.

Symbols:

  • tiG\mathbf{t}^{G}_itiG​: the global prompt for class iii.
  • tiL\mathbf{t}^{L}_itiL​: the local prompt for class iii.
  • vj\mathbf{v}_jvj​: the learnable context token in the global branch.
  • vj′\mathbf{v}'_jvj′​: the learnable context token in the local branch.
  • si\mathbf{s}_isi​: the class-name token embedding.

Read-aloud meaning: make two prompt versions for every class, one tuned for global evidence and one tuned for local evidence.

Why it matters: multi-label images often contain a dominant object and several secondary objects. A single global prompt can latch onto the dominant object and miss the rest. The local branch is there to rescue those smaller or less obvious classes.

The paper is careful about what this is not. These are not positive and negative prompts like DualCoOp. They are two positive branches with different jobs.

The easiest way to see why that matters is to look at the local branch behavior. The paper’s visualizations show that the local class embedding does not just act like another whole-sentence summary. It aligns with the specific words or regions that name the class.

What to notice in Figure 3 is that the local branch pays attention to the object words, not just the sentence-level <EOS> token. What to notice in Figure 4 is that the same local embedding can focus on the relevant image region at test time. That is the key cross-modal payoff: a text-trained local prompt can still behave like a patch-aware image prompt later.

How the same scoring rule works on text and images

The training and testing stages use the same frozen CLIP encoders. The only thing that changes is the input source. During training, the model reads text descriptions. During testing, it reads images.

Formula card: frozen CLIP provides global and local features

Purpose: keep both a coarse summary and a fine-grained feature map from CLIP.

Equation:

{f,F}=EncI(x),{h,H}=EncT(r)\begin{split} \{\mathbf{f}, \mathbf{F}\} &= \mathrm{Enc}_I(\mathbf{x}), \\ \{\mathbf{h}, \mathbf{H}\} &= \mathrm{Enc}_T(\mathbf{r}) \end{split}{f,F}{h,H}​=EncI​(x),=EncT​(r)​

Intuition: the image encoder gives one global image vector and one dense feature map, while the text encoder gives one global sentence vector and one token sequence.

Symbols:

  • x\mathbf{x}x: the input image during testing.
  • r\mathbf{r}r: the text description during training.
  • f\mathbf{f}f: the global image feature.
  • F\mathbf{F}F: the dense image feature map after flattening spatial positions.
  • h\mathbf{h}h: the global text feature.
  • H\mathbf{H}H: the sequential text token features.
  • EncI\mathrm{Enc}_IEncI​: the frozen CLIP image encoder.
  • EncT\mathrm{Enc}_TEncT​: the frozen CLIP text encoder.

Read-aloud meaning: extract both a global feature and a local feature set from either a text description or an image.

Why it matters: the whole method depends on preserving fine-grained information, not just the pooled CLIP embedding. That is what lets the local branch learn from words during training and from image patches at test time.

The global branch then compares the current query feature with the global class embedding.

Formula card: global and local similarity scores

Purpose: score each class against the current query at both granularities.

Equation:

pi=⟨u,Gi⟩,Pij=⟨Uj,Li⟩\begin{split} p_i &= \langle \mathbf{u}, \mathbf{G}_i \rangle, \\ P_{ij} &= \langle \mathbf{U}_j, \mathbf{L}_i \rangle \end{split}pi​Pij​​=⟨u,Gi​⟩,=⟨Uj​,Li​⟩​

Intuition: use one score for the whole query and a score map for the local tokens or patches.

Symbols:

  • pip_ipi​: the global similarity score for class iii.
  • PijP_{ij}Pij​: the local similarity between class iii and position jjj.
  • u\mathbf{u}u: the current global query feature, which is h\mathbf{h}h during training and f\mathbf{f}f during testing.
  • U\mathbf{U}U: the current local feature set, which is H\mathbf{H}H during training and F\mathbf{F}F during testing.
  • Gi\mathbf{G}_iGi​: the global class embedding for class iii.
  • Li\mathbf{L}_iLi​: the local class embedding for class iii.
  • jjj: the token or patch index.

Read-aloud meaning: compare the query to each class once at the global level, and compare it again position by position at the local level.

Why it matters: this is where the “texts as images” idea becomes operational. The same similarity rule can score text against prompts during training and image features against prompts during testing because CLIP already aligned the two modalities.

The local score map is then collapsed with a soft attention over positions.

Formula card: soft aggregation of local evidence

Purpose: turn the token or patch score map into a single class score.

Equation:

pi′=∑j=1Nexp⁡(Pij/τs)∑j=1Nexp⁡(Pij/τs) Pijp_i' = \sum_{j=1}^{N} \frac{\exp(P_{ij} / \tau_s)}{\sum_{j=1}^{N} \exp(P_{ij} / \tau_s)} \, P_{ij}pi′​=j=1∑N​∑j=1N​exp(Pij​/τs​)exp(Pij​/τs​)​Pij​

Intuition: give more weight to the positions that look most relevant for the class, instead of averaging every location equally.

Symbols:

  • pi′p_i'pi′​: the aggregated local score for class iii.
  • PijP_{ij}Pij​: the local similarity score at position jjj.
  • NNN: the number of token or patch positions.
  • τs\tau_sτs​: the local focusing temperature.

Read-aloud meaning: softly attend to the most relevant token or patch positions, then average the local evidence with that attention pattern.

Why it matters: a global average would drown out small objects or rare class cues. The softmax weighting lets the local branch zoom in on the strongest evidence instead.

That is the real reason TaI-DPT works better for multi-label tasks than a single prompt branch does. Global features are still useful when the main object is obvious, but local features keep secondary objects from disappearing.

Why ranking loss fits the geometry better

The paper does not train the prompt branches with binary cross-entropy. It uses a ranking loss instead. The reason is not cosmetic. CLIP cosine similarities are not nicely centered around zero, so forcing them through a sigmoid can make optimization awkward.

Ranking loss is a more natural match to the actual goal: positive classes should score higher than negative classes by a margin.

Formula card: ranking loss for both branches

Purpose: push positive classes above negative classes in both the global and local branches.

Equation:

L=Lglobal+Llocal,Lglobal=∑i∈{c+}∑j∈{c−}max⁡(0,m−pi+pj),Llocal=∑i∈{c+}∑j∈{c−}max⁡(0,m−pi′+pj′)\begin{split} \mathcal{L} &= \mathcal{L}_{global} + \mathcal{L}_{local}, \\ \mathcal{L}_{global} &= \sum_{i \in \{c^{+}\}} \sum_{j \in \{c^{-}\}} \max(0, m - p_i + p_j), \\ \mathcal{L}_{local} &= \sum_{i \in \{c^{+}\}} \sum_{j \in \{c^{-}\}} \max(0, m - p_i' + p_j') \end{split}LLglobal​Llocal​​=Lglobal​+Llocal​,=i∈{c+}∑​j∈{c−}∑​max(0,m−pi​+pj​),=i∈{c+}∑​j∈{c−}∑​max(0,m−pi′​+pj′​)​

Intuition: every positive class should outrank every negative class by at least a margin.

Symbols:

  • L\mathcal{L}L: the total training loss.
  • Lglobal\mathcal{L}_{global}Lglobal​: the ranking loss for the global branch.
  • Llocal\mathcal{L}_{local}Llocal​: the ranking loss for the local branch.
  • c+c^{+}c+: the set of positive classes for a text description.
  • c−c^{-}c−: the set of negative classes for that description.
  • pip_ipi​: the global score for class iii.
  • pi′p_i'pi′​: the aggregated local score for class iii.
  • mmm: the margin hyperparameter.

Read-aloud meaning: make every positive class score higher than every negative class, both for the whole-description branch and for the token/patch branch.

Why it matters: this loss matches the CLIP-style similarity geometry better than BCE. It also fits the multi-label setting naturally, because the model only needs to rank the right classes above the wrong ones.

This choice matters empirically too. The paper reports that ranking loss beats BCE and ASL across the three datasets, which is a strong hint that the objective is not a detail you can ignore.

Why the method still helps when images are available

TaI-DPT is not trying to replace image-side prompt tuning. It is trying to show that text-side prompting and image-side prompting are complementary. When few labeled images or partially labeled images are available, the paper fuses the scores from TaI-DPT with methods like CoOp or DualCoOp.

Figure 5 shows the basic idea.

The reason score fusion is used instead of embedding fusion is practical. Different methods may rely on different image encoders, so averaging class embeddings would not be a clean comparison. Fusing predicted scores is easier and keeps each method’s encoder setup intact.

This is also where the method’s role becomes clearer. TaI-DPT is not a competing worldview. It is a text-trained prompt source that can be plugged into an image-trained system when those image labels exist.

What to keep in mind

The paper’s headline result is that text-only prompt training can beat zero-shot CLIP by a wide margin on multi-label recognition. On VOC2007, for example, TaI-DPT reaches 88.3 mAP versus 77.3 for zero-shot CLIP with DPT. The same pattern holds on MS-COCO and NUS-WIDE.

But the method still has boundaries.

The pseudo labels are only as good as the noun filter and synonym coverage. Paraphrases can be missed, and noisy matches can creep in. The method is validated mainly on multi-label recognition, so transfer to other tasks is still open. And when enough labeled images are available, fully image-supervised prompting can still win.

So the best way to remember TaI-DPT is this:

  • collect text descriptions that mention target classes,
  • turn those descriptions into weak multi-label supervision,
  • learn separate global and local prompts with frozen CLIP,
  • score whole scenes and local cues with the same prompt space,
  • use ranking loss to keep positives ahead of negatives,
  • and, when needed, fuse those text-trained prompts with image-trained ones.

The deeper lesson is simple. If a vision-language model already aligns text and images, then the cheaper modality can sometimes teach the prompt learner what the expensive modality would have taught it. TaI-DPT is one clean example of that idea.
