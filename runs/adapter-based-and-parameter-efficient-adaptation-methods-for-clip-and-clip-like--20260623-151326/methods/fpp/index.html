Flatness-aware Prompt Pretraining for Test-Time Prompt Tuning

FPP is easiest to understand as a change in where adaptation begins. Instead of asking test-time prompt tuning to rescue a prompt that starts in a sharp, brittle region, FPP spends an offline step shaping the starting prompt itself. The goal is to keep the zero-shot meaning of the prompt while moving it into a flatter neighborhood of the loss landscape, so the later entropy-minimization step has less chance of falling into a fragile basin.

Think of it like sanding a road before you drive on it. If the road is already smoother, the same small steering correction at test time is less likely to throw you off course.

Why the paper stops blaming calibration only on test-time tuning

The paper begins from a useful reframe. Regularized TPT methods such as C-TPT and O-TPT do improve calibration, but they often pay for it with accuracy. FPP argues that this is not just an unfortunate side effect of the regularizer. Those methods implicitly push prompt optimization toward flatter minima.

That matters because sharp prompts are sensitive prompts. A tiny perturbation can change the text features enough to make the entropy-minimization update chase the wrong basin. The paper’s core observation is that flatter minima line up with lower calibration error, so the right move is not simply “add more regularization at adaptation time.” It is to start from a better prompt before adaptation begins.

The paper’s main diagnostic plot makes that point visually.

What to notice is the trend, not the exact scale. When the prompt lands in a flatter region, calibration improves. That is the reason FPP moves the flatness idea into pretraining instead of leaving it as a patch applied during each test-time update.

The appendix then makes the bridge more formal: the regularizers used by prior TPT methods behave like a SAM-style perturb-and-update rule, which is why they end up nudging prompts toward flat minima in the first place.

What FPP actually learns before test time

FPP only needs the class names and the standard zero-shot CLIP prompt template. No labeled images are used. The method creates small synthetic perturbations around the prompt and asks two questions at once: does the prompt stay close to the original zero-shot semantics, and does it remain stable when nudged?

Formula card: flatness loss

Purpose: Penalize prompt settings whose text features change too much under small perturbations.

Equation:

Lflat=dist⁡cos ⁣(fT(C+ε1;  θ+ε2),  fT(C; θ))\mathcal{L}_{\mathrm{flat}}=\operatorname{dist}_{\text{cos}}\!\big(f_{T}(C+\varepsilon_{1};\;\theta+\varepsilon_{2}),\;f_{T}(C;\,\theta)\big)Lflat​=distcos​(fT​(C+ε1​;θ+ε2​),fT​(C;θ))

Intuition: If a tiny random change to the class-name side or the prompt parameters barely changes the text embedding, the prompt sits in a flatter region of the landscape.

Symbols:

  • Lflat\mathcal{L}_{\mathrm{flat}}Lflat​: flatness loss
  • fTf_TfT​: CLIP text encoder
  • CCC: class-name set used during pretraining
  • C+ε1C+\varepsilon_{1}C+ε1​: a perturbed version of the class-name side of the text input
  • θ\thetaθ: prompt parameters
  • θ+ε2\theta+\varepsilon_{2}θ+ε2​: perturbed prompt parameters
  • ε1\varepsilon_{1}ε1​: small random perturbation applied on the class side
  • ε2\varepsilon_{2}ε2​: small random perturbation applied to the prompt
  • dist⁡cos\operatorname{dist}_{\text{cos}}distcos​: cosine distance between text features

Read-aloud meaning: Compare the text embedding before and after a small perturbation, and penalize it if the embedding changes too much.

Why it matters: This is the part of FPP that pushes the initialization away from sharp pockets. The perturbations are synthetic, so the method can do this without using any images or labels.

The important detail is that the method is not trying to make the prompt insensitive in a vacuum. It is trying to make the prompt insensitive while still staying close enough to the original text semantics that later adaptation remains useful.

Formula card: alignment loss

Purpose: Keep the pretrained prompt anchored to the semantics of the original zero-shot prompt.

Equation:

Lalign=dist⁡L2 ⁣(fT(C; θ),  fT(C; θ0zs))\mathcal{L}_{\mathrm{align}}=\operatorname{dist}_{\text{L2}}\!\big(f_{T}(C;\,\theta),\;f_{T}(C;\,\theta_{0}^{\text{zs}})\big)Lalign​=distL2​(fT​(C;θ),fT​(C;θ0zs​))

Intuition: The flatness loss wants to smooth the landscape, but that pressure can drift the prompt away from what CLIP originally meant. The alignment loss prevents that drift by pulling the learned text features back toward the zero-shot prompt.

Symbols:

  • Lalign\mathcal{L}_{\mathrm{align}}Lalign​: alignment loss
  • fTf_TfT​: CLIP text encoder
  • CCC: class-name set
  • θ\thetaθ: learned prompt parameters
  • θ0zs\theta_{0}^{\text{zs}}θ0zs​: standard zero-shot prompt
  • dist⁡L2\operatorname{dist}_{\text{L2}}distL2​: L2 distance between text features

Read-aloud meaning: Learn a new prompt whose text features stay close to the original zero-shot prompt’s text features.

Why it matters: Flatness alone is not enough. Without this anchor, the prompt can lose its original meaning so badly that zero-shot behavior collapses before adaptation even starts.

Formula card: final pretraining objective

Purpose: Combine semantic preservation and flatness into one data-free pretraining objective.

Equation:

LFPP=Lalign+λ Lflat\mathcal{L}_{\mathrm{FPP}}=\mathcal{L}_{\mathrm{align}}+\lambda\,\mathcal{L}_{\mathrm{flat}}LFPP​=Lalign​+λLflat​

Intuition: One term protects the original prompt meaning, while the other term pushes the initialization into a flatter region.

Symbols:

  • LFPP\mathcal{L}_{\mathrm{FPP}}LFPP​: full FPP pretraining objective
  • Lalign\mathcal{L}_{\mathrm{align}}Lalign​: alignment loss
  • Lflat\mathcal{L}_{\mathrm{flat}}Lflat​: flatness loss
  • λ\lambdaλ: weight that balances flatness against alignment

Read-aloud meaning: Learn a prompt that stays semantically close to the zero-shot prompt and at the same time becomes less sensitive to perturbation.

Why it matters: This is the whole method in one line. FPP is not “make prompts flatter” and it is not “keep prompts unchanged.” It is the combination of the two.

The paper also makes λ\lambdaλ slightly smaller when the label space gets larger. That is a practical way to soften the flatness pressure when preserving semantics becomes harder across more classes.

Why the two terms have to travel together

The ablation study makes the design choice obvious.

  • Flatness without alignment collapses zero-shot behavior.
  • Alignment without flatness keeps the original prompt mostly intact, but calibration remains weak.
  • Using both gives the best balance of accuracy, ECE, and SCE.

That is the key lesson: the flatness term changes geometry, but the alignment term keeps meaning. If you drop either piece, you lose something important.

This is also why FPP feels different from a regularizer added at test time. Regularized TPT tries to shape the update as it happens. FPP shapes the starting point so the ordinary TPT update is already operating in a safer region.

How it plugs into the existing pipeline

Once FPP has pretrained the prompt, nothing else has to change. The resulting prompt is just a better initialization for TPT, C-TPT, O-TPT, or even other prompt-based variants such as CoOp, MaPLe, and DynaPrompt. The later adaptation loop stays the same.

That is the practical advantage of moving the work offline:

  • no labeled data is needed for the pretraining step,
  • no extra test-time compute is added,
  • the downstream adaptation code can stay unchanged.

So the output of FPP is not a new classifier. It is a starting prompt that already sits in a better part of the landscape.

What the experiments are really showing

The headline result is that this change of starting point is enough to improve both calibration and accuracy. On CLIP-ViT/B16 under TPT, FPP reports 65.37 average accuracy with 4.13 ECE and 0.96 SCE. Against O-TPT, it keeps higher accuracy while lowering average sharpness from 1.158 to 0.742.

The more interesting part is that the same initialization also improves CoOp, MaPLe, and DynaPrompt variants. That tells you the effect is not tied to one specific adaptation loop. It is a more general prompt-initialization effect.

The paper also checks natural distribution shifts and still sees the same pattern: calibration improves without a large accuracy penalty. So the method is not just winning on one carefully tuned benchmark setting.

Where it is still narrow

FPP is still a CLIP-style prompt method. It relies on the class-name set during pretraining, even though the paper shows some robustness when those names are substituted or noised. The strongest evidence is in classification and calibration benchmarks, not broader multimodal tasks.

So the method is best read as a targeted design principle: if test-time updates are fragile, improve the starting basin instead of piling more constraints onto the update itself.

What to remember

FPP changes the order of operations.

It does not try to rescue a sharp prompt at test time. It pretrains the prompt so the same test-time update starts from a flatter, safer region. The alignment loss keeps the prompt meaningful, the flatness loss makes it less sensitive, and the combined objective gives you a drop-in initialization that improves calibration without adding inference cost.

If you want one sentence to keep in your head, make it this: FPP moves the calibration problem one step earlier, and that is enough to make later prompt tuning more reliable.
