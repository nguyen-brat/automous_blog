Test-Time Prompt Tuning for Zero-Shot Generalization in Vision-Language Models

TPT starts from a very small but important observation: in CLIP-like models, the prompt is not just decoration around the class name. It is the interface that tells the frozen model how to read the image. If that interface is slightly off for a new test image, the model can miss the right class even though the encoders themselves are still strong. TPT’s answer is to adapt that interface on the fly, using only the test sample that is already in front of the model.

That makes the method feel almost paradoxical at first. It is doing prompt tuning, but not on a training set. It is using test-time optimization, but not on the backbone. The thing that changes is only the prompt tokens, and the signal for changing them comes from multiple augmented views of the same image.

What to notice in the figure is the division of labor. The image encoder and text encoder stay frozen. The test image is turned into many slightly different views, and those views are used to decide how the prompt should move. The output is not a new model. It is the same model, but asked a better question for this one sample.

Why the prompt is the right thing to move

If you think about CLIP classification in the simplest way, the model compares an image embedding with text embeddings built from class names plus a prompt like “a photo of a”. That means the prompt is not a cosmetic wrapper. It changes the text representation that the image is being matched against. A bad prompt can shift the class texts just enough to make the decision worse.

That is why the paper spends time checking which part of CLIP should be optimized at test time. The ablation is informative: tuning the prompt tokens works better than tuning the text encoder, the visual encoder, or the whole model. The visual encoder is the worst place to touch because it can distort the pretrained feature geometry. Prompt tokens are the smallest and safest handle, so they preserve CLIP’s broad zero-shot knowledge while still letting the method adapt.

This is also the reason TPT feels more like interface repair than model retraining. The backbone already knows a lot. TPT is trying to make that knowledge easier to retrieve for the current sample.

How one unlabeled image becomes a learning signal

The key trick is to turn a single test image into a set of related views, then ask the model to be consistent across those views. If one crop keeps the important object in frame, it should vote with confidence. If another crop cuts away the object or leaves the image too ambiguous, it should not steer the prompt update very much.

That is where the entropy objective and confidence selection come in. TPT does not simply average every view and hope for the best. It first filters out the views that look unreliable, then averages the remaining predictions, and then minimizes the entropy of that average.

Read-aloud meaning: average the predictions from the confident views of one test image, then push that average distribution toward a clearer class decision.

Why it matters: this is the unsupervised signal that replaces labeled training data. It is the main reason TPT can move prompt tokens at test time without seeing any task annotations.

The order matters here. TPT does not minimize entropy view by view and then average the losses. It first builds a consensus distribution from the views it trusts, and then it sharpens that consensus. That makes the update favor agreement across views instead of letting one unlucky crop dominate the step.

Why confidence selection is more than a small tweak

Confidence selection is easy to underestimate, but it solves a real problem. Random augmentations are useful only if they still show enough of the object or concept to be informative. A crop that removes the dog’s head or the bicycle frame is not a better witness. It is a weaker witness.

So TPT ranks the augmented views by their own prediction entropy and keeps the low-entropy ones. The threshold is chosen per sample, not globally, by taking the entropy value at a percentile of the current view scores. That matters because easy and hard images should not be filtered the same way. A fixed cutoff would be too brittle; the percentile rule adapts to the sample in front of the model.

In the default classification setup, the paper uses this filtered batch for just one optimization step. That choice says a lot about the method. TPT is not trying to turn inference into a long training run. It wants just enough backpropagation to nudge the prompt in a better direction, then it stops before the cost starts to outweigh the gain.

The same idea on Bongard-HOI

The image-classification case is the cleanest place to see TPT, but the paper also applies it to context-dependent visual reasoning. Bongard-HOI is a good example because the label is not a fixed class name like “dog” or “car”. Instead, each test sample contains positive support images, negative support images, and a query image. The support sets define a hidden human-object interaction concept, and the model has to decide whether the query image contains that concept.

Here TPT changes the prompt in a slightly different way. It tunes the prompt prefix and the two binary class tokens together, using the support images from the current test sample. The support set is doing the job that training data would usually do: it teaches the model what the concept looks like. But because the support images are part of the test instance, the method is still test-time adaptation rather than training on a separate task dataset.

What to notice in this figure is that the support images are not just extra context for the query. They are the adaptation signal itself. The prompt is being tuned to separate the positive and negative support sets, and the adapted prompt is then used to classify the query.

What the results actually say

The evidence in the paper points in the same direction across several settings. On natural distribution shifts, TPT improves zero-shot CLIP by 3.6% on average, and on ImageNet-A it reaches a maximum gain of 6.9% over the handcrafted-prompt baseline. On Bongard-HOI, TPT with CLIP-RN50 reaches 66.59 average accuracy, beating HOITrans at 62.46.

The broader lesson is not just that TPT wins a few benchmarks. It is that a frozen foundation model can often adapt better by changing its input interface than by changing its internal weights. The paper also shows that TPT can sit on top of prompts learned by CoOp or CoCoOp and improve them further, which is a good sign that the test-time update is complementary rather than redundant.

The cross-dataset results tell a similar story. When categories move away from the source dataset, prompt-tuning methods trained on that source can overfit badly. TPT avoids that trap because it adapts per sample instead of per source dataset, so it keeps benefiting even when the target categories are not the ones it saw during training.

Where it can fail

TPT is attractive because it is small and modular, but it is not free. It needs backpropagation at inference time, and the multiple augmented views increase memory use and latency. The method also depends on the unsupervised objective being aligned with the task. If the augmentations erase the important evidence or the sample is genuinely ambiguous, confidence selection can end up reinforcing the wrong answer instead of correcting it.

That limitation is why the paper’s future direction is interesting: lower-memory adaptation signals, other vision-language models, and tasks where the test-time objective has to be redesigned from scratch.

What to remember

TPT is easiest to remember as prompt-side repair for a frozen model. Make several views of one test sample, keep the views you trust, average their predictions, and move only the prompt tokens. The backbone stays untouched. That is why the method can preserve zero-shot behavior while still recovering some robustness under shift.
