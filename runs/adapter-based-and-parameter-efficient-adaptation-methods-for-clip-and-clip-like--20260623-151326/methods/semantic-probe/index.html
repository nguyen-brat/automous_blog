Reviving In-domain Fine-tuning Methods for Source-Free Cross-domain Few-shot Learning

Semantic Probe starts from a surprisingly practical diagnosis: in cross-domain few-shot CLIP adaptation, the model does not mainly fail because it lacks parameters. It fails because the visual summary token, the CLS token, looks in the wrong place. Once the visual branch collapses into self-attention, the image feature stops tracking the class semantics carried by text, and the whole image-text alignment begins to drift.

The paper’s answer is not to ask for a bigger prompt or a heavier adapter stack. It uses the text side as a guide. More specifically, it takes the text EOS token, which tends to attend to the right discriminative regions more reliably, and uses that signal to repair the visual CLS attention. Then it adds a loss schedule that emphasizes alignment while the model still needs help, and gradually shifts attention toward class separation once the representations are already lined up.

What to notice in this figure is the difference between “something changed” and “something got fixed.” MaPLe barely moves the attention map after fine-tuning, so the CLS token remains stuck on itself or on unhelpful regions. LoRA, by contrast, shifts attention toward class-relevant patches. That is the paper’s first clue that the real bottleneck is attention repair, not simply more trainable parameters.

The failure mode is a bad summary token, not a bad classifier

The paper’s starting point is easier to understand if you think about what CLIP is trying to do. The image encoder and text encoder are supposed to meet in a shared space. In source-free cross-domain few-shot learning, we only have a few target examples, so we cannot afford to relearn that shared space from scratch. We need to preserve CLIP’s pretrained geometry while nudging it toward the target domain.

The trouble is that cross-domain fine-tuning often disturbs the visual branch in the wrong way. The CLS token becomes a lazy summary: instead of collecting patch evidence that matches the class description, it keeps paying attention to itself. That creates two downstream problems at once. The text and image features drift apart, and the image feature itself becomes less useful for separating classes.

The paper shows this in feature space as well. LoRA-CLIP produces tighter clusters and a smaller modality gap than prompt-based tuning, which is why LoRA keeps winning in this setting. But the important lesson is broader than LoRA. The gain comes from repairing where attention goes.

Semantic Probe in one glance

You can read Semantic Probe as a small plug-in workflow:

  1. feed semantically rich class text into CLIP,
  2. use the EOS token as a semantic guide,
  3. blend that guidance into the visual CLS attention,
  4. train with a loss that cares about both alignment and separation,
  5. classify by cosine similarity to class text embeddings.

This diagram is the method in miniature. The text branch is not just generating labels; it is acting as a teacher for the image branch. EAR, the EOS-guided Attention Rectification module, sits between the two encoders and edits the visual attention map. BAS, the Balanced Alignment and Separation loss, then steers training so the model first fixes the cross-modal mismatch and only later spends more effort on class separation.

That is why the method feels like a repair loop rather than a new architecture. The backbone is still CLIP. Semantic Probe just changes what the CLS token is allowed to focus on and how the optimizer decides what matters first.

Letting the EOS token point the CLS token at the right patches

The heart of the paper is a very human idea: if the image summary token is confused, ask the text endpoint to point at the right visual evidence.

The EOS token is useful because the paper does not feed CLIP a bare class name. It uses semantically richer class descriptions, so the EOS representation already carries a compact summary of what the class should look like. In the paper’s visualizations, EOS attention is broader and more discriminative than the visual CLS attention. That makes it a reasonable guide for rectifying the visual side.

What to notice here is the row structure. The CLS attention rows show a token that often under-attends to the image content. The EOS rows are more willing to spread over the right regions. After rectification, the visual attention becomes more spatially useful instead of collapsing onto the summary token itself.

The paper only applies EAR in the deeper CLIP layers. That choice is important. Early layers are still building generic structure, so it is better not to over-steer them. The later layers are where task-specific discrimination happens, which is the right place for a semantic correction to land.

A loss that changes its balance while training

EAR fixes where the model looks. BAS fixes how the optimizer spends its effort.

The paper observes that CLIP’s standard contrastive objective already helps with class aggregation, but it is not a strong enough constraint on modality alignment by itself. That means the image and text features can still drift apart even while the class separation improves. BAS adds an explicit alignment signal and then weights it dynamically, so the model can spend more time repairing alignment early in training and more time preserving separation later.

The paper’s ablations back this up. On MaPLe, EAR and BAS each help on their own, and the combination helps more. The same pattern holds for LoRA-CLIP. That is a useful sign that the two pieces are not redundant: EAR repairs the spatial prior, while BAS repairs the training objective.

What happens at the end

At inference time, the method stays simple. The image goes through the visual encoder, EAR-rectified CLS features flow through the later layers, and the final image feature is compared against the class text embeddings by cosine similarity. The text side still acts as the classifier, which means Semantic Probe remains a plug-in enhancement rather than a new end-to-end model.

That plug-in property is why the method works on both prompt-based and adapter-based baselines. It is not tied to MaPLe or to LoRA. It is trying to fix the same failure mode underneath both of them.

The reported gains are therefore a nice sanity check rather than the whole story. On 5-shot CDFSL, Semantic Probe raises MaPLe from 62.75 to 66.33 and LoRA-CLIP from 65.99 to 68.01. The more interesting evidence is structural: rectifying CLS attention increases the average attention entropy from 0.7303 to 0.8024, which is a compact way to say the token is no longer staring only at itself.

What to remember

Semantic Probe is easiest to remember as “let text teach the image where to look, then let training switch from alignment to separation at the right time.”

The paper’s deeper lesson is that few-shot CLIP adaptation is not only a parameter-efficiency problem. It is a representation-routing problem. If the CLS token collapses, no amount of clever prompt tuning will fully rescue the geometry. A better guide token and a better loss schedule can.

There are limits, though. The method depends on CLIP-style dual encoders, on text descriptions that are actually informative, and on EOS attention being a reliable guide. If class text is noisy or labels are ambiguous, the guidance can point the CLS token in the wrong direction. The paper is also validated on CLIP-based source-free cross-domain few-shot benchmarks, so the transfer story to SigLIP-style models or generative VLMs is still open.
