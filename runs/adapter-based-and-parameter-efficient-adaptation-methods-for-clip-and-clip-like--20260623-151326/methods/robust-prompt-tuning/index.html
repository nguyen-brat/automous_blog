Prompt Tuning Robustness to Noisy Labels in CLIP-like Vision-Language Models

Prompt tuning for CLIP-like models looks tiny from the outside, but this paper shows that tiny can be exactly what makes adaptation survive bad labels. The method keeps CLIP’s text structure fixed, learns only a small bank of shared soft prompt tokens, and optionally swaps cross-entropy for generalized cross-entropy so suspicious samples have less power to steer the update. That combination is not just parameter-efficient. It is hard to derail.

The payoff is visible in the high-noise regime. At 50% label noise, prompt tuning still reaches 73.13 on OxfordPets, far above the CLIP-initialized classifier and text-encoder fine-tuning baselines. Food101 shows the same pattern: 64.30 for prompt tuning versus 34.37 for the CLIP-initialized classifier. The deeper lesson is that robustness comes from keeping the class semantics anchored inside a frozen text encoder, not from giving the model more freedom.

The prompt is the whole trick, so the paper makes it tiny

The easiest way to understand the method is to look at what stays frozen and what moves.

What to notice in this figure is the asymmetry. The image encoder stays frozen. The text encoder stays frozen. The class names stay fixed. Only the shared prompt tokens are trained. That means prompt tuning is not learning a new classifier from scratch; it is learning a small task-specific context around a label space that CLIP already understands.

You can think of the method as “keep the meaning of the class names, but learn the best phrasing around them.” That is a much safer move when the labels are unreliable, because the noisy supervision never gets the chance to rewrite the entire semantic structure.

Formula card: the prompt template

Purpose: define the only trainable part of the class text input.

Pc=[p1,p2,…,pM,wc]\mathrm{P}_c = [\mathbf{p}_1, \mathbf{p}_2, \ldots, \mathbf{p}_M, \mathbf{w}_c]Pc​=[p1​,p2​,…,pM​,wc​]

Intuition: every class gets the same learned context tokens, and only the class-name token changes.

Symbols:

  • Pc\mathrm{P}_cPc​ is the full prompt for class ccc.
  • pm\mathbf{p}_mpm​ is the mmm-th soft prompt token shared across all classes.
  • wc\mathbf{w}_cwc​ is the fixed classname token for class ccc.
  • MMM is the number of learned prompt tokens.

Read-aloud meaning: build each class prompt from one shared learned prefix and one fixed class name.

Why it matters: the shared prompt can adapt to the target task, but it cannot invent a separate private prompt for each class. That constraint is a big part of the method’s noise robustness.

Once the prompt is built, CLIP turns it into a class embedding and compares that embedding with the image embedding.

Formula card: CLIP class posterior

Purpose: turn prompted text embeddings into class probabilities.

Pr⁡(y=i∣x)=exp⁡(sim⁡(fv,fit)/τ)∑j=1Kexp⁡(sim⁡(fv,fjt)/τ)\Pr(y=i\mid \mathbf{x})=\frac{\exp\bigl(\operatorname{sim}(\mathbf{f}^v,\mathbf{f}^t_i)/\tau\bigr)}{\sum_{j=1}^{K}\exp\bigl(\operatorname{sim}(\mathbf{f}^v,\mathbf{f}^t_j)/\tau\bigr)}Pr(y=i∣x)=∑j=1K​exp(sim(fv,fjt​)/τ)exp(sim(fv,fit​)/τ)​

Intuition: score the image against every prompted class, then normalize those scores into a probability distribution.

Symbols:

  • Pr⁡(y=i∣x)\Pr(y=i\mid \mathbf{x})Pr(y=i∣x) is the probability that image x\mathbf{x}x belongs to class iii.
  • fv\mathbf{f}^vfv is the normalized image embedding from CLIP’s visual encoder.
  • fit\mathbf{f}^t_ifit​ is the normalized text embedding for class iii built from the prompt.
  • sim⁡(⋅,⋅)\operatorname{sim}(\cdot,\cdot)sim(⋅,⋅) is cosine similarity.
  • τ\tauτ is the CLIP temperature.
  • KKK is the number of classes.

Read-aloud meaning: an image gets a class probability by comparing it to each prompted class text and softmaxing the similarities.

Why it matters: this is the bridge between the prompt and the final prediction. The method does not train a new classifier head; it changes the text side of CLIP just enough to steer those similarity scores.

So the data flow is simple: image in, frozen visual encoder, similarity against prompted class texts, softmax, then a loss that updates only the shared prompt tokens. That simplicity is the point.

Why fixed class names matter more than they first appear

The paper spends a lot of time on a subtle but important question: what should remain fixed inside the prompt? The answer is the classname token itself.

If you let the class name move, the model can start rewriting the label semantics rather than adapting around them. That sounds flexible, but under noisy labels it is exactly the kind of flexibility that becomes dangerous. A mislabeled example no longer just nudges the context; it can also pull the class identity around with it.

The experiments make that failure mode concrete. Learning the classname token, or learning the entire prompt, hurts high-noise robustness compared with keeping class names fixed. In other words, the class name is not just a string. It is the semantic anchor that keeps the prompt from drifting.

This is also why prompt tuning behaves better than the other CLIP adaptation baselines in the paper. A CLIP-initialized linear classifier starts from a good prior, but it still has a free classifier to fit noisy labels. Text-encoder fine-tuning has even more freedom to overfit. Prompt tuning keeps the adaptation surface much smaller, so noisy data has less room to distort the pretrained label space.

The contrast shows up clearly in the results. At 50% label noise on OxfordPets, prompt tuning reaches 73.13, while the CLIP-initialized classifier falls to 44.21 and text-encoder fine-tuning to 41.21. On Food101, prompt tuning gets 64.30, while the CLIP-initialized classifier drops to 34.37. Those are not small differences. They are the gap between “the model still understands the task” and “the supervision has taken over.”

The loss can be made kinder to suspicious samples

The paper does not stop at the architecture. It also changes the training objective so noisy examples contribute less damage.

The default objective is plain cross-entropy on the labeled or pseudo-labeled target set.

Formula card: standard prompt-tuning loss

Purpose: train the prompt with ordinary classification supervision.

LCE=−E(x,c)∈D[log⁡Pr⁡(y=c∣x)]\mathcal{L}_{CE}=-\mathbb{E}_{(\mathbf{x},c)\in\mathcal{D}}\left[\log \Pr(y=c\mid \mathbf{x})\right]LCE​=−E(x,c)∈D​[logPr(y=c∣x)]

Intuition: make the correct class probability large for every training example.

Symbols:

  • D\mathcal{D}D is the target training set.
  • (x,c)(\mathbf{x},c)(x,c) is an image-label pair from that set.
  • Pr⁡(y=c∣x)\Pr(y=c\mid \mathbf{x})Pr(y=c∣x) is the posterior probability of the correct class.

Read-aloud meaning: penalize the model whenever it assigns low probability to the true label.

Why it matters: this is the baseline prompt-tuning objective, and it is still vulnerable when labels are wrong.

Robust Prompt Tuning swaps that loss for generalized cross-entropy.

Formula card: generalized cross-entropy

Purpose: reduce the influence of outliers and mislabeled examples during prompt training.

LGCE=E(x,c)∈D[1−Pr⁡(y=c∣x)qq]\mathcal{L}_{GCE}=\mathbb{E}_{(\mathbf{x},c)\in\mathcal{D}}\left[\frac{1-\Pr(y=c\mid \mathbf{x})^q}{q}\right]LGCE​=E(x,c)∈D​[q1−Pr(y=c∣x)q​]

Intuition: keep the signal from correct examples, but soften the pressure from samples that look suspicious or noisy.

Symbols:

  • D\mathcal{D}D is the training set.
  • (x,c)(\mathbf{x},c)(x,c) is an image-label pair.
  • Pr⁡(y=c∣x)\Pr(y=c\mid \mathbf{x})Pr(y=c∣x) is the posterior probability of the correct class.
  • qqq is the robustness knob that controls how close the loss is to cross-entropy versus mean absolute error.

Read-aloud meaning: train with a loss that is less eager than cross-entropy to chase every problematic example.

Why it matters: GCE is the paper’s optional robustness upgrade. When qqq is near 0, it behaves more like cross-entropy. When qqq moves toward 1, it becomes more MAE-like and less sensitive to hard or noisy points. The paper reports that q=0.7q=0.7q=0.7 works well across settings.

The important detail is that GCE is not the whole story by itself. The paper shows that robust losses help the linear probe too, but the strongest effect appears when GCE is paired with the constrained prompt-tuning setup. The loss helps, but the narrow prompt parameterization is what makes that help stick.

The gradients tell the same story

The paper also checks the update directly instead of only looking at final accuracy. It measures the ratio between gradient norms from noisy samples and clean samples. Prompt tuning produces much lower noisy-to-clean ratios than linear probing.

That result is the mechanism in one sentence: bad labels have less leverage when the only trainable parameters are a small set of shared prompt tokens living inside a frozen semantic label space. A noisy sample can still push, but it cannot bend the whole classifier as easily.

This is why the method also handles correlated label noise better than you might expect. When the wrong label is chosen in a way that matches CLIP’s own confusion patterns, the task becomes harder for everyone. The gap shrinks, but prompt tuning still stays ahead of linear probing. So the paper is not claiming immunity. It is claiming a stronger default defense.

The same pattern also holds across backbones and context lengths. A ViT-B/32 backbone usually does a bit better than ResNet-50 in the paper’s tests, and the best number of prompt tokens depends on the dataset. That is a useful reminder that robustness is not magic. It is a property of the prompt bottleneck plus the pretrained backbone, and the exact amount of prompt capacity still matters.

The unsupervised version is willing to trust more noise

The most interesting application is unsupervised prompt tuning. Here the model first uses CLIP itself to generate pseudo labels for the target images, then reuses those pseudo labels to tune the prompt.

What to notice in this figure is the deliberate shift in sampling strategy. Prior work such as UPL tries to keep the pseudo-label set clean by selecting only the most confident examples. Robust Prompt Tuning is more willing to trade some label purity for diversity. It randomly samples pseudo-labeled images per class, then relies on the constrained prompt and GCE to absorb the extra noise.

That sounds reckless until you remember the earlier analysis. If prompt tuning really does suppress noisy gradients, then the training set does not have to be perfectly clean for the method to work. In fact, a more diverse pseudo-labeled set can be better than a narrow high-confidence set because it exposes the prompt to more of the target domain.

This is the paper’s unsupervised twist: diversity becomes an asset once the adaptation rule is robust enough. On ImageNet, robust UPL with GCE improves CLIP zero-shot accuracy from 58.18 to 62.14. The same pattern generally shows up across the other datasets too, though not every benchmark moves equally.

What the paper leaves open

The method is strong, but it is not a universal answer to noisy transfer. The evidence is centered on classification-style adaptation, not broader multimodal tasks. The gains also depend on context length and target difficulty, so there is no single prompt size that works everywhere. And when the noise is specifically designed to mimic CLIP’s own confusions, the problem becomes harder, even if prompt tuning still does better than linear probes.

So the right way to remember the paper is not “prompt tuning always beats noise.” The right way is “frozen class semantics plus a small shared prompt make CLIP much harder to corrupt than a free classifier does.”

What to remember

Robust Prompt Tuning is easiest to remember as a three-part constraint: freeze the CLIP encoders, keep the classname token fixed, and let a robust loss soften suspicious examples. The method is small on purpose, and that is why it works. Bad labels can only tug on a narrow shared prompt inside a strong pretrained semantic space, so they do not get to rewrite the label space the way they can in a free classifier.

The deeper lesson is bigger than this one paper. Parameter-efficient adaptation is not only about reducing trainable parameters. It is also about choosing where the adaptation is allowed to happen. For noisy supervision, the safest place is a tiny prompt wrapped around a frozen text encoder.
