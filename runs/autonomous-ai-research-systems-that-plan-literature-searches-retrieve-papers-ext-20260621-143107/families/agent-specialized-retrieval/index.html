Retrieval for Research Agents: From Query Mismatch to Trace-Aware Search

Research agents do not fail at retrieval for the same reason a normal search box fails. The hard part is not only finding relevant papers. It is finding the right evidence while the agent is decomposing a broad question, issuing imperfect subqueries, reading partial context, and trying to turn scattered papers into a coherent answer. That makes retrieval an interface problem between planning and search, not just an indexing problem.

This matters because literature-search agents are only as good as the evidence they can surface inside the loop. A strong reasoning model can still miss the right paper if its subqueries are too keyword-heavy for a dense retriever, too ambiguous for a query-only encoder, or too broad for full-document ranking. The common theme across this subtopic is not a single shared algorithm. It is a shared interface problem: the retrieval system has to match how an agent actually searches.

The Retrieval Loop Inside a Research Agent

The recurring setup is simple on paper:

In practice, each arrow hides a mismatch risk. A top-level research question gets broken into smaller searches. Those subqueries hit a corpus, often over long markdown papers. The agent then aggregates the returned evidence and decides what to ask next before writing a final synthesis.

Two details matter immediately.

First, the query the retriever sees is rarely the original user question. It is usually an agent-generated subquery, often short and keyword-heavy. Second, the retrieval unit may not be a whole paper. In many agent settings, a passage is more useful than a full document because it lets the system surface a specific claim or method detail without spending the agent’s context window on irrelevant sections.

Why Retrieval Breaks in Agentic Search

Classic retrieval assumes a relatively stable query and a user who can inspect results directly. Research agents violate both assumptions.

One failure mode is query mismatch. The agent may ask terse, web-style subquestions, while a semantic retriever was trained for richer natural-language queries. In that setting, lexical overlap can matter more than embedding similarity.

Another failure mode is search quality under context pressure. Even when the first-stage retrieval is reasonable, returning full papers instead of focused passages can waste context and hide the relevant evidence too deep in the result.

A third failure mode is query ambiguity. Many subqueries only make sense in light of the agent’s current reasoning. If the retriever sees only the latest surface query, it loses the hidden intent behind the search.

The last major failure mode is supervision mismatch. Most retrievers are trained from human-labeled relevance or generic QA data, but research agents browse, reject, revisit, and reason over documents in ways that do not look like ordinary human search logs.

The Main Design Axes

Several design choices keep reappearing because they decide whether retrieval helps the agent or fights it.

Lexical vs. semantic matching

For agent-issued subqueries, lexical matching is often stronger than people expect. If the agent breaks a task into keyword-heavy searches, BM25 can outperform an LLM-based retriever simply because the subquery format aligns better with sparse matching than with semantic abstraction.

Single-shot retrieval vs. iterative search

A research agent rarely searches once. It alternates retrieval and reasoning. That means retrieval quality should be judged by how well it supports repeated search steps, not only by standalone ranking quality.

Full documents vs. passages

Long markdown papers contain useful evidence, but most of that text is not useful on a given turn. Passage-level retrieval often gives the agent cleaner evidence, more room for additional search steps, and less distraction from irrelevant sections.

Surface query vs. contextualized query

Some systems retrieve from the query alone. Others inject more context: the agent’s current reasoning, a reformulated question, or trajectory-derived signals. This axis decides how much hidden intent the retriever can see.

Fixed retriever vs. agent-specialized retriever

You can keep the agent fixed and adapt the retrieval side, or train the retriever directly from agent behavior. The more the supervision reflects real search trajectories, the more likely the retriever is to optimize for agent-useful evidence rather than generic relevance.

When Query Style and Retriever Style Clash

A good example of query mismatch comes from Sage: A Benchmark for Scientific Agen…. Its lesson is not that sparse retrieval always wins. The lesson is narrower and more important: retrieval quality depends on compatibility with the agent’s search behavior.

In Sage, the agent often emits subqueries that look more like targeted keywords than polished natural-language questions. Under that condition, BM25 beats LLM-based retrievers by roughly 30 percent on short-form corpus search. That result is conceptually important because it shows why “better retriever” is the wrong abstraction. A retriever that is strong in isolation can still be weak inside a planning-and-search loop if the query distribution is mismatched.

Sage also shows a practical way to think about corpus interaction. Instead of retraining the whole agent, it augments document representations with salient signals such as metadata and generated keywords. The idea is simple: if the agent searches in a keyword-like way, the corpus can be shaped to meet it halfway.

Search Quality Is Also About What You Return

The next bottleneck is not query wording but the shape of retrieved evidence. Text Ranking in Deep Research: Retrie… is useful here because it isolates familiar IR choices inside an agent loop rather than treating search as a black box.

Its main teaching point is that retrieval units and reranking depth matter as much as retriever family. Passage retrieval beats document retrieval in most settings, especially for agents with tighter context limits. BM25 on passages reaches the best reported first-stage accuracy in the study before reranking, and adding reranking improves performance further.

Why does this happen? Because a research agent benefits from evidence that is both relevant and compact. A full paper may contain the needed paragraph, but also pages of setup, related work, and experiments that crowd out future search turns. Passage retrieval turns the problem from “find the right paper” into “find the right evidence slice at the right time.”

The study also highlights query reformulation. Neural rankers often benefit when raw agent queries are rewritten into cleaner question-like forms. That is a reminder that search quality is not only about the index. It is also about how much translation the system performs between the planner’s rough subquery and the retriever’s preferred input format.

Ambiguous Queries Need Access to the Agent’s State

Some agent queries are underspecified on purpose. A subquery like “scaling law evidence for this setup” only makes sense if you know what the agent currently believes, what papers it has already seen, and which branch of the investigation it is pursuing.

Reasoning-Aware Retrieval for Deep Re… tackles this by conditioning retrieval on the agent’s current reasoning trace, not just the latest query string. The core idea is intuitive: the agent already knows why it is searching, so the retriever should be allowed to read that explanation.

This changes the retrieval contract. The input is no longer just a query. It is the query plus the current reasoning state. The output is still ranked evidence, but the ranking can now resolve ambiguity using turn-specific intent. The paper reports that current reasoning is a better signal than prepending the global question or feeding the full history, which is also a useful warning: more context is not always better. Old hypotheses and stale branches can dilute the real search need.

This is one of the clearest design axes in the area. The question is not whether context helps in the abstract. It is which context reflects the active search state without dragging in noise.

Training Retrieval From What Agents Actually Do

Even a well-shaped query is only part of the problem. The retriever may still be trained on the wrong notion of relevance. That is the retrieval mismatch problem addressed by Learning to Retrieve from Agent Traje….

The important shift in LRAT is where supervision comes from. Instead of relying on manual relevance labels or ordinary click-style assumptions, it mines signals from agent trajectories. Browsed documents become candidate positives, ignored ones become negatives, and the agent’s post-browse reasoning helps estimate whether a document truly advanced the search.

This is conceptually powerful because agentic relevance is different from static relevance. A paper can be topically related yet unhelpful for the current reasoning step. Another paper may be extremely useful because it resolves exactly the uncertainty driving the next action. Trajectory-based supervision is an attempt to train toward that agent-specific notion of usefulness.

The method also makes an important correction to naive trace mining: opening a document is a noisy signal. LRAT filters and reweights those positives using what happens after the browse step. In other words, it asks not only “Was this document opened?” but “Did it actually contribute to continued reasoning?”

What the Evidence Supports

The compact input supports a few durable conclusions.

BM25 beats LLM-based retrievers by roughly 30 percent on short-form corpus search in Sage, which is strong evidence that query-retriever compatibility matters.

Corpus-level test-time scaling in Sage improves BM25 by 8.18 percent on short-form questions, suggesting that adapting document-side signals can help even without retraining the agent.

The BrowseComp-Plus reproduction study finds that passage retrieval beats document retrieval in most settings, and that BM25 on passages is the strongest first-stage baseline before reranking.

Reasoning-Aware Retrieval reports strong gains from conditioning on current reasoning rather than broader or less targeted context, supporting the claim that active plan state is a real retrieval signal.

LRAT reports improvements in evidence recall, task success, and efficiency across benchmarks, which supports the broader idea that agent trajectories are a scalable supervision source for retrievers.

Open Questions That Still Matter

The biggest open question is how much of this subtopic is about better retrievers versus better interfaces between planner and retriever. If agents learned to generate cleaner subqueries, some current lexical advantages might shrink. If retrievers were trained directly on agent traces, some query reformulation might become unnecessary.

Another open question is how much context should be exposed to retrieval. Current reasoning helps, but full history can hurt. That suggests the real problem is context selection, not just context inclusion.

There is also a deeper evaluation question. Exact answers are useful for short-form tasks, but open-ended literature search needs metrics that reward partial evidence quality. A weighted-recall style view is helpful here because it asks whether the agent recovered the most relevant papers, not only whether it produced a single correct answer.

What To Remember

Agent-specialized retrieval is about aligning search with the way an autonomous research system actually thinks and acts. The retrieval problem changes once queries are generated by a planner, evidence is consumed under context limits, and relevance depends on the current reasoning state.

That is why this area keeps returning to the same practical lessons: sparse matching can beat semantic retrieval when subqueries are keyword-heavy, passages often beat full documents when context is scarce, current reasoning can disambiguate search better than raw history, and agent trajectories can provide better supervision than generic labels. The field is not converging on one shared mechanism. It is converging on the idea that retrieval must be specialized to the agent loop.
