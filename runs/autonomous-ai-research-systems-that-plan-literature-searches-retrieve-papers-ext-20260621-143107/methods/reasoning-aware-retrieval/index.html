Reasoning-Aware Retrieval for Deep Research Agents

Deep Research agents already produce something conventional retrievers never get to see: the reasoning they write right before they search. That is the missing signal in this paper. A query can be tiny and ambiguous, but the agent’s current reasoning often explains what that query really means. Once you read that working note, retrieval stops being a keyword lookup and becomes an intent lookup.

A small example makes the point. A search phrase like “backroom studio early 2010s euphoric” could point in several directions if you look at it alone. The agent’s reasoning reveals the actual hunt: it is narrowing a composer, an award, a country, and a genre. With that extra context, the retriever is no longer guessing at the user’s intent. It is reading the agent’s own explanation of the task.

That small change has a big effect. Instead of forcing the retriever to infer the missing context from the surface query, the system gives it the agent’s current plan state.

Why the bare query is too thin

The paper is aimed at a very specific failure mode. Deep Research agents do not search like humans issuing one-off web queries. They search in turns. Each turn is shaped by what the agent already learned, what it still lacks, and what hypothesis it wants to test next.

That means the latest query is usually only half the story. The agent may have already learned one clue from earlier searches, ruled out another, and formed a fresh guess from its own reasoning. A plain retriever sees none of that. It sees the last query string and tries to rank documents from that alone.

Reasoning-aware retrieval fixes that by reading three kinds of information at once:

  • what the agent is trying to prove right now,
  • what earlier searches have already settled,
  • what target or clue the agent thinks is promising next.

The paper’s useful comparison is with query expansion methods like HyDE. HyDE-style expansion asks another model to imagine helpful context from the query alone. Reasoning-aware retrieval uses something better: the agent’s own current reasoning, which is already conditioned on the full interaction history. In other words, the retriever is not inventing a proxy thought. It is reading the actual thought the agent just used to choose the search.

That also explains why the method is attractive operationally. The reasoning trace is already generated inside the agent loop, so the retriever gets this signal “for free.” There is no extra expansion call at search time.

What the retriever actually sees

The method does not try to stuff the whole search history into the embedding model. It uses the current reasoning trace and the current query together, then trains the retriever to treat that combined text as one retrieval input.

This matters because the current reasoning is not just another log line. It is a cleaned-up working state. If the agent has already learned that one award is the Grammy, or that a likely genre is progressive house, that clue is worth preserving. But if the agent has three turns of half-finished speculation behind it, the latest reasoning usually summarizes the useful parts and quietly drops the dead ends.

So the input is not raw memory. It is curated context.

How DR-Synth turns ordinary QA data into search-step supervision

The hard part is training. Standard question-answer datasets do not give you labels for multi-turn Deep Research sub-queries. They usually tell you the global question, the answer, and a set of documents that support the answer overall. That is not enough for this problem, because the retriever needs to learn what a good document looks like at each intermediate turn.

DR-Synth solves that by manufacturing turn-level supervision from rollouts.

The pipeline is easiest to understand as a loop:

  1. Start from a normal QA triple.
  2. Run a Deep Research agent on that question with a regular query-only retriever.
  3. Extract each search turn’s reasoning trace and query.
  4. For that turn, retrieve a candidate pool, then use an LLM reranker with the current query, the global question, and the true answer.
  5. Mark the top-ranked document as the positive example and the lowest-ranked documents as hard negatives.
  6. Keep only the rollouts that actually solved the original question.

Figure 2 shows the reranking step that turns a QA rollout into training labels.

What to notice in the figure is that the reranker is not trying to build a final answer. It is acting like a teacher that decides which document best fits the current search clue while still staying aligned with the overall question. The documents that survive this step become the training signal for the retriever.

That choice is important for two reasons. First, it avoids manual annotation for every intermediate search turn. Second, it gives the model hard negatives that are actually confusing, not just random off-topic junk. Those are the negatives that teach a retriever to distinguish a near miss from the right evidence.

In plain language, the training goal is to make the combined reasoning-plus-query text land near the right document and far from the plausible wrong ones.

Why the latest reasoning beats the whole history

One of the paper’s most interesting findings is that more context is not automatically better. You might expect that embedding every older query and reasoning trace would help, but the paper shows the opposite trend. The latest reasoning is usually stronger than a long uncurated history.

Figure 3 makes that visible. As more history turns are embedded, accuracy stops improving quickly, and the coverage of new clues shows diminishing returns.

The explanation is quite intuitive once you think about how the reasoning is produced. The current reasoning is already conditioned on the full interaction history. It often recaps the facts that matter and drops the speculative guesses that no longer matter. So if you pile on older turns, you are not adding a lot of new signal. You are mostly adding repetition.

The noise analysis in Figure 4 makes that even clearer.

What to notice here is that the latest reasoning can act like a filter. It keeps the confirmed facts and quietly stops carrying around the stale guesses that were useful in earlier turns but are no longer useful now. The paper’s interpretation is that this kind of forgetting is a feature, not a bug. It keeps the retriever focused on the current clue instead of dragging along every discarded hypothesis from the search trail.

That is the real lesson behind the history ablation. The best retrieval signal is not the longest trail. It is the most curated one.

What this changes in practice

Once trained this way, the retriever becomes much better at the job Deep Research agents actually need. It can disambiguate the current search turn, preserve the useful clues from earlier turns, and ignore the stale hypotheses that would otherwise mislead the search.

The paper’s trained model, AgentIR-4B, shows that this is not just a nicer input format. It improves end-to-end performance and reduces the number of search calls the agent needs. It also transfers to different agents without retraining, which is a strong sign that the method is learning a general search state rather than a single model’s private style.

Just as important, the gain does not come from adding another inference pass at search time. The reasoning trace is already part of the agent loop. The retriever simply learns how to use it better.

Where the method depends on the agent

The approach is powerful, but it is not free of assumptions.

  • It depends on explicit reasoning traces.
  • It uses synthesized supervision rather than hand-labeled Deep Research sub-queries.
  • It works best when the latest reasoning is a good summary of the current search state.

That last point is the subtle one. If the reasoning trace itself is noisy, the retriever can still inherit that noise. The method is strongest when the agent already does some internal curation, because the retriever then gets a compact view of the investigation instead of a raw transcript.

So the open problem is not whether to use more context. It is how to curate context well enough that the retriever sees the right slice of the search history.

What to remember

Reasoning-aware retrieval is not query expansion with a different name. It is retrieval conditioned on the agent’s current plan state.

DR-Synth makes that trainable by turning ordinary QA data into turn-level supervision through rollout reranking. And the paper’s main empirical lesson is memorable: when a searcher is already thinking out loud, the latest reasoning is often a better retrieval signal than the whole history.

If you want one sentence to keep in mind, make it this: the retriever should read the agent’s current thought, not just its latest search string.
