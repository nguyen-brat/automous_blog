Source Attribution Evaluation for Deep Research Systems

Deep research systems can produce a report that looks trustworthy even when parts of it are only loosely grounded. The link works, the page is on topic, the prose sounds careful, and still the exact sentence may not be supported. This paper is about separating those cases instead of collapsing them into one vague idea of “citation quality.”

The main move is easy to remember: turn a Markdown report into claim-citation pairs, fetch the cited pages, and judge each pair along three different dimensions. One check asks whether the URL is reachable. One asks whether the page is about the same thing as the claim. One asks whether the source actually supports the specific fact being attributed to it.

What to notice in Figure 1 is the shape of the system. This is not a single score over a finished report. It is a funnel. The report is broken apart, the cited sources are retrieved, and then the evaluator asks three separate questions about each claim-source pair.

That structure matters because the failure modes are different. A dead link is not the same problem as a mismatched citation. A mismatched citation is not the same problem as a source that exists but does not actually support the sentence next to it. If you only check one of those things, you miss the others.

Why a working link is not enough

The paper starts from a problem that deep research users already feel in practice. A report can look polished and still hide weak citation behavior. You might click the reference and land on a real page, but that page may be only loosely related to the claim. Or the page may be on the right topic but fail to support the exact numbers, dates, or statements that were written.

That is why the framework refuses to treat citations as one-dimensional. The evaluation is intentionally layered. The easiest question is accessibility. The next question is topical alignment. The hardest question is factual support.

Thinking in layers is useful because each layer isolates a different failure. If the link is broken, the problem is infrastructure or freshness. If the page is reachable but off-topic, the problem is source selection. If the page is on-topic but the claim is still unsupported, the problem is synthesis or attribution. Those are very different bugs, and they deserve different checks.

From Markdown to claim-citation pairs

The first technical trick is also the most practical one: parse the Markdown structurally instead of trying to guess with regex. Deep research reports already have a visual grammar. They use headings, paragraphs, links, footnotes, numbered references, and occasionally code blocks. A Markdown AST gives the evaluator a way to walk that structure directly.

The parser does a few things in sequence. It normalizes whitespace and line endings so the same report does not produce different spans just because of formatting. It strips fenced code blocks so examples or snippets do not accidentally become citation evidence. Then it traverses the AST and looks for the citation styles real reports use: numbered references, footnote-style references, inline Markdown links, autolinks, and citation ranges.

That is only half the job. The parser also splits the text into sentence-level claims and keeps track of which citation belongs to which span. A useful detail here is the backward attribution rule: when a citation appears at the end of a passage, it is treated as support for the uncited sentences that came before it in that same passage. That matches how people actually write research prose. One citation often supports a whole mini-argument, not just the last sentence in the paragraph.

The result is a structured set of claim-citation pairs. Repeated URLs are deduplicated, and each text span stays attached to the source it is supposed to rely on. That gives the evaluator a clean unit of work: one claim, one cited source, one judgment.

The three layers of verification

Once the parser has turned the report into pairwise units, the framework evaluates them from easiest to hardest.

Link Works

Link Works is the simplest check, and that is exactly why it belongs first. The evaluator tries to fetch the cited URL and asks whether the page is actually accessible. If the request fails with an error, times out, or is blocked, the citation fails this layer.

This sounds basic, but it is not trivial in a live-web setting. URLs change. Pages disappear. Paywalls and bot defenses interfere. A citation that looked fine when the report was written can become useless later. Link Works catches those failures directly, without asking a language model to reason about them.

The important idea is that a citation cannot support a claim if the page cannot even be opened. Accessibility is the floor, not the finish line.

Relevant Content

Relevant Content asks a different question: is this source actually about the same topic as the claim?

Here the evaluator uses an LLM-as-a-judge. It reads the attribution text and the retrieved source content, then decides whether the source addresses the same subject matter. The point is not to produce a long essay. The point is to make a constrained judgment that can tell the difference between “this page is real” and “this page is relevant.”

That distinction matters because a working URL can still point to the wrong page, a nearby page, or a source that only mentions the topic in passing. A link check alone would call all of those successes. Relevant Content separates the citations that are merely reachable from the citations that are actually pointing in the right direction.

Fact Check

Fact Check is the strictest layer. It asks whether the source content really supports the specific facts in the claim: the numbers, the dates, the named entities, the causal statements, and the concrete assertions.

This is where the evaluation gets interesting. A source can be reachable and topically aligned and still fail here. That is the paper’s central warning. Surface-level citation quality can look strong while factual grounding is weak.

The paper uses an LLM-as-a-judge for this layer too, but not as an open-ended reviewer. The judge works against a rubric and is calibrated through human review so it stays closer to the way a person would mark support, contradiction, or uncertainty. In plain language, the evaluator is not allowed to improvise. It has to answer a specific support question about a specific claim.

That design makes Fact Check the most useful diagnostic signal in the framework. It is the layer that tells you whether the citation is doing real work or just decorating the paragraph.

What the search-depth experiment reveals

Once those three checks are separated, the paper can ask a sharper question: what happens when the research agent searches more?

The answer is counterintuitive if you assume that more retrieval always helps. The framework shows that link accessibility and topical relevance stay fairly stable as search depth increases, but factual support gets worse. In the ablation study, the average Fact Check score drops by roughly 42 percent as tool calls scale from a shallow search to a deep one.

What to notice in Figure 2 is the asymmetry. The system is still good at finding live, relevant pages. The failure is downstream of retrieval. As the evidence pile grows, the model has more material to juggle, more chances to blend facts across sources, and more opportunity to attach the wrong source to a plausible sentence.

That is why the paper reads the result as an information-overload effect in synthesis, not as a search failure. The agent can keep finding the right kind of pages. The hard part is turning many pages into one accurate claim without letting the details blur together.

The other notable result is that open-source models often struggle even earlier. Many of them do not reliably produce valid cited reports in a one-shot setting, so the benchmark is not only measuring citation quality. It is also measuring whether the system can complete the reporting task at all.

What the framework is really teaching

The paper is not just saying “LLMs make citation mistakes.” It is teaching a more specific lesson: citation quality has multiple axes, and you need all of them to understand the failure.

The most useful mental model is a three-part lock.

  • Link Works asks whether the door opens.
  • Relevant Content asks whether you are in the right room.
  • Fact Check asks whether the thing you claimed is actually there.

If any one of those fails, the citation is weaker than it looks. A live link is not enough. A related page is not enough. A paragraph that sounds polished is not enough. The final claim has to survive all three questions.

That makes the framework valuable beyond its benchmark numbers. It gives deep research teams a practical way to debug citation behavior. If a system passes the first two checks but fails Fact Check, the problem is not browsing coverage. It is synthesis. If the system fails Link Works, the problem is freshness or access. If it fails Relevant Content, the search policy is drifting.

Where the method is still vulnerable

The paper is careful about its own limits.

  • LLM judges can still be biased, even when they are constrained by a rubric. The authors reduce that risk with human calibration, but they do not pretend the bias disappears.
  • Web citations are temporally unstable. A URL can vanish or change after the report is written, which means a pass today may not be a pass later.
  • The framework is built for live web search, not private enterprise corpora, so it does not automatically solve evaluation for internal RAG systems.

Those limitations do not weaken the main idea. They define its current boundary. The framework is strongest when the evaluation target is a public, web-based deep research system that produces Markdown with inline citations.

What to remember

The paper’s core lesson is simple and easy to carry forward: a citation can be reachable, on topic, and still fail to support the sentence it is attached to.

That is why the framework splits evaluation into accessibility, topical alignment, and factual support. It is also why the search-depth result matters. More search can improve coverage without improving truthfulness, and sometimes it makes the final synthesis worse.

If you remember one sentence from this chapter, make it this: a good citation is not just a live link, it is a live link that actually earns the claim sitting next to it.
