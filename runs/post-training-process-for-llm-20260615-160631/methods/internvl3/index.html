InternVL3

InternVL3 is trying to fix a very specific weakness in the usual multimodal recipe. Most MLLMs begin life as text-only LLMs, then get vision bolted on later through adapters, freezing schedules, or alignment stages. That works, but it also creates a translation problem: the model has to learn how to speak vision after it already learned how to speak language.

InternVL3 takes the opposite stance. It treats images, video, and text as first-class training material from the beginning, so the model learns the two modalities together instead of stitching them together afterward. That single change is the center of the paper. Everything else in the method is there to make that decision scale: the architecture keeps the familiar ViT-MLP-LLM stack, V2PE stretches visual context, post-training sharpens the behavior, and test-time reranking squeezes out extra reasoning quality.

What to notice in the figure is the shape of the system. InternVL3 is not a special-purpose vision encoder with a separate language assistant attached at the end. It is a single pipeline: a pretrained ViT turns pixels into visual tokens, a small MLP bridges them into the language space, and the LLM produces the final text. The important part is that the whole stack is trained together, so the interface between vision and language is learned instead of hand-wired.

The big change is not the architecture, it is the training philosophy

The architecture looks familiar on purpose. InternVL3 starts from pretrained vision and language backbones rather than training everything from scratch. That keeps the cost manageable and gives the model a strong starting point. The vision side uses an InternViT checkpoint, the language side uses a pretrained base LLM, and the MLP bridge is a small two-layer mapper.

There is also a practical efficiency trick in the vision front end. InternVL3 uses pixel unshuffle so each 448x448 image tile is represented with fewer visual tokens. The point is not to make the vision encoder cleverer. It is to make long multimodal sequences less expensive so the language model is not overwhelmed by a huge pile of image tokens.

The deeper shift is how the model learns. Instead of first building a polished text model and then teaching it to tolerate images, InternVL3 mixes pure text and multimodal corpora during pretraining. The visual tokens are not treated as a side channel. They sit inside the same autoregressive training process as the text.

That sounds subtle, but it changes the whole failure mode. If vision is added late, the model can become awkwardly split: one part of the network is still thinking in language-only habits, while another part is being bent toward grounding. InternVL3 tries to avoid that split by letting both abilities develop together.

Native multimodal pretraining keeps the model language-centered

The heart of the method is still next-token prediction, but with one important restriction: the loss is only computed on text tokens.

That matters because it keeps the generation objective centered on language. Visual tokens are used as context, not as something the model must predict back verbatim. In other words, the image helps the model answer the next text token, but the training target remains the text the user would actually read.

That is a good fit for multimodal assistants. When a user asks a question about an image, the model does not need to reconstruct the image. It needs to use the image to produce a useful answer. Text-token-only loss puts the gradient where that behavior lives.

InternVL3 also uses square averaging for token weights. The paper’s concern is that simple token averaging can overweight long answers, while sample averaging can overweight short ones. Square averaging sits in the middle. It gives longer responses enough room to matter without letting them dominate the gradient. You can think of it as a compromise between “every token counts equally” and “every sample counts equally.”

The other important training choice is joint parameter optimization. All parameters are updated together during native multimodal pretraining. There is no long phase where the LLM is frozen while only a small adapter learns the bridge, and there is no assumption that language and vision should evolve separately. This is the opposite of the post-hoc recipe. It says: if the model is supposed to reason over both modalities, then every layer should have a chance to adapt to both modalities.

Why V2PE exists

Once you start mixing long text and dense visual inputs, position handling becomes a bottleneck.

Standard positional encoding increments by 1 for every token. That is fine for pure text, but it burns through the context window too quickly when a sequence contains many visual tokens. InternVL3’s answer is Variable Visual Position Encoding, or V2PE.

The intuition is simple. Text still moves forward one step at a time, because lexical order matters. Visual tokens, however, can advance by smaller fractional steps. That way, a long image or image sequence does not consume the context window as aggressively as text would.

The paper keeps the relative position structure inside each image intact, but compresses the rate at which visual positions grow. During training, the fractional step is sampled from a small set of values. At inference time, it can be chosen more flexibly depending on the sequence length. If the model does not need the extra compression, V2PE can fall back to the ordinary behavior when the visual increment is 1.

This is not a glamorous trick, but it is a useful one. The core benefit is that the model can keep longer multimodal contexts in play without the positions of later tokens drifting out of range too quickly. That makes the method better suited to documents, multi-image inputs, and video-like sequences where the model has to carry more visual context at once.

Post-training is where the model gets polished

Native multimodal pretraining gives InternVL3 the basic habit of thinking across modalities. Post-training makes that habit more useful in conversation and reasoning.

The first stage is supervised fine-tuning. Here the model is trained on richer instruction data, including tool use, 3D scenes, GUI operations, long-context tasks, video understanding, scientific diagrams, creative writing, and multimodal reasoning. This stage matters because pretraining teaches broad competence, but SFT teaches the model how to respond in the style users actually expect.

The second stage is Mixed Preference Optimization, or MPO. MPO exists because there is a gap between training and inference. During pretraining and SFT, the model always conditions on the ground-truth previous tokens. During actual generation, it has to condition on its own earlier outputs. That mismatch can hurt chain-of-thought style reasoning, where a small early mistake can snowball into a bad final answer.

MPO addresses that by mixing three signals:

  • a preference loss that compares chosen and rejected responses,
  • a quality loss that judges each response on its own,
  • and a generation loss that keeps the model fluent and generative.

The paper’s version of the idea is useful because it does not rely on one blunt preference signal. The preference term says which answer is better. The quality term says how good each answer is in absolute terms. The generation term keeps the model from becoming a ranking machine that forgets how to speak naturally.

You can think of MPO as the stage where the model learns not just to like the right answer, but to produce it reliably under its own decoding behavior. That is why it helps reasoning. The paper reports that MPO gives a clear bump on multimodal reasoning benchmarks, and the effect is larger at bigger scales.

Test-time scaling is a reranking problem

InternVL3 does not stop at training. At inference time, it uses Best-of-N with VisualPRM as the critic.

The idea is straightforward: generate several candidate solutions, score them, and keep the best one. That is especially helpful when the task is reasoning-heavy, because the model may produce multiple plausible-looking answers but only one actually carries the cleanest step-by-step logic.

VisualPRM is trained to judge solutions step by step. It scores each step in a solution and then averages those scores to get an overall quality estimate. In the paper’s setup, the scoring is framed as a multi-turn interaction so the critic can examine the solution incrementally rather than only at the end.

That gives InternVL3 a useful second pass at test time. The base model proposes. The critic selects. This does not replace learning better reasoning, but it makes the model’s strongest candidate more likely to survive when several outputs look similarly plausible.

The important thing to remember is that this is reranking, not magical extra knowledge. It is a way of spending more compute at inference time to let the model choose a better path among several near-misses.

What the method is really doing end to end

If you follow one input through the full method, the flow looks like this:

text, image, video, or interleaved multimodal sequence -> pretrained ViT and pretrained LLM -> joint native multimodal pretraining with text-token loss -> V2PE for longer multimodal context -> SFT on richer instruction data -> MPO for better reasoning under self-generated outputs -> Best-of-N reranking with VisualPRM when the task needs extra care

That flow is the paper’s main idea in one line. InternVL3 is not trying to glue vision onto a finished chat model. It is trying to make multimodal behavior emerge inside the same pretraining and post-training stack that already shapes language behavior.

The reason that matters is visible in the results the paper highlights. InternVL3-78B reaches 72.2 on MMMU, which is a new open-source high watermark. It also posts strong OCR and visual question answering scores, and the post-training stack meaningfully improves reasoning. The exact numbers are less important than the pattern: the native recipe gives the model broad competence without sacrificing language ability.

Where the recipe still bends

The method is strong, but it is not free.

The paper is explicit that grounding gains plateau at the largest scale. That means there is still a ceiling where simply making the model bigger no longer solves localization as cleanly as it improves other skills. Some benchmarks also remain out of reach of the best proprietary systems, so the open-source lead is real but not absolute.

There is also a systems cost. Native multimodal pretraining only works because the team can support large mixed pretraining and fairly heavy infrastructure. That is part of the tradeoff of removing the awkward alignment stack: you replace some of the algorithmic glue with more demanding joint training.

So the right mental model is not “InternVL3 solved multimodal alignment once and for all.” It is “InternVL3 shows that you can move multimodal learning earlier in the pipeline and get a cleaner, stronger base model, but scale and grounding still need careful treatment.”

What to remember

The easiest way to remember InternVL3 is this:

learn language and vision together, not in two disconnected phases.

Everything else follows from that.

  • Native multimodal pretraining makes vision tokens part of the main training story.
  • Text-token-only loss keeps the model language-centered.
  • V2PE stops visual context from exhausting the position budget too quickly.
  • SFT and MPO turn the pretrained model into a better reasoning assistant.
  • Best-of-N with VisualPRM lets the model spend extra inference compute on choosing the best answer.

That combination is why InternVL3 matters. It is a concrete example of a multimodal model that treats images as first-class training context from the start, rather than as an add-on glued onto a finished language model.
