Superpixel-based and Spatially-regularized Diffusion Learning for Hyperspectral Image Clustering

S^2DL is a reduction-first diffusion clustering pipeline for hyperspectral images. It does not try to make every pixel carry the full burden of graph construction. Instead, it first groups the image into spatially coherent superpixels, then keeps only a few high-density representatives from each region, then builds a spatially regularized diffusion graph on those representatives, and only then spreads labels back to the full image. That ordering is the whole trick: compress the scene without destroying its local geometry.

A good way to think about the method is as a neighborhood council. Each superpixel is a small local district. S^2DL keeps only the most representative voices from each district, lets diffusion clustering decide which voices belong together, and finally lets each district vote internally. The method is designed for hyperspectral scenes where local spatial coherence is real, but raw pixel-wise graphs are too large, noisy, and easy to fool.

Compared with SRDL and the other graph-based baselines in the family, S^2DL pushes the spatial idea earlier in the pipeline. It does not merely regularize a graph after the fact. It reduces the image first using superpixels, and that reduction already encodes a lot of the scene structure the later graph stage needs.

Here is the whole pipeline in one view.

What to notice in the figure is the order. The image is first compressed by ERS superpixels. Representative pixels are then selected inside each superpixel. The graph is built only on those representatives, and the graph itself is spatially regularized so it cannot connect arbitrary distant pixels. After diffusion mode finding and label propagation, the method ends with a majority vote inside each superpixel. Almost every design choice is there to keep the graph small without letting it lose the scene’s spatial structure.

Why superpixels come first

ERS is the first gate in the pipeline. It works on the first three principal components of the hyperspectral image, which gives it a compact view of the scene before segmentation starts. That matters because S^2DL does not want raw spectral noise to decide the boundaries of the reduction step. It wants a coarse but meaningful spatial partition.

ERS builds superpixels by optimizing a graph objective that rewards compact, homogeneous regions while also keeping the partition balanced.

Symbols:

  • A: the selected edge subset that defines the current partition.
  • A^*: the edge subset that maximizes the ERS objective.
  • E: the original graph edge set.
  • \mathcal{J}(A): the ERS objective.
  • \mathcal{H}(A): the entropy-rate term that favors compact, homogeneous regions.
  • \mathcal{B}(A): the balance term that discourages badly unbalanced superpixel sizes.
  • \alpha: the tradeoff between homogeneity and balance.
  • N_A: the number of connected components induced by A.
  • N_s: the target number of superpixels.

Read-aloud meaning: pick the partition that makes each region internally consistent while still producing the requested number of reasonably even superpixels.

Why it matters: this is the compression step that preserves local image structure instead of discarding it.

ERS is not just a preprocessing convenience. It decides which pixels are allowed to matter later. A random subsample might be smaller, but it would not know the difference between a stable patch of crop rows and a noisy boundary fragment. ERS keeps the local patch structure intact, which is exactly what S^2DL needs before it starts building a diffusion graph.

The optimization itself is conceptually simple even if the graph bookkeeping is not: start from an empty edge set, add edges with a lazy greedy rule, and stop once the requested number of superpixels has been reached. The result is a partition that tries to be compact without becoming fragmented or wildly uneven.

Keeping only the pixels that actually speak for a region

Once the image is split into superpixels, S^2DL does not keep every pixel inside them. It scores pixels with a local density estimate based on their nearest neighbors and retains only the top k pixels inside each superpixel. In plain language, it keeps the pixels that are most typical of their local neighborhood.

That choice is a better anchor strategy than random sampling. Random anchors can land on an outlier, a boundary pixel, or a rare noise spike. Density-selected representatives are more likely to sit near the center of a stable local mode, so they are safer inputs for later graph analysis. And because the selection happens inside each superpixel, every coherent region gets a chance to contribute representatives instead of letting one large class dominate the graph.

If you want a mental picture, imagine one crop patch that is almost uniform. You do not need every interior pixel to represent it. You only need a few pixels that are clearly inside the patch and clearly typical of it. S^2DL is doing exactly that, but over the whole hyperspectral image.

Diffusion on a graph that respects the image grid

The selected representatives are then connected by a sparse kNN graph, but with one extra constraint: edges are only allowed inside a spatial radius. That is the spatial regularization step. It prevents the graph from linking pixels that are spectrally similar but spatially implausible.

This matters because hyperspectral data are not just high-dimensional vectors. They are measurements on a 2D scene. Two pixels may look similar in spectrum while still belonging to unrelated parts of the image. The radius constraint keeps diffusion honest by making it pay attention to the scene layout.

On top of that graph, S^2DL uses diffusion distance as its notion of similarity.

Symbols:

  • D_t(x_i,x_j): diffusion distance between pixels x_i and x_j at time t.
  • t: diffusion time, which sets the scale of structure being observed.
  • N: the number of graph nodes used in the diffusion computation.
  • \lambda_k: the k-th eigenvalue of the transition matrix.
  • \psi_k: the k-th eigenvector of the transition matrix.
  • (\psi_k)_i: the i-th entry of \psi_k.
  • (\psi_k)_j: the j-th entry of \psi_k.

Read-aloud meaning: compare two points by asking whether the graph spreads mass outward from them in the same way after t diffusion steps.

Why it matters: diffusion distance captures nonlinear cluster geometry better than plain Euclidean distance, which is exactly what hyperspectral scenes need.

The diffusion time t is a scale knob. Smaller values emphasize local structure; larger values see broader cluster organization. The paper reports a fairly wide stable window for t, which is useful because it means the method is not extremely brittle to the exact time choice. The formula also explains why the method can be approximated efficiently in practice: the contributions from smaller eigenvalues die off quickly, so the leading diffusion directions matter most.

How cluster modes are chosen

S^2DL keeps a density-peak intuition, but it replaces Euclidean separation with diffusion separation. A good mode should be both dense and far from any denser point in diffusion space. That is what the score \Delta_t(x) is doing.

Formula card: diffusion-density mode score

Purpose: rank representative pixels so the best cluster exemplars are both dense and well separated.

Δt(x)=dt(x)ζ(x)\Delta_t(x)=d_t(x)\zeta(x)Δt​(x)=dt​(x)ζ(x)

Intuition: a pixel should become a mode only if it is locally crowded and also far from any higher-density representative under diffusion geometry.

Symbols:

  • \Delta_t(x): the final mode score.
  • d_t(x): diffusion-distance separation for x.
  • \zeta(x): the local density score for x.
  • x: a representative pixel in the reduced set.

Read-aloud meaning: multiply local density by diffusion separation, so a pixel wins only when both signals support it.

Why it matters: this is the step that turns a reduced graph into actual cluster exemplars instead of just a new neighborhood graph.

The logic of d_t(x) is easy to remember even if the paper writes it carefully. If a point is already the densest one, its separation is measured against the farthest other representative. Otherwise, its separation is the diffusion distance to the nearest representative that is at least as dense. In either case, the method is asking the same question: does this point stand apart from anything denser than itself?

That is why the paper can treat the chosen modes as reliable seeds. They are not just popular. They are popular in the right part of the graph. This is also the place where S^2DL most clearly inherits the spirit of diffusion peak clustering, but with a geometry that is better matched to hyperspectral structure.

From modes to a full image

After the modes are fixed, S^2DL labels the nearby representatives first. The local backbone around each mode gets the same label as the mode itself. This is a small but practical safeguard: it gives each cluster a coherent seed neighborhood before the remaining points are assigned one by one.

Then the rest of the representative pixels are processed in descending density order. Each one inherits the label of the nearest already-labeled representative with higher density in diffusion distance. That makes the labeling step follow the same trust hierarchy as the mode selection step. Labels spread from reliable exemplars to less reliable points, not the other way around.

Finally, the method returns to the superpixels. It assigns every pixel in a superpixel the majority label of its representatives. This last vote is what converts the reduced graph solution back into a dense pixel map. It is also where the method’s main assumption becomes visible: pixels inside the same superpixel are supposed to belong together often enough that local consensus is meaningful.

If the superpixel happens to cross a true class boundary, this final vote can blur a thin structure. That is the price of the speedup. S^2DL is deliberately choosing a useful approximation of the scene, not claiming that every pixel inside every patch is perfectly pure.

Why the method is fast

The speedup is not magic. It comes from shrinking the graph before diffusion has to work on it. The paper argues that the computational burden shifts away from full graph analysis and toward the density estimation used to choose representatives. In other words, S^2DL spends its money where it buys the most structure.

That is why the method can be both accurate and practical. On Salinas, the paper reports OA 0.887 with a runtime of 8.80 seconds, which is a good illustration of what the reduction step buys you. The graph is still doing real geometric work, but it is doing that work on a much smaller and cleaner set of points.

The more important takeaway is not the exact runtime number. It is that the method keeps the expensive part tied to a reduced representation that still preserves local spatial regularity.

Where it helps, and where it can blur

S^2DL is strong when the scene has local spatial coherence. That is exactly the case in many hyperspectral images, where nearby pixels often belong to the same material region. In those settings, superpixels are a useful compression unit, and diffusion distance can recover cluster geometry on the reduced graph.

The cost of that design is also clear.

  • The method depends on several tuned hyperparameters, especially the number of superpixels, the spatial radius, and the number of representatives per superpixel.
  • If a superpixel crosses a real class boundary, majority voting can blur thin or rare structures.
  • If representative selection misses a rare local mode, that mode may never get a chance to influence the final map.
  • The method is most natural for grid-structured hyperspectral data, not arbitrary tabular clustering.

That is the right tradeoff for the paper’s target setting. It is not trying to solve all clustering problems. It is showing that spatially aware reduction can make diffusion clustering much more scalable on HSIs without throwing away the geometry that matters.

What to remember

The simplest way to remember S^2DL is this:

  • Use ERS to turn the image into coherent superpixels.
  • Keep only the densest representative pixels in each superpixel.
  • Build a spatially regularized diffusion graph on those representatives.
  • Pick modes by density and diffusion separation.
  • Push labels back to the full image by local backbone and majority vote.

If you remember just one sentence, make it this: S^2DL makes diffusion clustering cheaper by reducing the image in a way that still respects local spatial structure.
