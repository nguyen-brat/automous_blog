Unsupervised Manifold Linearizing and Clustering

MLC sits at a very specific intersection: it wants the geometry wins of subspace clustering, but it starts from data that are too curved for a plain subspace model to be honest. The paper’s basic claim is that you can learn a representation and a clustering graph together so that each hidden cluster becomes easier to approximate as a low-rank subspace, while different clusters separate cleanly enough for spectral clustering to finish the job.

The best way to think about it is as a flattening process. Each cluster is a bent sheet hidden inside a high-dimensional space. MLC does not try to name the sheet first. It first learns features that straighten the sheet, and at the same time learns a balanced affinity graph that says which points should stay together. Once those two pieces start agreeing, clustering becomes much easier.

The method in one picture

The architecture is simpler than the paper’s name suggests. A shared backbone feeds two heads: one head produces the latent representation, the other produces pairwise affinities. Those affinities are then balanced with a Sinkhorn projection so they behave like a proper clustering graph rather than an arbitrary similarity table.

What to notice in the figure is the split between representation and membership. Z is the space the model wants to improve. Gamma is the structure the model wants to infer from that space. MLC trains both together, so the graph and the features keep pushing each other in the right direction instead of being learned in separate stages.

What the objective is trying to do

The core idea comes from maximal coding rate reduction, or MCR^2. The objective is designed to keep the whole batch informative while making each cluster-like group internally compact enough to look low-rank.

Formula Card: MLC objective

Purpose: learn features that are globally spread out, but compressible within the groups suggested by the learned affinity.

Equation:

R(Z;ϵ)−Rc(Z,Γ;ϵ)R(\mathbf{Z};\epsilon)-R_c(\mathbf{Z},\mathbf{\Gamma};\epsilon)R(Z;ϵ)−Rc​(Z,Γ;ϵ)

Intuition: the first term rewards a feature space that uses its volume well, while the second term subtracts the coding rate inside each affinity-weighted group. So the model is pushed to keep different clusters apart without collapsing each cluster to a single point.

Symbols:

  • \mathbf{Z} is the learned feature matrix for the current batch.
  • \mathbf{\Gamma} is the learned pairwise membership affinity.
  • \epsilon is the coding precision scale.
  • R is the global coding-rate term that rewards diversity in the full batch.
  • R_c is the cluster-conditioned coding-rate term that rewards compressibility inside learned groups.

Read-aloud meaning: make the whole batch diverse, but make each learned cluster simpler and lower rank.

Why it matters: this is the exact tension that lets MLC both separate clusters and preserve useful within-cluster variation.

In the supervised version of MCR^2, the cluster groups are known and the objective can use the true labels. MLC keeps the same geometric target, but replaces the labels with a learned affinity graph. That is the key move: instead of asking for a point-cluster assignment first, it asks for a structured pairwise graph that can later be turned into a partition.

Why the affinity is point-to-point instead of point-to-cluster

This is the most important design change in the paper. Many deep clustering methods predict an n x k membership matrix, where each point chooses one of k clusters. MLC does something different. It learns an n x n affinity matrix, so the model reasons directly about which points should be close to which other points.

That sounds less direct at first, but it is actually easier to stabilize. A pairwise graph is much closer to the object spectral clustering wants anyway. It is also easier to initialize from feature similarities than a full point-to-cluster classifier is. The paper takes that extra step because it helps with the chicken-and-egg problem: good features help clustering, and good clustering helps features.

Formula Card: Doubly stochastic affinities

Purpose: keep the learned point-to-point similarities balanced enough to behave like a real clustering graph.

Equation:

Ω={Γ∈Rn×n:Γ≥0, Γ1=Γ⊤1=1}\Omega=\{\mathbf{\Gamma}\in\mathbb{R}^{n\times n}:\mathbf{\Gamma}\ge 0,\ \mathbf{\Gamma}\mathbf{1}=\mathbf{\Gamma}^{\top}\mathbf{1}=\mathbf{1}\}Ω={Γ∈Rn×n:Γ≥0, Γ1=Γ⊤1=1}

Intuition: each point sends out one unit of affinity mass and also receives one unit. That balance prevents the graph from becoming arbitrarily peaked or arbitrarily dense.

Symbols:

  • \mathbf{\Gamma} is the affinity matrix.
  • n is the batch size or dataset size being modeled.
  • \mathbf{1} is the all-ones vector.

Read-aloud meaning: the affinity matrix is nonnegative, and every row and every column sums to one.

Why it matters: this is the structure Sinkhorn can project onto, and it is the reason the learned affinities behave like a balanced clustering graph instead of a free-form similarity table.

The paper also explains why this balance needs an entropy bias. If you maximize a convex objective over the doubly stochastic polytope without any extra regularization, the optimizer is tempted to move toward an extreme point, which for doubly stochastic matrices is a permutation matrix. That would make the graph too sharp and too isolated: each point would act like its own tiny cluster. Negative entropy regularization keeps the matrix from hardening too much, so the graph can still merge points that belong together.

The practical effect is easy to miss but important: MLC is not trying to produce a one-hot assignment inside the affinity head. It is trying to produce a balanced, cluster-friendly graph that spectral clustering can read at the end.

How the method gets off the ground

The method would be fragile if it started from random weights. The paper is explicit about that. Before the MLC objective is optimized, the feature branch is warmed up with self-supervised learning so that augmentations of the same sample land near each other and different samples do not collapse.

That warm start does two jobs at once. First, it gives Z some structure that already respects augmentation invariance. Second, it gives the affinity branch something meaningful to copy from. After the feature head has been initialized, the cluster head can be seeded from feature similarities, and the Sinkhorn projection can turn that seed into a doubly stochastic matrix in one shot.

This is why the initialization is not a minor implementation detail. It is the bridge between a self-supervised feature space and a clustering graph. The ablation in the paper makes the point brutally clear: replacing the self-supervised initialization with random initialization drops CIFAR-10 accuracy to 20.0%.

The same logic applies during training. Augmentation is used again while optimizing the MLC objective, not just at the beginning. The model keeps seeing different views of the same underlying sample, which makes the learned representation less brittle and the learned affinities less dependent on one exact pixel arrangement.

Following the data through the pipeline

Once the pieces are in place, the flow is straightforward.

The shared backbone processes a mini-batch. The feature head turns those samples into normalized embeddings Z. The cluster head turns the same batch into an intermediate matrix C, whose inner products produce a raw similarity structure. Sinkhorn projection then balances that raw structure into Gamma, which satisfies the doubly stochastic constraint.

At that point, the MLC objective compares two things:

  • how much volume the whole batch occupies in feature space, and
  • how much volume remains inside the affinity-weighted groups.

Optimization nudges the feature head so cluster members become easier to flatten into low-rank subspaces, and nudges the affinity head so the graph better reflects that emerging geometry. In the reported experiments, the backbone is kept fixed during this phase, while the two heads are updated batch by batch.

The last step is still spectral clustering. MLC does not pretend that the learned affinity matrix is itself the final partition. Instead, it uses the learned graph as the input to a conventional spectral method that converts the soft structure into discrete labels. That is consistent with the whole design: learn a better graph first, then let the partitioner do the final cut.

Why this is more than a clustering trick

The paper’s more interesting claim is not just that the clusters are accurate. It is that the learned features themselves look more like a union of subspaces. In other words, the representation is becoming easier to cluster because it is being linearized in the right way.

That shows up in the paper’s comparisons. On CIFAR-10, MLC reaches 86.3% ACC, and the learned features improve downstream subspace clustering as well. For example, EnSC goes from 72.2% ACC on self-supervised features to 81.5% ACC on MLC features. That is a useful signal: the representation is not just helping MLC’s own spectral step, it is actually making the geometry more subspace-like.

The same pattern is visible in the failure modes. If the warm start is weak, the affinity graph stays noisy. If augmentation is removed from the update step, performance drops. If the final spectral stage is removed, the learned graph no longer gets converted into a discrete partition. So the method works because the pieces reinforce each other, not because one piece alone is magical.

What to keep in mind

MLC is powerful, but it is not free of assumptions.

It depends heavily on a good self-supervised initialization. It is demonstrated mainly on image datasets, so the transfer story beyond vision is still open. And even after the whole training loop, spectral clustering is still a separate final step.

Those limits are worth remembering because they explain the method’s shape. MLC is not trying to solve unsupervised clustering from nothing. It is trying to take a noisy nonlinear clustering problem, warm-start it with a self-supervised representation, and then jointly refine features and affinities until the hidden manifolds look much closer to orthogonal subspaces.

If you want one memory hook, use this:

warm-start the features, balance the affinity graph, and let the representation flatten the manifolds for spectral clustering.

That is the whole method in one line.
