CPP: Clustering via the Principle of Rate Reduction and Pretrained Models

CPP is a clustering pipeline for unlabeled images that treats strong pretrained features as a starting point rather than the final answer. The paper’s bet is simple: CLIP already knows a lot about semantics, but its geometry is not yet shaped for clustering. So CPP freezes the CLIP image encoder, adds small trainable heads, refines the embedding with a rate-reduction objective, learns a balanced pairwise affinity, and then uses spectral clustering, code-length selection, and CLIP text voting to turn that structure into clusters with names.

If you remember one sentence, remember this: CPP does not replace CLIP, it repairs the feature space CLIP gives it.

What to notice in the figure is the three-stage flow. Training first reshapes the representation, the middle stage picks the number of clusters without retraining, and the final stage turns each cluster into a caption. The backbone stays frozen; the extra work happens in the heads and the post-processing.

Start from CLIP, then make the space cluster-friendly

CLIP features are already semantically strong, but the paper shows that they can still be too correlated across classes. That is a problem for clustering, because a good clustering space needs two things at once: clusters should be spread out from one another, and each cluster should still have internal structure instead of collapsing to a point.

CPP attacks that geometry problem directly. It starts with frozen CLIP image features, then runs a short warmup that only maximizes global rate reduction. That early diversification step gives the later joint optimization a better starting point than random heads would.

Formula Card: MLC objective

Purpose: Learn features and pairwise affinities that are globally spread out but locally compact.

Equation:

max⁡η,ω  R(Z;ε)−Rc(Z,Π;ε)\max_{\eta,\omega}\; R(\mathcal{Z};\varepsilon)-R_c(\mathcal{Z},\Pi;\varepsilon)η,ωmax​R(Z;ε)−Rc​(Z,Π;ε)

Intuition: Spread the whole embedding cloud apart, but make each cluster occupy a tighter subspace inside that cloud.

Symbols:

  • Z\mathcal{Z}Z: learned feature representation after the trainable head.
  • Π\PiΠ: pairwise clustering membership matrix.
  • RRR: global coding-rate term that rewards overall spread.
  • RcR_cRc​: clusterwise coding-rate term that measures the spread left inside clusters.
  • ε\varepsilonε: coding precision scale.
  • η,ω\eta, \omegaη,ω: trainable parameters of the feature head and the cluster head.

Read-aloud meaning: Increase overall feature volume while reducing the volume that remains inside each cluster.

Why it matters: This is the core mechanism that turns CLIP’s generic semantics into clustering geometry.

You can think of the objective as a volume game. RRR asks how much space the whole dataset occupies. RcR_cRc​ asks how much space is still occupied once you condition on clusters. CPP wants the first number high and the second number low. The result is a union-of-subspaces picture: each cluster can still spread out internally, but different clusters point in different directions.

The paper’s visual evidence matches that story. The cosine-similarity matrix ∣Z⊤Z∣|\mathcal{Z}^\top \mathcal{Z}|∣Z⊤Z∣ becomes much more block diagonal after training, which is the visual signature of clusters occupying different subspaces instead of one crowded direction.

Why the affinity matrix is pairwise instead of direct labels

CPP does not learn an n×kn \times kn×k classifier head that bakes the number of clusters into training. It learns an n×nn \times nn×n membership matrix Π\PiΠ, where each entry says how strongly two samples should be linked. That is a more flexible object: the same trained affinity can later be partitioned into different kkk values.

To keep that affinity honest, CPP projects it onto the doubly stochastic set. Every row and column sums to one, so no sample can dump all of its mass into a few neighbors and quietly create a degenerate cluster. The entropy knob γ\gammaγ controls how sharp that matrix becomes. Larger γ\gammaγ makes the affinity smoother and more uniform. Smaller γ\gammaγ makes it more selective and reduces false cross-cluster links, but it also weakens inter-cluster connectivity if you push it too far.

This is one of the paper’s quiet but important design choices. A soft pairwise affinity is easier to reuse later than a hard cluster assignment would be.

Why the warmup exists

The optimization is nonconvex, so initialization matters. CPP does not start from random trainable heads and hope for the best. It first runs a brief warmup with the rate-reduction term alone, usually only one or two epochs, to make the features more diverse.

Then it seeds the cluster head from the feature head rather than from unrelated random weights. That simple copy is easy to miss, but it keeps the two heads aligned at the start of the full MLC optimization. In other words, the cluster head begins by looking at the same geometry the feature head has already learned to create.

Choosing the number of clusters without retraining

This is where the pairwise affinity pays off. Because CPP never hard-codes kkk into the training objective, the same trained representation can be reused for many candidate cluster counts. The model does not have to be rerun from scratch every time you want to test a different kkk.

Formula Card: coding length for model selection

Purpose: Score how expensive a representation is to describe.

Equation:

L(Z)=(n+d)R(Z;ε)L(\mathcal{Z})=(n+d)R(\mathcal{Z};\varepsilon)L(Z)=(n+d)R(Z;ε)

Intuition: A representation that needs many bits to describe is paying for complexity. A better cluster count should reduce the total description cost once labels are taken into account.

Symbols:

  • Z\mathcal{Z}Z: feature matrix.
  • nnn: number of samples.
  • ddd: feature dimension.
  • RRR: global coding-rate term.
  • ε\varepsilonε: coding precision.

Read-aloud meaning: The code length grows with how many samples and feature dimensions you need to describe.

Why it matters: This gives CPP a way to choose kkk after training instead of retraining for every candidate.

Algorithmically, the loop is straightforward. Pick a maximum plausible KKK. Run spectral clustering on the same affinity matrix for each candidate kkk. Compute the code cost for the resulting cluster-wise features and labels. Then choose the kkk with the smallest total cost.

That is why the paper can recover 10 clusters for CIFAR-10 and 20 for CIFAR-100 without retraining. The cluster count is not forced into the learned representation. It is selected afterward, from the same affinity geometry.

Turning clusters into words

CPP is not satisfied with hard cluster IDs. It also wants a human-readable caption for each group. That is where CLIP’s text-image bridge comes back into the story.

The idea is not to invent labels from scratch. Instead, the paper builds a candidate vocabulary, compares cluster images against those candidate texts with CLIP similarity, and votes for the label that best fits the cluster as a whole. In practice, the candidate set mixes LLM-generated labels with ImageNet class names, because a static high-quality vocabulary is ideal but hard to obtain at scale.

This step is useful, but it is also the most vocabulary-dependent part of the pipeline. If the candidate list is thin or biased, the caption will be too. So the captioning stage should be read as semantic naming, not as a guaranteed ontology.

The nice part is that it fits the same pattern as the rest of CPP. The method does not ask CLIP to solve everything in one pass. It uses CLIP where it is strongest: semantic similarity between images and texts.

Why the learned space is better for search, not just clustering

The training objective is easy to motivate with clustering, but the payoff is broader. Once the embedding space becomes more block structured, nearest-neighbor search gets better too. That is because the learned geometry starts to group semantically related images together instead of merely matching surface shape.

What to notice here is the difference between visual resemblance and semantic resemblance. In the paper’s example, CPP treats a shampoo bottle as a daily-hygiene object and returns that kind of neighborhood. Plain CLIP leans more toward cylindrical lookalikes. That is a good sanity check for the whole method: the representation is not only clusterable, it is more meaning-aware.

The same improvement shows up in the paper’s cosine-similarity plots and PCA/SVD views. CLIP features are relatively compact, but CPP spreads the data across more useful dimensions and makes the cluster blocks easier to see.

What the paper shows in practice

The strongest anchor result is on ImageNet-1k. CLIP plus KMeans gets 49.2 ACC, while CPP reaches 66.2 ACC and 86.8 NMI. That is a large jump, and it comes from reshaping the pretrained geometry rather than replacing the backbone.

The paper also shows that CPP beats MLC, SCAN, IMC-SWAV, and TEMI on the standard benchmarks it reports, while converging in fewer epochs than older deep clustering pipelines. More importantly, the same recipe extends to larger uncurated collections such as MS-COCO, LAION-Aesthetics, and WikiArt, where both cluster count selection and semantic captioning matter.

The practical message is not just “higher accuracy.” It is that a pretrained embedding can be turned into a complete clustering system if you add the right geometry repair and the right post-hoc selection step.

Where CPP can still stumble

CPP is strong, but it is not magic.

  • It is focused on image clustering rather than general high-dimensional modalities.
  • Caption quality depends on the candidate text vocabulary.
  • The final pipeline still relies on spectral clustering and coding-length heuristics after training.
  • If the pretrained features are weak or the dataset is not semantically organized, refinement can still merge or split concepts poorly.

That last point matters. The paper itself shows that not every pretrained backbone is equally useful here. The approach works best when the starting representation already carries some semantic structure that can be sharpened rather than invented from nothing.

What to remember

CPP is really three problems glued together carefully: geometry repair, model selection, and semantic naming. CLIP gives the raw semantic field, MLC makes that field clusterable, coding length chooses how many clusters to keep, and CLIP text voting gives the clusters names humans can read.

If you want the one-line memory hook, use this:

frozen CLIP -> rate-reduction refinement -> doubly stochastic affinity -> spectral clustering by code length -> CLIP-based captions

That is why the method feels practical. It turns pretrained features into a complete unsupervised pipeline rather than just another embedding.
