Scalable Incomplete Multi-View Clustering with Structure Alignment

SIMVC-SA is built around a simple idea: when views are incomplete, you should not force them to agree too early. Each view may see a different subset of the samples, so the anchor geometry learned in one view can drift away from the anchor geometry learned in another. If you collapse everything into one shared graph before that mismatch is handled, you risk mixing the views in a way that looks consistent but actually erases useful structure.

The paper’s answer is to keep the views separate for one more step. Each view learns its own compact anchor graph first. Only after that does SIMVC-SA align those graphs softly into a shared space and cluster from the aligned result. In other words, it draws several compressed maps of the same territory, then rotates those maps into one atlas.

Why the anchor problem appears in incomplete views

The paper’s first figure makes the failure mode concrete. Missing samples and random anchor initialization can push different views into different anchor layouts, so the anchors are no longer comparable in the way a shared clustering pipeline would like them to be.

What to notice here is that the anchor mismatch is not a minor nuisance. Once the observed samples differ across views, the anchors can become structurally unaligned even if they are each locally reasonable inside their own view. That is the anchor-unaligned problem for incomplete data, and it is exactly why a naive shared representation can become misleading.

The key lesson is that incomplete multi-view clustering needs two things at once:

  • preserve each view’s local structure;
  • recover cross-view correspondence without pretending the views are identical.

SIMVC-SA is designed to satisfy both constraints at the same time.

Let each view keep its own map first

The first thing SIMVC-SA does well is resist the temptation to over-share. For view v, the method works with the observed samples only, using an index matrix H_v to select the samples that are actually available in that view. The expression X_vH_v is therefore not a full data matrix for the whole dataset; it is the data that view v can really see.

From there, the method learns a view-specific anchor matrix A_v. You can think of anchors as a small set of representative landmarks for the view. Instead of storing an all-pairs graph, the method stores how each observed sample relates to those landmarks. That relation is the anchor graph Z_v.

This matters for two reasons.

First, the graph is much cheaper than a full sample-by-sample similarity matrix, so the method can scale to large datasets.

Second, the graph is view-specific by design. The anchors in one view do not have to be identical to the anchors in another view before the model has figured out how the views relate.

The orthogonality constraints on A_v are there for a reason. They keep the learned anchors from collapsing into a degenerate code and make the anchor update behave more like a clean linear-algebra problem than a brittle nonlinear search. The same logic applies to the final fused representation F: the paper keeps it orthogonal so the shared space stays discriminative.

The anchor graph itself is also constrained to stay meaningful. Z_v is nonnegative and column-normalized, so each sample gets a soft distribution over anchors rather than an arbitrary unconstrained vector. That makes the graph easier to interpret as a compressed similarity structure.

Soft alignment instead of strict matching

The most important new piece in SIMVC-SA is not the anchor graph itself. It is the way the method aligns those graphs.

If you try to force one anchor in view v to match one specific anchor in another view, you are assuming the views already share the same geometry. In incomplete data, that is often false. The views may have different missing samples, different feature dimensions, and different local density patterns. A strict one-to-one correspondence can therefore be too brittle.

SIMVC-SA softens that requirement. It introduces an orthogonal alignment matrix P_v for each view and asks whether the whole anchor graph can be brought close to the shared fused representation F. That is a much more forgiving question. It allows the graph to move as a whole, rather than demanding that every anchor have a single perfect twin somewhere else.

That is also why the method does not need a hand-picked reference view. Some earlier methods align every view to the first view, or keep choosing a reference view during the iterations. SIMVC-SA instead learns F directly, so the shared target is adaptive rather than inherited from one possibly bad view.

The orthogonality constraint is the quiet but important detail here. It keeps the alignment soft. Instead of allowing arbitrary squashing or stretching, P_v behaves like a rotation or reflection in anchor space, which is much better matched to the idea of “same structure, different coordinate system.”

How the alternating solver moves the pieces together

The optimization problem is not convex when all variables are considered at once, so the paper solves it by alternating between smaller subproblems. That is the practical reason the method remains usable.

Updating the anchors

When the graphs, alignments, fused representation, and view weights are fixed, each anchor matrix A_v can be updated independently. The paper turns that block into an orthogonal Procrustes-style problem and solves it with singular value decomposition. The important idea is that anchor learning is not bolted on as a preprocessing step. The anchors keep adapting to the current graph structure while the optimization runs.

That is better than fixed k-means initialization for two reasons. The learned anchors can better reflect the current incomplete-view geometry, and the method avoids the extra cost of repeatedly running k-means.

Updating the anchor graphs

Once the anchors and alignments are fixed, the graph update becomes a column-wise projection step. Each sample keeps a soft, nonnegative distribution over the anchors, but the distribution must still sum to one. In plain language, the sample is allowed to be explained by several anchors, but it cannot spread its mass arbitrarily.

This is where the Newton step appears in the paper: the one-dimensional shift used in the projection is solved efficiently, and then the result is clipped to stay nonnegative. The exact algebra is not the main thing to remember. The important part is that the update stays closed-form-ish and local, which is what keeps the whole alternating loop practical.

Updating the shared representation

The fused representation F is the consensus target all views are trying to reach. The paper updates it from the accumulated aligned graphs using another SVD-based step. You can think of this as extracting the strongest shared directions after each view has been rotated into the same space.

That makes F more than a simple average. It is a structured consensus that keeps the dominant shared geometry while ignoring the view-specific noise that should not be carried into clustering.

Updating the alignment matrices

The alignment matrices P_v are updated in the same spirit as the anchors: each one is solved as an orthogonal subproblem. This is the “soft rotation” step that lets each view join the shared atlas without being forced into a hard correspondence table.

Updating the view weights

Not every view should matter equally. Some views reconstruct their observed samples much better than others, especially under missingness. SIMVC-SA therefore learns a weight for each view. A view that is currently well explained gets more influence in the next round; a weak or noisy view gets less.

Put together, these alternating blocks do something very deliberate. They let each view rebuild its own compressed geometry, they align those geometries softly, and they then refine the shared representation from the aligned pieces. That is why the method can repair cross-view mismatch instead of amplifying it.

Why the method scales

The scalability claim is not just a benchmark talking point. It comes directly from the data structures the method chooses.

Full graph-based IMVC methods pay for pairwise sample relationships, which become expensive very quickly. SIMVC-SA replaces those dense graphs with anchor graphs, so the main objects are sample-to-anchor matrices rather than sample-to-sample matrices. That is the reason the method can stay roughly linear in the number of samples for fixed anchor count.

There is a tradeoff, though. The alignment machinery adds extra work, and that work grows with the number of views. The paper is explicit about that: SIMVC-SA is still efficient, but it is not free. If you have many views, you pay for more alignment steps.

The optimization itself is also non-convex, so the alternating solver converges to a local optimum rather than a guaranteed global one. That is normal for this class of methods, but it is still worth remembering.

The remaining limitation is more structural: the paper notes that future work is needed for anchors with different counts. In other words, the current alignment is soft, but it still assumes a compatible anchor count across views. A more flexible many-to-many correspondence model could be a natural next step.

What the experiments suggest

The reported results match the design pretty well. SIMVC-SA beats or matches strong incomplete multi-view baselines on the main benchmarks, with especially clear gains over FIMVC-VIA on datasets such as ProteinFold, BDGP, SUNRGBD, NUSWIDEOBJ, and Cifar10. The alignment ablation is the strongest evidence that the new module matters: when structure alignment is removed, accuracy drops sharply on several datasets. The learned anchors also outperform fixed k-means initialization, which is exactly what you would expect if the anchor geometry itself needs to adapt to incomplete data.

There is another useful pattern in the plots: performance stays relatively stable as the missing rate increases. That stability is a better sign than a single high score, because the whole problem is about surviving missingness without letting the representation drift apart.

What to remember

SIMVC-SA is easiest to remember as “preserve local anchor structure first, then align softly.”

Each view draws its own compressed map.

The maps are not forced to match immediately.

Instead, orthogonal alignment matrices rotate those maps into one shared representation.

Then the method clusters from that aligned consensus.

That order is the real insight. For incomplete multi-view data, it is usually safer to keep view-specific structure alive a little longer and only then ask the views to agree.
