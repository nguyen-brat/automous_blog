Self-Expressive Network

Self-Expressive Network, or SENet, takes a classic subspace-clustering idea and turns it into a reusable rule. In the old formulation, each point is reconstructed from the other points, and the coefficients that make that happen are solved directly for a specific dataset. That is a strong idea, but it is also a brittle one: the coefficient table is quadratic in the number of samples, and nothing in the solution tells you how to handle a point you have never seen before.

SENet keeps the same intuition and changes the object being learned. Instead of optimizing a fresh coefficient matrix every time, it learns a pairwise function that predicts how much one point should help reconstruct another. Once that function is trained, the model can score new points without rebuilding the whole coefficient table from scratch.

This architecture is worth reading from left to right. Each point goes through one network to produce a query embedding and another to produce a key embedding. The pairwise coefficient then comes from the inner product of those embeddings, followed by a thresholding step that removes weak relations. The model is not trying to classify points directly. It is trying to generate a coefficient graph that spectral clustering can use later.

Why the old self-expressive model hits a wall

The classical self-expressive model says that a point should be explained by other points from the same subspace. If the coefficients are allowed to be optimized directly, the solution often has a useful block structure: points inside one subspace explain one another, while cross-subspace links stay small. That block structure is exactly what spectral clustering wants to see.

The problem is the price of that table. A coefficient matrix for N samples has N x N entries, which is fine for a small dataset and painful for a large one. Even worse, those coefficients are tied to the specific dataset they were optimized on. If a new point arrives, the old table does not automatically tell you what to do.

SENet treats that as the real bottleneck. The method is not trying to invent a different clustering objective. It is trying to make the self-expressive rule reusable.

The pairwise score is the heart of the model

Formula Card: Self-expressive coefficient function

Purpose: define the learned rule that generates one coefficient from a source point and a target point.

Equation:

f(xi,xj;Θ)=α Tb(uj⊤vi)f(\mathbf{x}_i,\mathbf{x}_j;\Theta)=\alpha\,\mathcal{T}_b(\mathbf{u}_j^\top \mathbf{v}_i)f(xi​,xj​;Θ)=αTb​(uj⊤​vi​)

Intuition: the model learns two embeddings for each point, scores their compatibility with an inner product, and then prunes weak scores with thresholding.

Symbols:

  • f(\mathbf{x}_i,\mathbf{x}_j;\Theta): the predicted coefficient telling how much point i contributes when reconstructing point j
  • \Theta: all trainable parameters in the query and key networks, plus the threshold
  • \alpha: a fixed scale factor that keeps the raw score from becoming too large
  • \mathcal{T}_b: soft-thresholding with threshold b
  • \mathbf{u}_j: query embedding of target point \mathbf{x}_j
  • \mathbf{v}_i: key embedding of source point \mathbf{x}_i

Read-aloud meaning: the coefficient is a scaled, thresholded similarity between a target embedding and a source embedding.

Why it matters: this is the compact mechanism that replaces a dataset-specific coefficient table with a learned pairwise generator.

The important part is not just that the model uses two embeddings. It is that the embeddings are learned separately for the target and the source point, so the network can ask a directed question: “How much should point i help explain point j?” That direction matters because self-expression is not just about neighborhood similarity. It is about building a reconstruction that stays inside the right subspace.

The threshold is doing real work

Formula Card: Soft-thresholding

Purpose: turn dense similarity scores into sparse coefficients.

Equation:

Tb(t)=sgn(t)max⁡(0,∣t∣−b)\mathcal{T}_b(t)=\mathrm{sgn}(t)\max(0,|t|-b)Tb​(t)=sgn(t)max(0,∣t∣−b)

Intuition: subtract a threshold from the magnitude of the score, and clip anything too small to zero.

Symbols:

  • t: the raw inner-product score before pruning
  • b: the threshold that controls how aggressively weak relations are removed
  • \mathrm{sgn}(t): the sign of the score

Read-aloud meaning: keep only the part of the score that is large enough to survive the threshold.

Why it matters: self-expressive clustering needs sparse but not empty affinity graphs. Thresholding is the mechanism that turns a smooth similarity score into a usable sparse coefficient pattern.

This is why the paper does not stop at an inner product. A plain similarity score would usually be too dense. The threshold makes the output behave more like the sparse self-expressive solutions that classical subspace clustering already knows how to use.

Why this is not just attention

SENet looks a little like Transformer-style self-attention at first glance, but the design goals are different.

  • The query and key maps are multilayer perceptrons, not simple linear projections. That gives the coefficient function more expressive power.
  • The method does not force coefficients to be a convex combination. That matters because some points, especially vertex-like points in a subspace, cannot be reconstructed well if every coefficient has to behave like a probability weight.
  • The output is thresholded to promote sparsity. Self-attention usually cares about mixing information smoothly; SENet cares about keeping only the relations that help the subspace graph stay clean and connected.

So the resemblance is structural, not conceptual. Both methods score pairwise interactions. Only SENet is trying to make those scores behave like self-expressive coefficients for clustering.

Sparse, but not too sparse

The regularizer is the other half of that design.

Formula Card: Elastic net regularizer

Purpose: keep the coefficient graph sparse while avoiding the disconnected graphs that pure sparsity can create.

Equation:

r(c)=λ∣c∣+1−λ2c2r(c)=\lambda|c|+\frac{1-\lambda}{2}c^2r(c)=λ∣c∣+21−λ​c2

Intuition: the \ell_1 term pushes weak coefficients to zero, while the quadratic term keeps the graph from becoming too brittle.

Symbols:

  • r(c): penalty applied to one coefficient
  • c: a self-expressive coefficient
  • \lambda: balance parameter between sparsity and density

Read-aloud meaning: use both a sparsity penalty and a gentle quadratic penalty on each coefficient.

Why it matters: pure sparsity can delete too many same-subspace links and leave the affinity graph fragmented. Elastic net keeps the graph sparse enough to be useful and dense enough to stay connected.

That same tradeoff shows up in the soft-thresholding layer. The network needs small coefficients to vanish, but it also needs enough edges left over for spectral clustering to see the block structure. SENet is really trying to thread that needle.

Training without materializing the whole matrix

The training objective is still self-expressive at its core: for each target point, the network tries to reconstruct it from the other points and pays a penalty for using the coefficients it predicts. The difference is that the coefficients come from the neural function above, so the model can be trained with standard gradient-based methods.

The naive way to train this would be expensive, because each target point touches all the source points. SENet avoids that with a two-pass SGD strategy. The first pass computes the reconstruction residual for a target point by accumulating the predicted contributions from the other points. The second pass reuses that residual to accumulate gradients online, without having to store the entire N x N computation graph.

In practice, the memory story is the important part:

  • the query embeddings can be computed independently for all points
  • the key embeddings can be computed independently for all points
  • pairwise scores are just inner products, so they can be batched or parallelized
  • the gradient can be accumulated in streaming fashion instead of holding the whole table in memory

That is why the paper can talk about out-of-sample inference and large-scale clustering in the same breath. The method is not free, but it is no longer chained to one fixed coefficient table per dataset.

What the coefficients look like when it works

When the model is doing its job, the coefficient matrix becomes visually blocky. Coefficients stay concentrated inside subspace blocks, and cross-subspace links fade away. On synthetic data, the paper shows that this structure appears after training and can be produced for unseen test points as well.

The important thing to notice in the heatmap is not the exact color scale. It is the shape. Same-subspace relations accumulate inside blocks, which is exactly the sort of structure spectral clustering likes. The same trained SENet can also infer coefficients on out-of-sample data drawn from the same distribution, which is the practical payoff of learning a reusable coefficient rule instead of a one-off matrix.

One subtle point from the synthetic experiments is worth keeping in mind: the network is not trying to become a perfect reconstructor of unseen points. The test reconstruction loss can stay high even when the coefficient structure becomes more subspace-preserving. In other words, the learned rule is mainly useful because it captures which points should connect, not because it exactly reproduces every test vector.

Large-scale clustering benchmarks

The large-scale clustering benchmarks are where the method’s practical value becomes obvious.

On CIFAR-10, SENet reaches 0.765 ACC versus 0.613 for EnSC. If the soft-thresholding layer is removed, the score drops to 0.610, which is a strong sign that the thresholding step is not cosmetic. On MNIST-test, the method approaches the convex baseline as the amount of training data grows, reaching 96.66% ACC with 10k training samples. On Fashion-MNIST-test, the same trend appears more modestly, with SENet eventually getting close to EnSC. On EMNIST, SENet reaches 0.721 ACC while EnSC exceeds 24 hours, which is the clearest evidence that the learned pairwise function changes the scale at which the method is usable.

Those numbers are not just leaderboard decoration. They support the core claim that a learned coefficient generator can match a direct convex self-expressive solver closely enough to be useful, while also giving you out-of-sample reuse and much better large-scale behavior.

Where SENet can stumble

The method still rests on the subspace assumption. If the data are not well modeled as a union of low-dimensional subspaces, the learned coefficients will have a harder time staying subspace-preserving.

The network size matters too. If the query and key MLPs are too small, the learned coefficient function may not approximate the target self-expressive solution well enough. The paper shows that this approximation quality improves as the network gets more expressive, but it is still an approximation, not an exact solver.

There are also pipeline-level limits. SENet still uses spectral clustering at the end, so it inherits the usual sensitivity of graph-based clustering. And for the real-image benchmarks, the method still depends on dataset-specific feature pipelines before the clustering stage. So the method scales the coefficient inference problem, but it does not erase every preprocessing or downstream clustering choice.

What to remember

SENet is easiest to remember as “learn the self-expression rule once, then reuse it.”

The whole paper is really about that one shift. Classical self-expressive clustering optimizes a coefficient table for one dataset. SENet learns a pairwise scoring function with query and key MLPs, sparsifies it with thresholding and elastic-net pressure, and then feeds the resulting affinity graph to spectral clustering. That gives you the original subspace-clustering intuition, but with reusable inference and much better large-scale behavior.

If you want the shortest mental model, keep this sentence: SENet replaces a quadratic coefficient matrix with a learned pairwise rule that stays sparse enough for clustering and reusable enough for new data.
